# 大地图分层渲染黑屏根因与契约

## 2026-06-18 复现结论

用户提供的 `wxgame-oplog-test1-20260618-152155Z.json` 显示，复现过程中服务端地图数据没有消失：相关 action 响应里 `wmTiles=25`。黑屏来自前端 map-home 分层合成协议。

关键日志特征：

- `worldMap:hitTargetsSynced` 在问题期间反复出现 `mapTargetCount=27 -> 0 -> 27 -> 0`。
- 黑屏帧常见组合是 `baseHitTargetCount=27`、`hitTargetCount=27`、`sourceHitTargetCount=0`、`mapTargetCount=0`、`preserved=true`。
- 这说明 runtime 正在保留旧 hitTargets，但当前视觉底图帧没有被证明有效。

## 根因

原协议把两个不同事实混在一起：

1. 输入层事实：`WorldMapRuntimeHitTargetPolicy` 可以在 snapshot、drag、actor-only 帧保留稳定地图 hitTargets，避免 partial renderer source 污染输入。
2. 视觉层事实：`worldMap` baked layer 是否已经绘制、未 dirty、backing store 未变化、snapshot render 是否真的 `drewFrame`。

当上层只根据 `worldMapLayerRendered` 或已有 hitTargets 设置 `skipWorldMapLayer/preserveCanvas` 时，`mainHud` 会跳过底图绘制并继续画上层遮罩。若底图 backing store 已失效、snapshot miss 或未提交 baked 状态，就表现为地图区域黑屏。

## 修复契约

新增统一的 frame/composition state：

- `visualLayerValid`：底图视觉层可复用的唯一门禁。
- `hitTargetsPreserved` / `hitTargetsFresh`：输入缓存状态，只能说明 hitTargets 来源，不能证明底图可复用。
- `mapTargetCount` / `sourceHitTargetCount`：保留诊断日志里的关键计数字段。
- `worldMapRuntimeContext`：actor/HUD/picking 所需的当前地图上下文。

硬规则：

- `skipWorldMapLayer` 只能由 `WorldMapRuntimeRenderPolicy.canSkipWorldMapLayer(frameState)` 决定。
- `preserveCanvas` 只能在 `visualLayerValid=true` 时开启。
- `hitTargetsPreserved=true` 不能让任何入口跳过底图。
- snapshot/drag/actor-only 入口如果没有 `drewFrame`，不得提交 baked layer 状态。
- H5 Shell、多层 canvas 入口和 App/minigame 单 canvas 入口必须消费同一套 policy，不能各自复制判断。

## 落地模块

- `WorldMapRuntime.syncHitTargetsFromRenderer()` 继续返回 hitTargets 数组以保持兼容，同时记录 `lastHitTargetSync`。
- `WorldMapRuntime.getWorldMapFrameState()` 暴露 runtime 当前组合状态。
- `WorldMapRuntimeRenderPolicy` 增加 frame state 与 composition option 纯函数。
- `CanvasGameShellRenderingRuntime`、`CanvasGameAppRenderingRuntime`、`CanvasGameAppWorldMapRuntimeBridge` 统一通过 policy 决定 `skipWorldMapLayer/preserveCanvas`。

## 回归覆盖

- preserved hitTargets + invalid baked layer => `skipWorldMapLayer=false`。
- preserved hitTargets + valid visual layer => 允许 overlay 复用。
- App snapshot miss => 不跳过底图、不 preserve canvas。
- snapshot frame 只有在 `drewFrame !== false` 后才提交 baked layer 状态。

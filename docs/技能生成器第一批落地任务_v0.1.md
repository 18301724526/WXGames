# 技能生成器第一批落地任务 v0.1

日期：2026-05-29
实现版本：`0.1.148`
提交口径：每次实现提交必须带版本号和简介，并在本文记录改动、测试与 commit。

## 目标

本任务把 `docs/战斗系统设计_v0.2.md` 中已经确认的技能生成与战斗释放规则落到代码第一版，让新生成名人拥有 `abilityKit`，战斗报告按确定性条件释放主动技能，并向前端提供技能 cut-in 所需字段。

## 本次范围

- 新增技能生成器 `skill-gen-v1`。
- 名人生成时写入 `abilityKit`。
- 战斗型和斥候型名人保留 `skills[0]` 作为主动战法兼容字段。
- 内政/魅力型名人生成两个内政技能，战斗中只普攻，不注入 fallback 战斗技能。
- 第一批战斗效果池只开放：
  - `directDamage`
  - `secondHit`
  - `firstStrike`
  - `lifesteal`
  - `heal`
  - `shield`
  - `attributeBonus`
- 主动技能按 `castPolicy: 'conditional'` 和 `castConditions` 判断释放。
- 技能行动写入 `actorPortrait` 与 `presentation.cutIn`，供前端技能立绘飘出表现使用。
- 版本号从 `0.1.147` 升到 `0.1.148`。

## 本次不做

- 不实现 `burn`、`poison`、`armorBreak`、`lowSoldierGuard` 的真实状态结算。
- 不实现内政技能在城市、岗位、事件或外交中的生效入口。
- 不做岗位系统。
- 不做武将升级、属性点分配、品质保底重构。
- 不做敌方守军完整随机技能生成器；守军暂时保留固定模板。

# 常规事件系统设计 v0.1

## 目标

常规事件用于给玩家提供持续的小决策：事件以标签红点的形式进入队列，玩家主动打开事件页并选择处理方式。第一版不做自动弹窗、不做隐藏随机结果，避免打断当前建造、资源与引导节奏。

## 设计原则

- 后端驱动：事件生成、选项效果、资源扣减、奖励、惩罚、buff 生效和过期全部由后端决定。
- 前端渲染：前端只展示 `eventQueue`、红点、详情弹窗和后端下发的选项。
- 明示选择：第一版每个选项展示 `preview`，玩家能预期结果。
- 低压力：常规事件默认不过期，队列达到上限后不再生成。
- 与特殊事件并存：`special` 事件仍用于教程和时代关键节点，`regular` 事件用于日常循环。

## 生命周期

```text
状态同步/玩家操作/服务器 tick
→ 后端清理过期 buff
→ 检查常规事件生成条件
→ 到达 nextAt 且队列未满则生成一个 regular 事件
→ 写入 eventQueue
→ 生成时立即安排下一次 nextAt
→ 前端事件标签红点显示数量
→ 玩家选择选项
→ 后端应用 effects
→ 事件从 eventQueue 移到 eventHistory
```

## 生成规则

- 常规事件只在新手闭环完成后生成：`tutorial.completed === true` 或 `tutorial.phaseCompleted.era2 === true`。
- 时代至少为聚落时代：`currentEra >= 2`。
- 队列上限：常规事件最多 3 个；特殊事件不计入常规事件上限。
- 时间间隔：第一版固定 240 秒，后续可改为 180-300 秒随机。
- 避免重复：最近 3 个模板不重复；如果模板池不足则允许回退。

## 第一批模板

| 模板 | 类型 | 选择 |
|---|---|---|
| 丰收的预兆 | 资源/buff | 获得食物；或消耗食物换食物产出 buff |
| 迷路的行商 | 资源交换 | 给食物换知识；或给木材换食物 |
| 学者的争论 | 资源/buff | 消耗食物获得知识；或开启知识产出 buff |
| 族人请求休整 | 幸福/buff | 消耗食物换离线收益 buff；或获得少量幸福感叙事奖励 |
| 旧屋修缮 | 资源交换 | 消耗木材换幸福；或拆除旧料获得木材 |
| 猎人带回消息 | 资源/风险 | 获得食物；或消耗知识换更多食物 |

## Buff 第一版范围

- `resourceMultiplier` + `target: food|knowledge|wood`
- `offlineEfficiencyBonus`
- `happinessFlat`

资源产出 buff 由 `ResourceTickCalculator` 统一读取。离线收益 buff 由离线收益计算读取。幸福度 buff 在派生幸福度时叠加。

## 前端

- 事件红点继续使用 `eventQueue.length`。
- 事件弹窗改为根据 `event.options` 渲染多个按钮。
- 每个按钮展示 `label` 和 `preview`。
- 点击按钮调用 `claimEvent(eventId, optionId)`。
- 历史记录展示玩家选择的选项和结算摘要。

## 后续扩展

- 事件过期和倒计时留给威胁事件。
- 概率、隐藏结果、连锁事件留到系统稳定后扩展。
- 战争系统事件复用同一队列，新增 `type: "threat"`。

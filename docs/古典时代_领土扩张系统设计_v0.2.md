# 古典时代领土扩张系统设计 v0.3

> 状态：本版实施范围文档
> 日期：2026-05-17

## 目标

古典时代的重点不是预设一排可占领地块，而是让玩家从自己的城市出发，逐步发现外部世界。当前游戏页面仍然以城市管理为核心；底部只保留“军事”入口，军事页内分为“军队 / 侦察 / 世界”三个子视图。侦察完成后由后端生成周边世界点和文字报告，世界子视图用雷达式地图展示已经发现的地点。

```text
边境时代完成防御准备
→ 进入古典时代
→ 军事页·侦察子视图派出八方向侦察
→ 侦察完成，后端生成报告与已知世界点
→ 军事页·世界子视图在雷达坐标中显示地点
→ 玩家对已发现地点发起占领
→ 占领后命名城市
→ 第二块控制地后命名势力
```

## 设计原则

- 后端驱动：地点生成、坐标、规模、威胁、报告、占领结果和加成都由后端计算和持久化。
- 动态地图：服务器只保存玩家已经知道的世界点，不预生成整张地图。
- 军事内聚合：不新增独立疆域底部标签；军事页内通过子视图切换军队、侦察和世界。
- 八方向侦察：玩家从军事页·侦察子视图选择北、东北、东、东南、南、西南、西、西北进行侦察。
- 城市视角：主体验仍是城市经营；外部世界通过报告和疆域地图逐步显现。
- 轻地图：第一版不使用地形贴图，不展示方格网；用雷达式同心圈、方位感和透明底城池/营地/遗迹图标表达外部世界。
- 兼容士兵模型：仍不拆守卫/战士，出征士兵只是暂时进入任务。

## 世界点类型

| type | 表现 | 默认归属 | 用途 |
|---|---|---|---|
| capital | 大城 | player | 玩家起点 |
| outpost | 小型据点 | neutral | 低风险扩张目标 |
| town | 村镇/小城 | neutral | 中等扩张目标 |
| camp | 部落营地 | tribe | 带威胁和叙事接触 |
| ruins | 遗迹 | neutral | 知识收益、较高风险 |

## 侦察规则

- 古典时代后开放侦察。
- 侦察从玩家已控制地的边界向外扩张，但第一版坐标可以由“离首都最近的未发现坐标”决定。
- 八方向坐标：
  - 北 `(0, -1)`
  - 东北 `(1, -1)`
  - 东 `(1, 0)`
  - 东南 `(1, 1)`
  - 南 `(0, 1)`
  - 西南 `(-1, 1)`
  - 西 `(-1, 0)`
  - 西北 `(-1, -1)`
- 如果目标方向最近一格已发现，继续沿同方向寻找下一格。
- 侦察任务完成后，后端生成一个新的世界点和一份侦察报告。
- 前端不自己生成地图内容，只渲染后端返回的 `territoryState.territories`、`warMissions` 和 `scoutReports`。

## 生成规则

第一版使用确定性模板，避免完全随机导致测试和调参困难：

| 距离 | 可能发现 | 规模 | 威胁 |
|---:|---|---:|---:|
| 1 | outpost / town | 1-2 | 1-3 |
| 2 | town / camp / ruins | 2 | 3-5 |
| 3+ | camp / ruins / town | 2-3 | 5+ |

后端根据方向、距离、已发现数量和时代选择模板。后续可以加入科技、事件、敌方势力和随机种子。

## 占领规则

- 只有 `status: "discovered"` 的世界点可被占领。
- 玩家派出推荐士兵数或更多士兵发起占领。
- 出征期间士兵计入 `soldiersOnMission`，暂时不可再次派出。
- 任务完成后玩家点击“完成占领”领取结果。
- 成功后地点变为 `occupied`，进入城市命名。
- 失败后地点保持 `discovered`，记录上次战斗结果。

第一版结算保持确定性：

```text
出征战力 = 派出士兵数 × 1
如果 出征战力 >= 地点防御：
  占领成功
  伤亡 = floor(地点防御 / 3)，至少 0，最多 派出士兵 - 1
否则：
  占领失败
  伤亡 = ceil(派出士兵数 / 2)
```

## 命名规则

- 首都可改名。
- 每个 `occupied` 世界点都可设置 `cityName`。
- 新地点刚占领且未命名时，前端弹出城市命名弹窗。
- 当已占领地点数达到 2，且 `polity.name` 为空时，前端弹出势力命名弹窗。
- 后端校验命名长度：1-12 个字符，去除首尾空白。

## 前端展示

- 底部导航不新增“疆域”入口，统一进入“军事”。
- 军事页内提供三个子视图：
  - **军队**：士兵数量、防御、训练进度、可用士兵、出征中士兵。
  - **侦察**：八方向罗盘式控制。按钮按真实方位围绕首都排布，状态分为可派出、侦察中、报告待领取。
  - **世界**：可拖动雷达式地图、地点详情弹窗和侦察报告。
- 世界地图展示规则：
  - 中心是首都。
  - 已发现地点按后端坐标换算为雷达方位和距离放置。
  - 不渲染方格、不渲染未发现格子。
  - 我方地点用势力色光晕表达控制范围。
  - 中立、部落、遗迹用不同状态色。
- 世界地图交互：
  - 玩家可用鼠标或手指拖动雷达盘查看已发现区域。
  - 地图提供“回到本城”按钮，一键将首都重新居中。
  - 点击雷达盘上的地点图标弹出详情窗。
  - 详情窗显示地点名、类型、归属、状态、距离、规模、威胁、防御、报告摘要和可用操作。
- 地图下方显示最近侦察报告。
- 不再在雷达下方渲染独立“已知地点”卡片列表，避免地图和列表形成两套主入口。

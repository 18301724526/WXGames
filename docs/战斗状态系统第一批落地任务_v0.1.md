# 战斗状态系统第一批落地任务 v0.1

日期：2026-05-30
目标版本：`0.1.162`

## 目标

把战斗技能中的状态效果从“战报文字”推进为真实结算机制，让后续技能生成器不再只能生成一次性伤害换皮。

第一批只做确定性、可解释、容易测试的状态，不引入战斗中随机触发。

## 本批范围

- 后端战斗单位保留并结算 `statuses`。
- 行动单位正式行动前，先结算自己身上的持续状态。
- 技能效果可以施加真实状态。
- 战报 `turns` 和 `detailEvents` 记录状态触发、状态施加、护盾抵消和状态过期。
- 前端战斗场景根据战报展示轻量状态飘字。

## 第一批状态

| 状态 | key | 结算方式 | 玩家表现 |
| --- | --- | --- | --- |
| 守御 | `shield` | 自身获得护盾值，后续受到伤害时先抵消护盾。 | `守御 +N`、`守御抵消 N` |
| 破甲 | `armorBreak` | 目标受到后续兵刃伤害提高，持续按目标行动前扣减。 | `破甲` |
| 灼烧 | `burn` | 目标行动前受到一次谋略持续伤害，持续按目标行动前扣减。 | `灼烧 -N` |
| 中毒 | `poison` | 目标行动前受到一次谋略持续伤害，持续按目标行动前扣减。 | `中毒 -N` |

## 结算顺序

```text
行动单位准备行动
→ 结算行动单位自身的持续伤害状态
→ 扣减行动单位自身状态持续次数
→ 移除过期状态
→ 若行动单位兵力归零，本次不再行动
→ 判断技能冷却和释放条件
→ 执行技能或普攻
→ 结算伤害、护盾抵消和技能附加状态
→ 若任一方兵力归零，清空相关状态
```

## 数值约束

- `shield.value` 按最大兵力比例生成护盾，默认 `0.08`。
- `shield` 总量上限为自身最大兵力 `30%`。
- `armorBreak.value` 默认 `0.12`，总增伤上限 `30%`。
- `burn/poison.value` 默认 `0.12`，每次触发按谋略伤害公式和该倍率动态计算。
- 状态默认持续 `2` 次目标自身行动。
- 同类负面状态最多 `3` 层；超过上限只刷新持续时间。

## 本批不做

- 不做暴击、闪避、反击。
- 不做净化、免疫、驱散。
- 不做复杂 UI 状态栏，只做战斗飘字和战报行。
- 不把状态接入内政、侦查或大地图。
- 不扩大普通技能池。已有技能先获得真实结算，后续再扩展技能模板。

## 人工测试用例

1. 用带 `shield` 的统帅或守军进入战斗。
   - 预期释放技能后出现 `守御 +N`。
   - 后续受击时出现 `守御抵消 N`，实际损兵少于原始伤害。

2. 用测试技能或后续生成技能施加 `burn/poison`。
   - 预期目标下一次行动前先出现 `灼烧 -N` 或 `中毒 -N`。
   - 目标可能因持续伤害败退并跳过本次行动。

3. 用测试技能施加 `armorBreak`。
   - 预期目标身上存在破甲时，后续受到兵刃伤害提高。
   - 状态持续次数按目标自己的行动前减少。

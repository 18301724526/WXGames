# 文明火种 — 知识产出机制重构与前期节奏调优

> 状态：已定稿 | 配套文档：v2.1 聚落时代策划案、v2.0 数值框架 | 日期：2026-05-16

---

## 一、问题诊断

### 1.1 玩家卡点在哪

当前（调优前）农耕→聚落时代最优路径推演：

| 步骤 | 动作 | 耗时 | 资源变化 |
|------|------|------|---------|
| 1 | 进入农耕时代 | 0秒 | -80食物，剩余20 |
| 2 | 攒30食物建民居 | ~4秒 | 3农民净产2.4/秒 |
| 3 | 等人口增长 | **120秒** | growthInterval硬性等待 |
| 4 | +1人口，分配学者 | 即时 | 但只能分1个 |
| 5 | 攒15知识 | **300秒** | 1学者×0.05/秒=0.05/秒 |
| | **合计** | **~7分钟** | |

**瓶颈明确：知识。** 食物54秒就够，知识要5分钟。玩家在这5分钟里盯着几乎不动的知识条，体感极差。

### 1.2 为什么知识特别"卡"

- **隐性资源**：不像食物有"产出-消耗"的动态变化，知识是单向积累，没有反馈感
- **门槛前置**：必须先等120秒人口增长，再分配学者，再攒知识——三层等待嵌套
- **零基数开局**：0知识意味着从0开始线性爬升，没有任何"文明积累"的叙事感

## 二、设计目标

| 目标 | 指标 |
|------|------|
| 原始→农耕 | 保持现有节奏（~10-15分钟） |
| 农耕→聚落 | **40秒内**，食物和知识同步达标 |
| 聚落→青铜 | 学者第一次成为**明显刚需**（有学者≈37秒，无学者≈100秒） |
| 青铜以后 | 学者+学院是绝对主力，人口基础产出占比<40% |

**核心原则**：前期知识像空气，不用想，它就在那儿；后期知识像电力，得专门派人去搞。

## 三、改动方案

### 3.1 机制重构：从"学者专属"到"全民口耳相传"

**旧机制**：
```
知识产出 = 学者数量 × baseKnowledgePerPerson
只有学者产知识，其他人=0
```

**新机制**：
```
每人基础产出 = baseKnowledgePerPerson（口耳相传，全民都有）
学者额外加成 = scholarKnowledgeBonus（在base之上叠加）
总知识产出 = Σ(每人base) + Σ(学者bonus × 加成系数)
```

| 人口类型 | 知识产出/秒 | 说明 |
|---------|-----------|------|
| 农民/普通人口 | 0.05 | 口耳相传 |
| 学者 | 0.15 | 0.05 base + 0.10 bonus |

> 学者不是"唯一产知识的人"，而是"产得更快的人"（3倍普通人）。这样前期不需要学者，知识也会自然积累；后期分配学者则是效率飞跃。

### 3.2 时代知识成本调整

| 时代跃迁 | 旧成本 | 新成本 | 改动理由 |
|---------|--------|--------|---------|
| 原始→农耕 | food: 80 | **不变** | 节奏已合适 |
| 农耕→聚落 | food: 150, knowledge: 15 | **food: 120, knowledge: 5** | 食物降30（民居已花30），知识降10（自然积累就够） |
| 聚落→青铜 | food: 300, wood: 100, knowledge: 40 | **food: 300, wood: 100, knowledge: 50** | 知识+10，让学者第一次成为门槛 |

## 四、具体代码改动点

### 4.1 backend/config/GameConfig.js

```javascript
    baseMax: 3,
    growthIntervalSeconds: 120,
  },

```

### 4.2 backend/calculators/ResourceTickCalculator.js — calculateKnowledgePerSecond

```javascript
function calculateKnowledgePerSecond(population, effects) {
  const totalPop = population?.total || 0;
  const scholars = population?.scholars || 0;
  
  // 全民口耳相传基础
  const baseOutput = totalPop
    * GameConfig.resources.baseKnowledgePerPerson
    * (effects?.globalOutputMultiplier || 1);
  
  // 学者额外加成（在base之上叠加，不受globalMultiplier重复计算）
  const scholarBonus = scholars
    * GameConfig.resources.scholarKnowledgeBonus
    * (effects?.knowledgeOutputMultiplier || 1)
    * (effects?.globalOutputMultiplier || 1);
  
  return baseOutput + scholarBonus;
}
```

> 注意：学者bonus受 `knowledgeOutputMultiplier`（学院加成）和 `globalOutputMultiplier` 影响，但baseOutput不受 `knowledgeOutputMultiplier` 影响——避免学院对全民基础产出过度放大。

## 五、推演验证

### 5.1 农耕→聚落时代（目标：40秒内）

**场景**：玩家从农耕时代开局，手速正常（有教程引导）

| 时间点 | 状态 | 食物 | 知识 | 备注 |
|--------|------|------|------|------|
| T+0 | 刚进入农耕，3农民 | 20 | 0 | 原始时代知识积累已在进阶时清零？不，保留 |
| T+0 | 假设原始时代积累 | 20 | **~3.0** | 原始时代约10分钟，3人×0.05×600=90？等等，不对 |

**修正推演**：原始时代baseKnowledgePerPerson=0.05已经存在，只是之前只有学者调用。现在全民都有，所以：

| 阶段 | 人口 | 时间 | 知识积累 |
|------|------|------|---------|
| 原始时代 | 3人 | ~10分钟(600秒) | 3×0.05×600 = **90知识** |

但时代进阶不应该清零知识！进入农耕时代时玩家已经有90知识，远超5点需求。这意味着**原始→农耕本身就够知识**，但农耕→聚落的食物还需要攒。

**重新推演——农耕→聚落（从0知识开始的最坏情况）**：

| 步骤 | 动作 | 时间 | 食物 | 知识 |
|------|------|------|------|------|
| 开局 | 农耕时代，3农民 | 0 | 20 | 0（最坏情况） |
| 教程引导 | 点文明标签→进阶 | ~5秒 | 20 | 0.75 |
| 点建筑标签 | 引导锁建筑页 | ~3秒 | 20 | 0.90 |
| 建造农田 | -0食物（farm免费） | ~2秒 | 20 | 1.00 |
| 教程引导 | 等时代进阶按钮亮起 | — | 开始自然增长 | — |
| 食物净产 | 3农民×1.0 - 3人×0.2 = 2.4/秒 | 到120食物需42秒 | 120 | **3.1** |
| | **达标** | **约52秒** | 120≥120✅ | 3.1≥5❌ |

**问题**：知识还差1.9。需要再等38秒？

**修正：把知识门槛降到3？或者保持5但给初始知识奖励？**

或者：进入农耕时代时给少量知识奖励，比如 `进入新时代奖励：knowledge + 5`。这样立刻达标。

让我重新算一个更合理的方案：

**最终推演——带"时代知识奖励"机制**：

进入农耕时代时，系统奖励 **knowledge + 5**（文明跃迁的启蒙）。

| 步骤 | 动作 | 时间 | 食物 | 知识 |
|------|------|------|------|------|
| 开局 | 农耕时代，3农民 | 0 | 20 | **5（奖励）** |
| 攒食物 | 净产2.4/秒 | 到120需42秒 | 120 | **7.1** |
| | **达标** | **约42秒** | 120≥120✅ | 7.1≥5✅ |

**结论**：约 **42秒** 进入聚落时代。知识在前5秒就已达标（靠奖励），之后只是等食物。

如果嫌42秒还是长，可以把食物门槛再降到100（从120），这样只需要33秒。

但42秒对于一个"教学阶段"是可以接受的——玩家需要在这段时间里熟悉界面、点完tutorial步骤、看引导文字。

### 5.2 聚落→青铜时代（学者第一次成为刚需）

**场景**：聚落时代中期，10人口，伐木场Lv1，工坊Lv0，学院Lv0

| 配置 | 人口分配 | 知识产出/秒 | 攒50知识耗时 |
|------|---------|-----------|-----------|
| 无学者（全农民/工匠） | 10人，0学者 | 10×0.05 = **0.50** | **100秒** |
| 1学者 | 1学者，9其他 | 10×0.05 + 1×0.10 = **0.60** | **83秒** |
| 2学者 | 2学者，8其他 | 10×0.05 + 2×0.10 = **0.70** | **71秒** |
| 2学者+学院Lv1 | 2学者 | 0.50 + 2×0.10×1.2 = **0.74** | **68秒** |

**有学者 vs 无学者**：83秒 vs 100秒，差距17%。

但等等，这差距不够大。学者不够"刚需"。

**问题**：baseKnowledgePerPerson=0.05对10人口来说已经有0.5/秒，学者bonus只有0.1/秒，增量不够明显。

**方案调整**：提高学者bonus到0.15，让学者总计0.20/秒（4倍普通人）。

| 配置 | 产出/秒 | 攒50知识耗时 |
|------|--------|-----------|
| 无学者 | 0.50 | 100秒 |
| 1学者 | 0.50 + 0.15 = 0.65 | 77秒 |
| 2学者 | 0.50 + 0.30 = 0.80 | 63秒 |
| 2学者+学院Lv1 | 0.50 + 0.36 = 0.86 | 58秒 |

**有学者 vs 无学者**：58秒 vs 100秒，差距**42%**。学者第一次带来明显的效率飞跃。

但前期会不会受影响？3人口0学者：3×0.05=0.15/秒。农耕→聚落需要5知识：5÷0.15=**33秒**。加上时代奖励5点，知识立刻够。前期仍然不卡。

**确定采用**：scholarKnowledgeBonus: **0.15**

### 5.3 后期学者价值验证（青铜以后）

**场景**：20人口，学院Lv3（knowledgeOutputMultiplier=1.6），5学者

| 来源 | 计算 | 产出/秒 | 占比 |
|------|------|--------|------|
| 20人口口耳相传 | 20×0.05 | **1.00** | 40% |
| 5学者 bonus×学院 | 5×0.15×1.6 | **1.20** | 48% |
| 全局加成（假设10%） | (1.00+1.20)×0.1 | **0.22** | 9% |
| **合计** | | **2.42** | |

学者占比 **48%**，是绝对主力。没有学者的话：20×0.05=1.00/秒，攒100知识要100秒。有学者只要41秒。

**结论**：后期学者价值明确，人口基础产出虽然存在但只占40%，不会废掉学者体系。

### 6.3 AdvanceEraAction.js — 时代知识奖励

```javascript
// 在 execute() 中，进入新时代时给予知识奖励
function execute(gameState, tutorial) {
  // ... 原有资源扣除逻辑 ...
  
  // 时代知识奖励：每次文明跃迁给予启蒙奖励
  const eraKnowledgeBonus = { 1: 5, 2: 0, 3: 0 }; // 进入农耕+5，其他不加
  const bonus = eraKnowledgeBonus[config.nextEra] || 0;
  if (bonus > 0) {
    gameState.resources.knowledge = (gameState.resources.knowledge || 0) + bonus;
  }
  
  // ... 原有tutorial推进逻辑 ...
}
```

> 只有进入农耕时代时给+5知识奖励，确保玩家原始时代积累的知识即使为0也能立刻达标。聚落→青铜不给奖励，因为此时玩家已经有学者体系。

## 八、风险与兜底

| 风险 | 影响 | 兜底方案 |
|------|------|---------|
| 全民产知识导致后期膨胀 | 学者价值稀释 | 学者bonus=0.15（4倍），后期学者占比仍>40% |
| 原始时代知识积累过高 | 进入农耕时知识远超需求 | 知识是软上限资源，多出来不浪费，只是加速后续 |
| 离线收益知识过高 | 回来直接跳时代 | offline效率0.8封顶8小时，知识离线收益已计入 |
| 测试后42秒仍然感觉慢 | 食物门槛可能还是高 | 可再降农耕→聚落食物成本到100（33秒） |

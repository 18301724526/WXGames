# 战场接战表现任务 v0.1

日期：2026-05-29

## 目标

把战斗小人从“左右站桩 + 每回合程序前冲”的传统回合制表现，改成“开局双方接战 + 接战后原地按序列帧攻击”的战场表现。

本任务只调整前端视觉层，不改变后端战斗结算、回合顺序、伤害、兵力变化和战报结构。

## 表现规则

- 开局第一手的 `move` 阶段作为双方接战阶段，己方与敌方小兵都从各自布阵点移动到中部接战区。
- 接战后小兵保持在中部战场位置，不在每回合攻击时退回左右两边。
- 每个可见小兵拥有独立延迟、目标 lane 和轻微站位差，避免整队像一个整体平移。
- `attack/skill` 阶段只播放攻击 PNG 序列帧，不再额外叠加程序 x 方向前冲。
- `hit` 使用 `idle` 帧 + 抖动 + 闪白，抖动只作为受击反馈。
- `die/defeated` 只播放死亡序列帧，不做额外死亡位移；如果该手结算后兵力归零，即使分组为空也保留一个视觉单位播完死亡。
- 每次攻击在 `impact` 阶段根据当前战报 turn 飘出伤害文字；伤害值优先读取 `turn.damage`，没有显式字段时按受击方 `soldiersBefore - soldiersAfter` 推导。该表现只读战报，不参与结算。

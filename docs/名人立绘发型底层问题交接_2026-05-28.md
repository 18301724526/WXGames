更新时间：2026-05-28 12:43:24 +08:00

## 结论

当前的 `hairBase` 补底方向是错的。它解决了“透明洞”问题，但没有解决“可组合发型底层”问题。

图 1 `fp-layer-v2-art01-hairBase-bound-topknot-filled-01.png` 的问题不是单纯不好看，而是底层结构错误：额前被补成一整块厚重的横向发帘，中间还有明显的硬折角和暗块。它看起来像另一个前发/刘海层，而不是被刘海遮住时应该存在的自然发根底层。

历史旧稿里的 `fp-layer-v2-art01-frontHair-short-01.png` 曾作为正确参考：美人尖应该从头顶发流自然向额前收束，中心有清晰但不突兀的尖点，左右发束沿头型向两侧分流。它现在不再保留在当前素材目录里，只能从 Git 历史查看；后续所有刘海、前发或发型底层都要围绕这个锚点逻辑去做。

## 当前错误点

1. `hairBase` 的额前底层太厚，形成了第二层刘海。
2. 中心美人尖不是自然收束，而是像被裁出来的硬三角/硬折线。
3. 发流方向错误：当前补底主要是横向铺开，正常应从发顶向额前和两侧分流。
4. 黑色暗块过重，叠加刘海后会显脏，缩放或偏移时尤其明显。
5. 这张图不能靠坐标、缩放、裁剪调好，因为素材本身的底层语义不对。

## 正确目标

`hairBase` 应该是一张可独立存在、但不抢戏的发型底层：

- 需要补出刘海下面的发根和头发纹理，不能留透明洞。
- 不能把补底做成完整刘海或明显前发。
- 美人尖位置要和 `frontHair-short` 的中心尖点一致。
- 发流应从头顶和发冠区域自然向额头中心、两侧鬓角过渡。
- 刘海缩小、放大、左右偏移时，下面露出来的部分仍然像自然发根，而不是一块假发片。

- `frontend/assets/art/famous-person/layers/fp-layer-v2-art01-hairBase-bound-topknot-filled-01.png`

其中 `hairBase-bound-topknot-filled-01` 是当前错误补底稿，只能作为反例和临时占位。旧 `frontHair-short-01` 已从当前素材目录清理，必要时从 Git 历史取回作为美人尖和发流锚点参考。

## 后续重做建议

下一步不要继续在当前 `filled` 图上做简单拉伸或切割。建议重新出一张新的束发底层：

1. 保留束发、发冠、两侧长发、发带的整体风格。
2. 重做额前底层，只保留自然发根和头发纹理。
3. 用 `frontHair-short-01` 的美人尖作为中心锚点。
4. 输出后先做四种压力预览：底层单独显示、正常刘海、刘海缩小、刘海左右偏移。
5. 只有四种预览都不穿帮、不像假发片，才接入实验台和运行时代码。

## 接手检查方法

1. 打开名人立绘实验台：`/tools/famous-portrait-lab.html`
2. 只显示 `hairBase`，检查额前是否像自然发根，而不是横向厚发帘。
3. 打开 `bangs` 后，尝试缩放和左右偏移。
4. 如果刘海露出的底层仍然自然，就可以继续调参数。
5. 如果一移动就露出硬块、黑块或第二层刘海，就说明素材需要重做，不应该继续调坐标。

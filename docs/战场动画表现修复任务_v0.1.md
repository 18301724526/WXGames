# 战场动画表现修复任务 v0.1

日期：2026-05-29

## 目标

修复当前战场演出中“第一手空等、兵力提前扣除、结束后最后一击循环、移动跳变”的表现问题。

本任务只调整前端战斗播放层，不改后端战斗结算、伤害公式、速度规则和战报结构。

## 问题记录

- `turnIndex=0` 时没有行动事件，导致进入战斗后先空等一拍。
- 播放某一手时立即读取 `attackerSoldiersAfter / defenderSoldiersAfter`，导致移动和攻击前兵力已经变化。
- 最后一手结束后仍保留最后一个 `activeTurn`，画面会继续循环最后一次攻击/受击。
- 移动阶段使用整手进度，切到攻击阶段时位置会跳回攻击偏移。
- 当前只有第一个小兵动作，队伍感偏弱。

## 修复方案

- 将 `turnIndex` 解释为“当前正在播放的战报事件下标”，`0` 即第一手。
- 每手拆成四个播放阶段：
  - `prepare`：双方待机，展示行动开始。
  - `move`：行动方前排小兵跑向接战区。
  - `impact`：行动方攻击或释放技能，目标方受击，显示伤害战报。
  - `settle`：双方回到待机，兵力与小兵数量切换到本手结算后。
- 结束或跳过时 `activeTurn=null`，展示最终兵力，按钮切为完成。
- 移动阶段使用阶段内归一化进度，避免跑动到一半突然回弹。
- 前排多个小兵错帧参与移动、攻击、受击。

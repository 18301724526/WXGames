## 四、后端运行时与安全改动

### 1. 版本接口与更新体验

新增 `backend/routes/versionRoutes.js`，`/api/version` 支持：

- `ETag`
- `If-None-Match`
- 304 响应
- 部署 manifest metadata

`VersionService` 忽略本地 SQLite/log/runtime artifacts，避免本地 playtest 或运行态文件误触发版本变化。

前端 `GameAPI.getVersion()` 复用 ETag 缓存，`UpdateChecker` 添加失败退避和恢复逻辑。

### 2. API 请求稳定性

`frontend/js/api/GameAPI.js` 增加：

- 默认 10s timeout。
- request id。
- 结构化错误。
- GET 短重试。

`GameStateSync` 和 `UpdateChecker` 在失败后使用指数退避窗口，避免在线异常时持续打满接口。

### 3. 安全配置与管理员授权

新增 `backend/config/SecurityConfig.js`：

- production 要求 `JWT_SECRET`。
- production 要求 `CORS_ORIGINS`。
- development 保持本地可运行。

新增 `backend/middleware/adminMiddleware.js`：

- 登录认证与管理员授权分离。
- development 默认允许 `codexqa`。
- production 必须显式配置 `ADMIN_USERS`。

管理员配置发布和指标接口均走管理员权限边界。

## 前置条件

- 使用服务器正式页面和有账号数据的账号测试。
- 账号最好已经进入古典时代，能打开世界/军事地图并派兵出征。
- 至少拥有或可寻访到几名名人：军事型、内政/魅力型、斥候型各一名最好。

## 用例 1：有主地点进入自动战斗

步骤：

1. 登录服务器账号。
2. 进入世界/军事地图。
3. 选择一个有主地点。
4. 选择一名军事型名人领队，配置兵力后出征。
5. 等行军完成，进入或查看战斗结算。

预期：

- 有主地点不再只按“兵力大于防御”硬判断直接结算。
- 战斗会生成结构化战报和战斗场景播放数据。
- 战斗结束后地点、己方兵力、战报结果同步更新。

## 用例 2：速度决定先后手

步骤：

1. 派速度较高的名人打一场有主地点战斗。
2. 再派速度较低的名人打一场相近战斗。
3. 对比战斗首轮行动顺序。

预期：

- 速度高的一方优先行动。
- 同速时当前规则默认己方先行动。
- 速度只决定每轮先后手，不出现追击、额外行动或行动条。

## 用例 3：技能释放和冷却

步骤：

1. 使用有主动战法的军事型名人进入战斗。
2. 观察技能名和立绘 cut-in。
3. 继续观察后续几次自身行动。

预期：

- 技能只在自身回合、冷却为 `0` 且释放条件满足时释放。
- 冷却未结束时执行普攻。
- 冷却结束但释放条件不满足时也执行普攻，下次自身行动重新判断。
- 技能 cut-in 只出现在技能行动，普攻不显示技能名和名人立绘。

## 用例 4：技能 cut-in 表现

步骤：

1. 使用会释放主动战法的名人进入战斗场景。
2. 观察释放技能瞬间。

预期：

- 战斗普通节奏保持较快。
- 技能释放前会暂停战斗时间播放表现。
- 名人立绘从左侧进入，技能名从右侧进入，中间短暂停留后飞出。
- 玩家能看清立绘和技能名。

## 用例 5：内政/魅力型名人上战场

步骤：

1. 使用内政型或魅力型名人作为领队。
2. 派其进入有主地点战斗。
3. 观察战斗行动。

预期：

- 内政/魅力型名人不释放战斗技能。
- 系统不会给它们注入 fallback 战斗技能。
- 战斗中只执行普攻循环。
- 名人自身仍可在名人数据中保留两个内政技能，当前阶段只展示不生效。

## 用例 6：斥候型名人

步骤：

1. 使用斥候型名人作为领队。
2. 派其进入有主地点战斗。
3. 观察首轮和后续行动。

预期：

- 斥候拥有轻战斗主动战法。
- 若技能带 `firstOwnAction` 条件，只在第一次自身行动时释放。
- 斥候的侦查/行军特质当前只作为存档和展示能力，不影响侦查实际收益。

## 用例 7：寻访生成技能结构

步骤：

1. 在名人系统中连续寻访几次。
2. 分别查看军事型、内政/魅力型、斥候型候选。

预期：

- 军事型候选显示主动战法和战斗被动特质。
- 内政/魅力型候选显示两个内政技能，不显示战斗技能。
- 斥候候选显示一个轻战斗主动战法和一个斥候特质。
- 同一候选被接纳后，技能、立绘和来源信息保持不变。

## 用例 8：名人技能面板展示

步骤：

1. 打开名人面板。
2. 查看已加入名人与候选名人的卡片。
3. 分别观察军事型、内政/魅力型、斥候型名人的技能徽章。
4. 将鼠标移到技能徽章上，或在触屏设备点击技能徽章。
5. 查看右侧六维蛛网图。
6. 在触屏设备或浏览器点击同一个技能徽章两次。
7. 点击技能徽章打开详情后，再点击名人面板内部空白区域。
8. 已加入名人超过 3 位时，点击列表底部上一页/下一页。

预期：

- 军事型名人卡片只常驻显示 `主动战法：技能名` 和 `战斗被动：技能名`。
- 主动战法详情使用玩家能理解的完整效果句，例如 `效果：发动战法攻击目标，造成一次兵刃伤害，并施加倒戈（倒戈：将敌方本次损失兵力的一部分转换为自己的兵力）。`
- 主动战法参数展示为 `冷却：3 回合 · 发动率：100%`；后续若技能模板有真实概率，则展示真实概率。
- 详情中不再出现 `说明`、`规则`、`出手机会`、`直接伤害`、`属性修正`、`吸血`、`冷却3次自身行动`、`冷却就绪`、`目标存活`、`cooldownReady`、`targetAlive` 这类工程口径。
- 内政/魅力型名人卡片只常驻显示 `内政主技：技能名` 和 `内政副技：技能名`。
- 当前未接入实际生效入口的内政或斥候特质在详情中显示 `暂未接入实际收益`。
- 六维蛛网图固定展示统帅、武力、智力、政治、魅力、速度，速度必须出现。
- 名人卡片和蛛网图不出现 `工巧`。
- 卡片文字不挤出边界，不遮挡接纳/放弃按钮。
- 第一次点击技能徽章固定显示技能详情，再次点击同一技能徽章收起详情。
- 固定详情后点击名人面板内部空白区域只收起技能详情，名人面板仍保持打开。
- 已加入名人列表可以翻页查看更早加入的名人，翻页时已固定的技能详情会被清理。

## 用例 9：旧名人技能兼容

步骤：

1. 使用已有老账号或老存档登录。
2. 打开名人面板，查看旧版本生成过的名人技能。
3. 选择旧名人进入有主地点战斗。

预期：

- 面板不再出现 `combo`、`ambush`、`morale`、`counter` 这类旧英文效果。
- 旧连击口径表现为 `追击（根据本次攻击的一部分伤害再次打击目标）`。
- 旧伏击口径表现为 `先机打击`。
- 旧士气增强口径表现为对应属性提高，例如 `统帅提高`。
- 旧反击口径当前不会显示为已生效技能效果。
- 战斗仍能正常播放和结算。

## 用例 10：旧名人完整技能机制升级

步骤：

1. 使用已有老账号或老存档登录。
2. 打开名人面板，分别查看旧版本已加入名人和仍在候选列表中的名人。
3. 对比军事型、内政/魅力型、斥候型的技能行。
4. 选择旧军事型或旧斥候型名人进入有主地点战斗。

预期：

- 旧军事型名人显示 `主动战法` 和 `战斗被动`，原主动战法尽量保留并迁移效果。
- 旧内政/魅力型名人显示 `内政主技` 和 `内政副技`，不显示战斗主动技能，上战场只普攻。
- 旧斥候型名人显示 `主动战法` 和 `斥候特质`。
- 旧候选名人与旧已加入名人都能自动升级，不需要重新寻访或手动修档。
- 技能详情仍有释放间隔、触发或 `暂未接入实际收益` 等元信息，卡片显示不空白。

## 用例 11：战斗状态系统第一批

步骤：

1. 使用带 `shield` 效果的技能进入战斗。
2. 观察释放技能后和后续受击时的战场飘字与战报行。
3. 使用测试技能或后续带状态技能观察 `burn / poison / armorBreak`。

预期：

- `shield` 不再只是文字，释放后会生成守御状态，后续受击先显示 `守御抵消 N`，实际损兵减少。
- `burn / poison` 在目标自己的行动前结算，显示 `灼烧 -N` 或 `中毒 -N`，目标可能因此败退并跳过行动。
- `armorBreak` 持续期间提高目标后续受到的兵刃伤害，不提高谋略伤害。
- 战报 `turns` 里记录 `statusEvents / floatingTexts / statusesBefore / statusesAfter`，方便回放和排查。
- 普通攻击节奏和技能 cut-in 节奏不被整体拖慢。

## 5. 新玩法扩展准入清单（后端兼容性宪法，owner 意图 2026-07-15）

owner 定调：游戏早期、玩法必然扩（外交行为、三战式资源地及其守军配置、名人系统可能从随机改固定等），后端必须严格、通用、不与前端强耦合。落成硬规则——**任何新玩法系统进后端，六项全过才准入**：

1. **领域数据 = 新窄行 aggregate**（M3 模板：expand→backfill→shadow→原子切换唯一 writer→contract），带版本头与 DB 约束；新系统**直接生窄行**，禁止再塞进 game_states 大 JSON。
2. **数值与内容 = Excel 配置表管线**（table-schemas 契约 + freshness 门禁）；设计尚未定型处（如名人固定 vs 随机）做成 **provider 缝**：来源选择本身是一行配置，切换不改服务代码。
3. **玩家操作 = 命令管线 definition**（幂等 receipt/owner 锁/提交器免费获得），登记 COMMAND_OWNER_RULES 并进 M0 writer inventory。
4. **状态变化 = 领域事件**（M4 stream）；DTO 只投影领域事实，显示文案/美术路径/排版不进存档不进权威表（与 asset-path 工单同族原则）。
5. **定时行为 = scheduled_jobs 闹钟**（due_at 驱动），禁止新增轮询扫描类 worker 逻辑。
6. **资金/权限类操作**标注 regulated command class（V24-04 清单），commit 点复核。

已点名玩法的落位（先记账，不立项）：

| 未来玩法 | 落位 | 现成度 |
|---|---|---|
| 外交系统（行为层） | 数据层+tick 已在 main 跑；补行为消费者（aiFactionCore 接线）+ 外交命令 + diplomacy 事件；bordering 谓词接真实领土相邻 | 数据/调参表全齐，只缺行为接线与 UI |
| 三战式资源地 | world 域新 aggregate（resource_nodes），产出走 scheduled_jobs，争夺复用 encounter/battle 链 | **owner 已拍板（2026-07-15）：设计照三战原样即可（玩法机制不受保护，可直接采用）**；占领/守军/战斗链全部现成可复用 |
| 资源地守军配置 | garrison 配置表族新维度（策划直接配 Excel） | 管线现成，GarrisonPolicy 单源可扩 |
| 名人固定/随机 | FamousPersonProvider 缝：random（现生成器参数表化）/ fixed（famous_persons 内容表）/ hybrid（固定名将+随机白身） | 不必现在决定，只需在触碰该服务时把缝切出来 |

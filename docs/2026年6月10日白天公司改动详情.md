# 2026年6月10日白天公司改动详情

本文用于下班交接，记录 2026 年 6 月 10 日白天在公司完成的大地图、世界行军、HUD 分层与回归验证工作。

## 一、今天解决的核心问题

今天主要处理了世界大地图行军链路上的两个实测问题：

1. 回城和停止按钮疑似无效。
2. 点击未知区域时，屏幕坐标映射到目标 tile 的偏差很大。

同时按当前架构要求复查并修正了职责边界：

- 物理 `worldMap` 层只负责地图绘制、地图上下文发布、地图/地块/站点/角色命中目标采集。
- `mainHud` 层负责所有可交互 HUD，包括行军目标面板、队伍回城/停止按钮、编队选择面板、教程高亮和其它玩家输入。
- 大地图层不能直接画 HUD，也不能注册编队选择这类 HUD 控件。

## 二、前端改动

### 1. 大地图未知区域坐标映射

涉及文件：

- `frontend/js/platform/WorldMapRuntime.js`
- `frontend/js/platform/CanvasGameShellInputRouter.js`
- `frontend/js/platform/WorldMapRuntime.test.js`

改动内容：

- 新增 `WorldMapRuntime.getLayerPointFromHudPoint(point)`，作为 HUD 坐标和物理世界地图层坐标之间的唯一转换边界。
- 点击来自 `mainHud`，推导目标 tile 前会：
  - 加回 `worldMap` 层 padding：`viewportOffsetX/Y`
  - 减掉临时拖拽层偏移：`dragLayerOffset`
- 背景 `selectWorldMarchTarget` 命中不再直接信任旧命中目标上的 `targetQ/targetR`，而是用实际点击点重新推导 tile。

影响：

- 修复未知区域点击目标偏差大的问题。
- 拖拽/快照偏移期间点击也能落到正确 tile。
- 行军目标选择仍由前端只提交意图，最终路线仍由后端权威计算。

### 2. HUD 按钮命中优先级

涉及文件：

- `frontend/js/platform/renderers/CanvasSurfaceHitTargets.js`
- `frontend/js/platform/renderers/CanvasSurfaceRenderer.test.js`

改动内容：

- 新增全局高优先级命中序列：
  - `returnWorldMarch`
  - `stopWorldMarch`
  - `openWorldMarchFormationPicker`
  - `startWorldMarch`
  - `closeWorldMarchHud`
  - `selectWorldActor`
- 修复 idle 队伍的“回城”按钮与地图角色头像命中框重叠时，点击被 `selectWorldActor` 抢走的问题。

影响：

- 玩家看到的 HUD 按钮现在优先于地图角色头像。
- 角色头像仍然优先于城池/地块命中，保留原有地图交互体验。

### 3. HUD 与 worldMap 物理层职责拆分

涉及文件：

- `frontend/js/platform/renderers/WorldMapTileMapRenderer.js`
- `frontend/js/platform/renderers/WorldMapLayerCanvasRenderer.js`
- `frontend/js/platform/renderers/HudOverlayCanvasRenderer.js`
- `frontend/js/platform/renderers/CanvasFrameRenderer.js`
- `frontend/js/platform/renderers/WorldMarchHudCanvasRenderer.js`
- 对应测试文件

改动内容：

- `WorldMapTileMapRenderer` 不再在地图层绘制行军 HUD，只发布 `lastWorldTileMapContext`、actors、viewport、geometry、frame。
- `WorldMapLayerCanvasRenderer.collectMapHomeWorldSiteHitTargets()` 只采集地图/站点/角色命中，并发布 `lastMapHomeWorldHudContext` 给 HUD pass。
- `HudOverlayCanvasRenderer` 和 `CanvasFrameRenderer` 在 `mainHud` pass 中调用 `renderWorldMarchHud()`。
- 当 runtime context 缺少 actors 时，HUD pass 会从 `state.worldExplorerState` 通过 `WorldMarchSystem.buildActors()` 派生 actors，避免只有选中 id、没有 actor 数据导致按钮不出现。

影响：

- 大地图物理层不再画 HUD，符合成熟引擎分层职责。
- HUD 命中、绘制、教程高亮都统一在主输入层。
- 后续扩展大地图渲染、雾层、WebGL 层时，不会把输入控件混进地图层。

### 4. 队伍 HUD 状态

涉及文件：

- `frontend/js/platform/renderers/WorldMarchHudCanvasRenderer.js`
- `frontend/js/domain/WorldMarchProgressSnapshot.js`
- 对应测试文件

改动内容：

- active 行军队伍显示“回城 + 停止”。
- idle 停驻队伍只显示“回城”，不显示“停止”。
- 前端能把没有 route 的 idle mission 渲染成停驻 actor。

影响：

- UI 状态与后端状态一致。
- 停止后的队伍不会从地图上消失。
- idle 队伍可以被再次选中并回城。

## 三、后端改动

### 1. idle mission 保留与客户端可见性

涉及文件：

- `backend/services/worldExplorer/WorldExplorerMissionNormalizer.js`
- `backend/tests/WorldExplorerService.test.js`
- `backend/tests/GameRoutesTutorial.test.js`

改动内容：

- normalizer 原先会丢弃空 route mission。现在允许 `status === 'idle'` 且 route 为空的 mission 保留。
- idle mission 的 `nextStepAt` 规范化为 `null`。
- route 为空时，target 回退到 `position`。

影响：

- 停止在当前 tile 的世界行军不会从 `gameState.worldExplorerState` 和客户端状态中消失。
- stop 返回后的 DB 状态、客户端状态、HUD actor 状态一致。

### 2. idle 队伍允许回城

涉及文件：

- `backend/services/worldExplorer/WorldExplorerActions.js`
- `backend/tests/WorldExplorerService.test.js`

改动内容：

- `returnWorldMarch` 现在支持 active mission 和 idle mission。
- idle mission 回城路线从 `mission.position` 开始，而不是错误地从 origin 或最后 revealed route 推断。
- `stopWorldMarch` 仍保持 active-only 语义。

影响：

- 停止后的队伍可以点击“回城”。
- 后端仍然保持服务端权威路线计算。

## 四、动作分发改动

涉及文件：

- `frontend/js/platform/CanvasActionDispatchRegistry.js`
- `frontend/js/platform/CanvasActionDispatcher.test.js`

改动内容：

- `returnWorldMarch` 和 `stopWorldMarch` 只向 legacy handler 传 `missionId/actorId`。
- `claimExplore` 只传 mission id。

影响：

- 前端不会把多余坐标或整包 action 传入世界行军终结类命令。
- 与服务端“intent-only”权威模型一致。

## 七、影响面与风险

正向影响：

- 世界行军停止、回城、未知区域点选这条核心链路已经从后端状态、前端渲染、输入命中、真实浏览器截图四个层面闭环。
- 大地图层与 HUD 层边界更清晰，后续做 WebGL fog、地图缓存、AI AOI 或多人同步时更安全。
- idle mission 不再被 normalizer 丢弃，减少“服务端成功但客户端消失”的状态不一致问题。

潜在注意点：

- `CanvasSurfaceHitTargets` 的优先级序列现在明确偏向 HUD 指令。后续新增 HUD 命令时，如果会和地图角色/站点重叠，需要加入同一优先级策略并补测试。
- `WorldMapRuntime.getLayerPointFromHudPoint()` 是输入坐标边界，后续若改物理 layer padding、drag compositor 或 viewport origin，必须同步此函数测试。
- idle mission 现在是合法客户端可见状态，后续任何清理任务不能再用“route 为空”作为删除 mission 的唯一条件。

## 八、后续交接建议

1. 下一位开发如果继续做大地图，只读 `WorldMapTileMapRenderer` 和 `WorldMapLayerCanvasRenderer` 时要记住：它们发布 HUD context，但不画 HUD。
2. 新增玩家可点击控件，优先放在 `mainHud` pass，并给 `CanvasSurfaceHitTargets` 增加命中优先级测试。
3. 新增世界行军状态，必须同时补：
   - 后端 normalizer 测试
   - `WorldMarchProgressSnapshot` actor 派生测试
   - HUD 可见/命中测试
   - 真实浏览器截图验证
4. 继续上线前建议再跑一次线上真实流程，特别是手机视口下按钮是否与角色/城池命中重叠。

### 名人来源

第一版名人生成暂时只开放寻访。

已关闭：

- 事件投奔生成新候选。
- 战后归附生成新候选。
- 世界探索生成新候选。

实现方式：

- `backend/services/FamousPersonService.js` 新增 `ENABLED_SOURCE_TYPES = ['seek']`。
- 请求 `postWar/event` 等来源生成候选时，会回退为 `seek`。
- `backend/services/TerritoryService.js` 新增 `POST_WAR_FAMOUS_PERSON_ENABLED = false`，战后不再塞候选。
- 旧存档中已有 `event/postWar` 来源仍可展示，不做删除。

### 技能设计

已写入 `docs/战斗系统设计_v0.2.md`：

- 技能系统采用“模块化肉鸽生成 + 确定性战斗结算”。
- 普通武将不维护固定技能表。
- 普通技能由效果原子、组合规则、命名规则和强度预算生成。
- 生成后写入武将存档，战斗中确定结算。
- 后期独特技能走 `uniqueSkillId`，不进普通随机池。

名人品质：

| 品质 | 概率 |
| --- | ---: |
| `common` | `32.4%` |
| `good` | `26%` |
| `great` | `36%` |
| `legendary` | `5.6%` |

寻访保底：

- 无冷却。
- 无资源消耗。
- 无抽卡道具。
- 连续 `29` 次寻访未出 `legendary`，第 `30` 次强制 `legendary`。
- 出 `legendary` 后计数清零。
- 只统计寻访，不跨来源。

状态系统已定：

- 状态在目标自己的行动前结算。
- 清状态、净化、免疫优先生效。
- 同类状态允许叠加，但有上限。
- 持续时间按目标自己的行动前扣减。
- 持续伤害每次触发动态计算，不锁定伤害。
- 谋略伤害为智力攻击目标智力。
- 单位死亡后，自己身上的状态和自己施加出去的状态全部清空。

## 下一步建议

3. 需要确认：
   - 每个品质的初始属性区间。
   - `common/good/great/legendary` 是否都必带主动战法和被动特质。
   - 内政、魅力、斥候定位在第一版是否也生成战斗技能。
   - 旧名人已有技能如何迁移到新技能原子。
   - 寻访保底计数字段放在 `famousPersonState.seek` 还是单独字段。

## 注意事项

- 当前代码里的战斗技能仍是早期兼容底座，不要直接按旧 `lifesteal/combo/counter/ambush/morale` 扩。
- `combo` 后续应迁移为确定性的 `secondHit`。
- `counter` 第一版暂不进入普通池。
- `ambush` 后续应迁移为确定性的 `firstStrike`。
- `morale` 在士气系统未启用前，不要进入普通池。
- 战斗结算不要引入战斗中随机触发。随机只发生在生成阶段。
- 文档讨论阶段可以不改代码；一旦改代码，继续保持版本号、测试和双远端推送。

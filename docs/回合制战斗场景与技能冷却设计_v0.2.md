# 回合制战斗场景与技能冷却设计 v0.2

日期：2026-05-29

## 目标

本版战斗场景不再使用程序绘制的抽象小人作为最终表现方向，而改为资源驱动的战场演出：

- 战场是美术背景图。
- 士兵是透明 sprite sheet 序列帧资源。
- 小兵在场上跑位、接战、攻击、受击。
- 玩家只负责战前搭配，进入战场后自动播放黑盒战斗。
- 战斗内核仍是回合制，由速度和技能冷却决定行动。

## 表现规则

`100兵=1个小兵`只用于画面表现和性能控制，不是战斗单位拆分。

示例：

```text
500 兵 = 5 个小兵表现
501 兵 = 6 个小兵表现
```

战斗结算仍按真实兵力计算，501 兵不会被拆成 6 个独立战斗单位。

## 资源规范

当前落地资源：

| 类型 | 路径 | 用途 |
| --- | --- | --- |
| 战场背景 | `frontend/assets/art/battle/battlefield-forest-camp.png` | 战斗场景底图 |
| 己方士兵帧表 | `frontend/assets/art/battle/soldier-player-sheet.png` | 己方小兵序列帧 |
| 敌方士兵帧表 | `frontend/assets/art/battle/soldier-enemy-sheet.png` | 敌方小兵序列帧 |

士兵帧表规格：

```text
4 行 x 4 列
第 1 行：idle 待机
第 2 行：move 跑动
第 3 行：attack / skill 攻击或技能
第 4 行：hit / defeated 受击或退场
```

后续增加兵种时沿用同一结构，避免渲染器按单个资源写死。

## 战斗内核

### 行动顺序

每轮按速度排序：

```text
速度高的一方先行动
速度低的一方后行动
```

若速度相同，当前版本己方先行动。

### 技能释放

每个单位有一个主动战斗技能。

规则：

```text
开局双方第一次行动必定释放技能。
技能释放后进入冷却。
之后轮到自己行动时：
  如果技能冷却结束，释放技能；
  否则执行普攻。
冷却结束不会插队，必须等到自己的行动回合才释放。
```

当前冷却以“自身行动次数”为单位递减，而不是以真实时间递减。这样可以保持纯回合制语义，前端动画只是战报事件的演出。

### 战报事件

后端输出 `speed-skill-cooldown-v1` 战报。

关键字段：

```js
{
  system: "speed-skill-cooldown-v1",
  skillRules: {
    openingSkill: true,
    cooldownTicksOnOwnTurnOnly: true,
    fallbackAction: "basicAttack"
  },
  turns: [
    {
      actor: "attacker",
      target: "defender",
      action: "skill" | "basicAttack",
      skillName: "...",
      cooldownBefore: 0,
      cooldownAfter: 3,
      damage: 42,
      attackerSoldiersAfter: 474,
      defenderSoldiersAfter: 457
    }
  ]
}
```

前端只消费这些事件播放动画，不参与战斗计算。

## 前端演出

每个行动事件拆成两个视觉阶段：

```text
move：攻击方小兵向接战区跑动
impact：攻击方播放攻击/技能帧，目标方播放受击帧
```

技能事件会额外显示一次技能冲击特效和技能名。普攻只显示普通接战特效。

## 回退边界

本版保留了进入战斗场景、跳过、关闭、战报回放等通用框架。若需要回退表现层，可只替换 Canvas renderer 的战斗绘制实现；后端 `speed-skill-cooldown-v1` 战报结构应继续保留，因为它描述的是正确的战斗规则。

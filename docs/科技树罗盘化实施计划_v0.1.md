# 科技树罗盘化实施计划 v0.1

更新时间：2026-05-24 17:44:33 +08:00<br>
计划版本：`0.1.110`<br>
目标：把科技树从“卡片节点 + 顶部详情”改成接近军事罗盘的“大画布点位 + 点击弹窗详情”，让树本身更清爽，路线关系更容易看。

## 1. 核心结论

- 科技树节点不再使用大卡片底壳。
- 科技节点只显示主题图标、短标题和极轻量状态，不在节点内部显示效果说明。
- 点击科技节点打开详情弹窗，参考军事罗盘地点弹窗承载说明、效果、前置条件和研究按钮。
- 拖动逻辑、时代右侧常驻栏、树状连线和现有科技前置规则继续保留。
- 不新增双端分支逻辑，仍维护同一套 Canvas 代码。

## 2. 美术资源方案

每个科技主题单独一张图，不做图集。生成源图使用纯绿色背景，扣成透明后放入 `frontend/assets/art/`。

## 3. 分步任务

### Step 1：生成并导入科技主题图标

任务：
- 逐张生成 10 个科技主题图标，背景为纯 `#00ff00`。
- 使用本地扣图工具输出透明 PNG。
- 加入 Canvas 预加载资源列表。

测试方法：
- 检查所有 `tech-*-cutout.png` 都存在且有 alpha 通道。
- 运行资源相关测试，确认资源路径被纳入渲染器。

预期结果：
- 科技树后续可以直接按路线拿到对应主题图标。
- 没有绿色底、没有图集切偏问题。

### Step 2：科技节点改为罗盘点位视觉

任务：
- 将 `renderTechNode` 从矩形卡片改成圆形/徽章点位。
- 节点显示：主题图标、短标题、状态外圈。
- 研究状态用亮度、外圈和小标记表达。
- 去除节点内部效果、状态长文和卡片底壳。

测试方法：
- 渲染科技页，确认节点 hit target 仍然可点击。
- 运行 Canvas 渲染器单测，确认科技节点仍注册 `selectTechNode`。

预期结果：
- 科技树视觉接近地图点位，不再像卡片列表。
- 连线和路线色仍清晰可见。

### Step 3：科技详情改为点击弹窗

任务：
- 取消顶部大详情面板，只保留轻量标题和科技点信息。
- 点击节点后打开详情弹窗。
- 弹窗包含科技名、时代、方向、描述、研究后效果、前置科技、科技点、研究按钮。
- 点击背景或关闭按钮可退出弹窗。

测试方法：
- 点击科技节点后弹窗出现。
- 未满足条件时研究按钮不可用，并显示原因。
- 可研究科技点击研究按钮后仍走现有 `research` 动作。

预期结果：
- 科技树画布空间更大。
- 详情信息不再和树节点挤在一起。

### Step 4：布局与交互回归

任务：
- 保留科技树拖动范围、右侧时代常驻栏、横纵滚动条。
- 检查弹窗不遮挡底部主导航，不影响拖动关闭后的树操作。
- 检查小屏宽度下标题不重叠、不溢出。

测试方法：
- 运行科技树相关前端单测。
- 运行全量测试。
- 如本地服务可用，用浏览器手动检查科技页拖动、点击、关闭、研究。

预期结果：
- 拖动逻辑和前置逻辑不回退。
- UI 不再出现大卡片底壳和文字堆叠。

### Step 5：版本、提交与推送

任务：
- 版本号升级到 `0.1.110`。
- 更新缓存串到 `tech-tree-route-lanes-v1`。
- 在本文档追加执行记录。
- 提交并推送 GitHub 与服务器远端。

测试方法：
- `node --check` 检查改动文件语法。
- 运行目标测试和全量测试。
- 推送后检查服务器健康接口版本。

预期结果：
- 线上版本显示 `0.1.110`。
- 用户可直接测试科技树新交互。

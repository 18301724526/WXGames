# 战斗系统解耦任务计划 v0.1

日期：2026-05-29

## 目标

把战斗规则从领土扩张流程中拆出来，让后续武将属性、技能、速度和战斗表现可以独立迭代。

本计划只处理结构解耦，不重新设计战斗数值。

## 当前问题

- `backend/services/TerritoryService.js` 同时负责侦察、出征、占领、战斗模拟、战报生成、战后归附，职责过重。
- `speed`、攻击力、技能冷却、守军模板和战场资源选择都写在领土服务里，后续会阻碍属性和技能系统重做。
- 前端已经有战斗场景播放链路，短期不需要改播放结构。

## 目标结构

```text
backend/services/TerritoryService.js
  只负责领土、任务、出征、占领和战后结算。

backend/services/BattleService.js
  负责自动战斗模拟、双方战斗快照、战报结构和视觉资源字段。

backend/config/BattleConfig.js
  负责战斗常量、守军模板、战场资源、fallback 领队和 fallback 技能配置。

backend/services/FamousPersonService.js
  保留名人生成、候选、接纳、立绘和早期技能数据。
```

## 本轮执行记录

- 2026-05-29：完成第一批结构解耦。
  - 新增 `backend/services/BattleService.js`。
  - `TerritoryService` 不再保存战斗公式，只在出征时获取名人快照、在结算时调用 `BattleService.simulateConquestBattle()`。
  - 新增 `backend/tests/battle-service.test.js`，覆盖战报系统名、开局技能、501 兵显示 6 组和历史 fallback。
  - 本轮没有重做武将属性、`speed` 或技能数值。
- 2026-05-29：完成第二批配置解耦。
  - 新增 `backend/config/BattleConfig.js`。
  - 守军模板、战场地图、战斗表现资源、fallback 领队和 fallback 技能迁出 `BattleService`。
  - `BattleService` 保留战斗流程和公式实现，后续重做 `speed`/技能时优先改配置和规则层。
  - 新增配置副本测试，避免调用方修改全局模板。
- 2026-05-29：开始战斗系统 v0.2 设计。
  - 新增 `docs/战斗系统设计_v0.2.md`。
  - 已确认武将六维属性：统帅、武力、智力、魅力、政治、速度。
  - 已确认初始属性 `1-100`，面板属性不设硬上限。
  - 已确认每 `10` 级固定获得 `10` 点自由属性点，品质只影响自动成长数值。
  - 已确认四档品质：一般、良才、英杰、传奇；自动成长总点数暂定为每 `10` 级 `4/6/8/10`。
  - 已确认经验来自战斗表现，胜败都有经验，主要按敌军损失计算，己方战损提供少量苦战经验。
  - 已确认速度暂时只决定每轮先后手，不影响行动频率。
  - 已确认无上限等级的平滑经验曲线、按武将定位分配自动成长点数、兵刃/谋略伤害公式方向和结构化详细战报表现。
  - 已确认士气第一版只保留字段不影响公式；技能 v1 采用 `1` 主动战法 + `1` 被动特质；战斗流程和战后结算进入可实现状态。
- 2026-05-29：`0.1.151` 落地第一批正式战斗内核。
  - `BattleConfig` 战斗系统号升级为 `attribute-auto-battle-v2`。
  - 战报增加 `ruleVersion`、`actionOrder`、`actionDecision`、自身行动计数和冷却推进字段。
  - 技能释放按 `castConditions` 决定，不再开局无条件必放；冷却只在自身行动推进。
  - 谋略伤害改为攻击方智力对目标智力，兵刃伤害仍为武力对统帅。
  - `preBattle` 被动特质中的属性修正会在准备阶段生效并写入战报。
- 2026-05-29：`0.1.152` 落地技能生成器接口收口。
  - `abilityKit` 写入 `source`、`seed`、`generatorInput`、`availableEffectPool`、`budgetChecks` 和 `budgetStatus`。
  - 技能模板按定位效果池过滤，内政型只生成内政技能，斥候只使用轻战斗和侦查/行军池。
  - 同一 seed 可以稳定复现同一组技能模板，便于后续排查和回放。
  - 技能能力补充 `description`，为下一步名人技能面板详情展示做准备。
- 2026-05-29：`0.1.153` 落地名人技能面板展示。
  - `UIStatePresenter` 新增 `skillDetails`，按槽位输出主动战法、战斗被动、内政主技、内政副技和斥候特质。
  - Canvas 名人卡片从只显示一条技能摘要改为显示两条紧凑技能详情。
  - 内政/斥候存档展示型能力会明确标注 `当前仅展示`。
  - 本批只改展示，不改战斗数值和战斗结算。
- 2026-05-30：`0.1.156` 落地名人卡片展示 v2。
  - 名人卡片改为名称、称号、两条技能徽章和右侧六维蛛网图，详情通过悬停或点击技能徽章显示。
  - 六维口径固定为统帅、武力、智力、政治、魅力、速度；速度是正式六维之一。
  - `工巧/craft` 不属于武将或名人属性，只保留人口/工匠等历史系统语境，不再参与名人生成和展示。
  - 卡片不常驻展示归附、招募等过程性文本，避免挤压技能和属性展示。
- 2026-05-30：`0.1.157` 修复名人技能详情关闭交互。
  - 点击同一个技能徽章会切换收起已固定的技能详情。
  - 固定详情后，点击名人面板内部空白区域只收起详情，不关闭名人面板。
  - 关闭名人面板、切换 Canvas 页签或回到资源页时同步清理技能详情状态，避免旧 tooltip 残留。
- 2026-05-30：`0.1.158` 修正名人技能说明和列表翻页。
  - 主动战法详情改为 `释放后，需要等自己再出手 N 次` 等玩家口径，去掉 `冷却就绪 / 目标存活 / 自身行动` 这类工程描述。
  - 新生成的主动技能 `description` 同步改为 `再出手 N 次才能再放`，避免新寻访名人继续写入旧文案。
  - 名人面板已加入列表增加上一页/下一页，超过当前可见数量后可以继续查看更早加入的名人。

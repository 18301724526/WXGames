# 武将品质自动成长属性写回第一批落地任务 v0.1

日期：2026-05-30

目标版本：`0.1.167`

## 背景

`0.1.164` 已经把战斗经验写回出征主将，`0.1.165` 已经实现自由属性点分配，`0.1.166` 已经把加点入口收口到名人详情页。

当前缺口是：文档中已确定“每 10 级按品质获得自动成长点”，但代码里升级节点只发放自由属性点，还没有把品质自动成长写入六维。

## 本批目标

- 每达到 `10 / 20 / 30...` 等级节点，继续固定获得 `10` 点自由属性点。
- 同一节点按品质获得自动成长总点数：
  - 一般 `common`：`4`
  - 良才 `good`：`6`
  - 英杰 `great`：`8`
  - 传奇 `legendary`：`10`
- 自动成长按武将主定位权重分配到六维，并立即写入 `person.attributes`。
- 自动成长和玩家手动加点分别记录，避免后续重置自由点时误动自动成长。
- 旧已加入名人如果已经达到成长节点但没有自动成长记录，归一化时补齐应有自动成长，并写入节点账本。

## 数据规则

已加入名人新增或补齐：

```js
{
  autoAttributeGrowth: {
    command: 1,
    force: 4,
    speed: 2
  },
  earnedAutoAttributePoints: 7,
  autoGrowthMilestones: [10]
}
```

字段含义：

- `autoAttributeGrowth`：历史累计自动成长明细，只记录大于 `0` 的六维。
- `earnedAutoAttributePoints`：历史累计自动成长总点数。
- `autoGrowthMilestones`：已经应用过自动成长的等级节点，用于防重复。
- `assignedAttributePoints` 继续只记录玩家手动分配的自由属性点。

## 分配算法

1. 先根据品质得到本次自动成长总点数。
2. 按定位权重计算每个属性的原始点数。
3. 先取整数部分。
4. 剩余点数按小数部分从高到低补足。
5. 小数相同时，优先补权重更高的主属性；仍相同则按六维固定顺序兜底。

## 实现范围

- `FamousPersonService`
  - 新增品质成长点表和定位权重表。
  - 新增自动成长计算、合并和写回 helper。
  - `normalizeProgression()` 补齐自动成长字段。
  - `normalizePerson()` 对旧名人补齐历史成长节点。
  - `grantBattleExperience()` 在升级跨过节点时同时写回自由点和自动成长。
  - `assignAttributePoint()` 保留自动成长账本，并同步 `strategy/governance` 旧别名。
- `UIStatePresenter`
  - 名人详情 view state 输出 `autoGrowthText / autoGrowthTotal`。
- `CanvasGameRenderer`
  - 详情页在等级与自由点区域下方展示一行累计自动成长摘要。

## 不做范围

- 不做自由属性点重置。
- 不做自动成长撤销。
- 不调整经验曲线和战斗公式。
- 不改变候选名人展示；候选还没有等级成长字段。
- 不在头像册列表塞自动成长详情，避免列表再次拥挤。

## 验证点

- 9 级升到 10 级时，自由属性点增加 `10`，自动成长也按品质写入六维。
- 自动成长总点数与品质一致。
- 同一成长节点不会重复写入。
- 旧 10 级以上名人归一化后补齐自动成长账本。
- 玩家手动加点后，`assignedAttributePoints` 和 `autoAttributeGrowth` 不混在一起。
- 名人详情页能看到累计自动成长摘要。

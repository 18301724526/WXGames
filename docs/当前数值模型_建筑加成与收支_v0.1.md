## 1. 建模范围

本文只记录当前代码实际生效的数值模型，不改变任何玩法逻辑。

覆盖范围：

- 建筑建造、升级、开放等级后的成本与加成曲线
- 在线资源 tick、离线收益、人口成长、人才方针
- 时代进阶消耗与进阶奖励
- 主线任务奖励、事件奖励/消耗/惩罚/buff
- 军事训练、侦察、占领、分城初始资源
- 当前已配置但未生效的建筑维护成本

资源键：

| 资源 | key | 当前持续产出状态 |
| --- | --- | --- |
| 粮食 | `food` | 有产出，有人口消耗 |
| 木材 | `wood` | 有产出，需要伐木场 + 工匠 |
| 铁矿 | `iron` / `metal` | 资源栏存在，持续产出为 0 |
| 石料 | `stone` | 资源栏存在，持续产出为 0 |

- 宜居度：0

## 2. 在线资源模型

在线推进入口会把距离上次更新时间的秒数限制到最多 60 秒，然后对每座城市执行一次资源、人口、军事训练推进。

### 2.1 粮食

变量：

- `F`：农民数量
- `P`：总人口
- `H`：当前幸福度
- `foodOutputMultiplier`：建筑粮食倍率，基础为 1
- `territoryFoodOutputBonus`：城市所在地块的粮食产出加成
- `globalOutputMultiplier`：全局产出倍率，当前基础为 1
- `eventFoodBuff`：事件粮食产出 buff，形式为 `1 + buffValue`

公式：

```text
happinessFactor = max(0.5, H / 100)
粮食总产出/s = F * 1.0 * foodOutputMultiplier
             * (1 + territoryFoodOutputBonus)
             * happinessFactor
             * globalOutputMultiplier
             * eventFoodBuff

人口粮食消耗/s = P * 0.2
粮食净产出/s = 粮食总产出/s - 人口粮食消耗/s
```

实际写回资源时会做下限保护：

```text
food = max(0, food + 粮食净产出/s * 秒数)
```

所以粮食可以被负净产出扣到 0，但不会变成负数。

### 2.2 知识

变量：

- `P`：总人口
- `S`：学者数量
- `knowledgeOutputMultiplier`：学院提供的学者产出倍率，基础为 1
- `territoryKnowledgeOutputBonus`：城市所在地块的知识产出加成
- `globalOutputMultiplier`：全局产出倍率，当前基础为 1
- `eventKnowledgeBuff`：事件知识产出 buff，形式为 `1 + buffValue`

公式：

```text
人口基础知识/s = P * 0.05 * globalOutputMultiplier
学者知识/s = S * 0.15 * knowledgeOutputMultiplier
          * (1 + territoryKnowledgeOutputBonus)
          * globalOutputMultiplier

知识产出/s = (人口基础知识/s + 学者知识/s) * eventKnowledgeBuff
```

注意：学院和领土知识加成只作用在“学者知识”部分，不作用在“人口基础知识”部分。

### 2.3 木材

变量：

- `C`：工匠数量
- `woodOutputBase`：伐木场提供的基础木材产能，基础为 0
- `craftsmanOutputMultiplier`：工坊提供的工匠倍率，基础为 1
- `territoryWoodOutputBonus`：城市所在地块的木材产出加成
- `globalOutputMultiplier`：全局产出倍率，当前基础为 1
- `eventWoodBuff`：事件木材产出 buff，形式为 `1 + buffValue`

公式：

```text
如果 C <= 0 或 woodOutputBase <= 0，则木材产出/s = 0

木材产出/s = C * 1.0 * woodOutputBase
           * craftsmanOutputMultiplier
           * (1 + territoryWoodOutputBonus)
           * globalOutputMultiplier
           * eventWoodBuff
```

### 2.4 铁矿与石料

当前实际公式：

```text
铁矿产出/s = 0
石料产出/s = 0
```

铁矿、石料已经进入资源结构和 UI，但目前没有自然产出、任务奖励、事件奖励或建筑消耗链路。

## 3. 人口、幸福度、宜居度

### 3.1 人口上限

当前启用双上限：

```text
民居上限 = 3 + 民居建筑加成
时代上限 = eraCaps[currentEra]
实际人口上限 = min(民居上限, 时代上限)
```

时代上限：

| 时代 | 上限 |
| --- | ---: |
| 原始 | 3 |
| 农耕 | 6 |
| 聚落 | 9 |
| 城邦 | 12 |
| 边境 | 16 |
| 古典 | 20 |

### 3.2 人口成长

```text
成长倍率 = clamp(1 + 宜居度 / 100, 0.5, 1.5)
成长进度 += 秒数 * 成长倍率
```

当成长进度达到 120 时，尝试增加 1 人：

- 当前粮食必须大于 0
- 总人口必须小于实际人口上限
- 成功时：总人口 +1，未分配人才 +1，成长进度 -120

### 3.3 幸福度

```text
幸福度 = 100 + 建筑幸福度加成 + 事件临时幸福度
```

当前没有任何建筑提供幸福度加成。实际只有事件“旧屋修缮 / 修好屋顶”会临时提供 `幸福度 +5`，持续 5 分钟。

幸福度当前主要影响粮食产出：

```text
happinessFactor = max(0.5, 幸福度 / 100)
```

### 3.4 宜居度

宜居度由城市地理和建筑等级搭配计算。注意：这里使用的是建筑等级，不是多栋建筑数量，因为当前每类建筑只能建 1 个、通过等级表示规模。

地理参数：

| 地理 | 农田/民居目标 | 生产容忍度 | 倾向 |
| --- | ---: | ---: | --- |
| 平原 | 1.60 | 0.38 | 农业、民居、均衡 |
| 山地 | 1.10 | 0.58 | 防御、采掘、紧凑 |
| 沿海 | 1.25 | 0.45 | 探索、贸易、均衡 |
| 森林 | 1.20 | 0.62 | 木材、早期工业 |
| 河谷 | 1.45 | 0.42 | 农业、知识、人口 |

公式：

```text
如果民居等级 <= 0：
  宜居度 = 0

农田民居比 = 农田等级 / max(1, 民居等级)
农田项 = clamp(18 - abs(农田民居比 - 地理目标) * 18, -18, 18)

生产比 = (伐木场等级 + 瞭望台等级 * 0.5) / max(1, 民居等级)
如果生产比 <= 地理生产容忍度：
  生产项 = +8
否则：
  生产项 = -clamp((生产比 - 地理生产容忍度) * 22, 0, 18)

兵营比 = 兵营等级 / max(1, 民居等级)
如果 兵营等级 > 0 且 民居等级 >= 2 且 兵营比 <= 0.45：
  军事项 = +5
否则如果 兵营比 > 0.45：
  军事项 = -10

组合项：
  如果 民居>=2 且 农田>=2 且 伐木场>=1，则 +5
  如果 民居>=3 且 农田<=1，则 -8

宜居度 = round(clamp(总分, -50, 50))
```

描述档位：

| 分数 | 描述 |
| ---: | --- |
| >= 24 | 舒展 |
| >= 8 | 良好 |
| >= -7 | 平稳 |
| >= -24 | 紧张 |
| < -24 | 拥挤 |

## 4. 人才方针模型

人才方针不消耗资源，只改变人口分配，从而影响粮食、知识、木材产出。

职业：

| 职业 | 人口字段 | 解锁 |
| --- | --- | --- |
| 农民 | `farmers` | 原始时代 |
| 学者 | `scholars` | 原始时代 |
| 工匠 | `craftsmen` | 聚落时代 |

系统方针：

| 方针 | 权重 | 优先补余数 |
| --- | --- | --- |
| 均衡发展 | 农民 2 / 学者 1 / 工匠 1 | 农民、学者、工匠 |
| 农业优先 | 农民 4 / 学者 1 / 工匠 1 | 农民、学者、工匠 |
| 知识优先 | 农民 2 / 学者 3 / 工匠 1 | 学者、农民、工匠 |
| 工业优先 | 农民 2 / 学者 1 / 工匠 3 | 工匠、农民、学者；聚落时代解锁 |
| 扩张准备 | 农民 3 / 学者 1 / 工匠 2 | 农民、工匠、学者 |

分配算法：

```text
1. 只保留当前时代已解锁职业
2. 按权重计算每个职业 exact = 总人口 * 职业权重 / 权重和
3. 先取 floor(exact)
4. 剩余人口按：权重高 > 小数余数高 > 方针优先级 > 职业名 排序补 1
```

自定义方针是在系统方针基础上做三档微调：

- 档位 3：对应职业权重 +2
- 档位 2：不变
- 档位 1：对应职业权重 -1
- 最低权重保护为 1

## 5. 建筑加成模型

基础建筑效果：

```text
foodOutputMultiplier = 1
populationCap = 3
happinessBonus = 0
knowledgeOutputMultiplier = 1
craftsmanOutputMultiplier = 1
woodOutputBase = 0
offlineEfficiencyBonus = 0
threatDefense = 0
globalOutputMultiplier = 1
```

每个建筑的加成字段来自 `effects.perLevel`。当前所有建筑都开启了开放规模：

```text
如果 level <= 原配置 maxLevel：
  总加成 = level * perLevel

如果 level > 原配置 maxLevel：
  总加成 = maxLevel * perLevel + 额外等级加成
```

额外等级加成曲线：

```text
linear:      每个额外等级效率 = 1
step:        每个额外等级效率 = 0.5
diminishing: 第 i 个额外等级效率 = 0.05 + 0.95 / sqrt(i + 2)
```

最终加成保留 6 位小数。小于 1% 但大于 0 的正向 UI 提升显示为“提升 <1%”。

## 6. 建筑建造/升级消耗

建造和升级都作用在当前激活城市；时代进阶只作用在主城。

建造条件：

- 建筑存在
- 当前时代实际已解锁
- 当前城市尚未建造该建筑
- 当前城市资源足够

升级条件：

- 建筑存在
- 当前城市已建造该建筑
- 当前建筑可升级
- 当前城市资源足够

扣除模型：

```text
resources[key] = max(0, resources[key] - cost[key])
```

因为扣除前已经校验资源足够，所以正常情况下不会靠 `max(0)` 吞掉赤字。

士兵奖励会受到当前主城兵营士兵上限限制：

```text
发放后士兵 = min(当前兵营上限, 当前士兵 + 奖励士兵)
```

因此如果兵营上限不足，士兵奖励会被截断。

每日任务、赛季任务、挑战任务目前为空，没有奖励模型。

## 10. 事件消耗、获得与 buff

事件资源效果当前只支持粮食、木材、知识，不支持铁矿、石料。

普通事件：

- 开启条件：聚落时代及以后，并且教程聚落阶段完成
- 生成间隔：4 分钟
- 队列上限：3
- 过期时间：5 分钟
- 普通事件过期：不触发资源惩罚，只记录错过

威胁事件：

- 开启条件：边境时代及以后
- 生成间隔：6 分钟
- 队列上限：2
- 过期时间：5 分钟
- 威胁事件过期：自动执行最差惩罚

## 11. 离线收益

登录时若离线超过 60 秒，会计算离线收益。最大离线时长：

```text
maxOfflineHours = 8
```

每座城市单独计算：

```text
实际离线秒数 = min(离线秒数, 8小时)
离线效率 = 0.8 + 神庙离线效率加成 + 事件离线效率 buff

离线粮食 = max(0, floor(粮食净产出/s * 实际离线秒数 * 离线效率))
离线知识 = max(0, floor(知识产出/s * 实际离线秒数 * 离线效率))
离线木材 = max(0, floor(木材产出/s * 实际离线秒数 * 离线效率))
离线铁矿 = 0
离线石料 = 0
```

离线不会扣负收益：如果粮食净产出为负，离线粮食收益按 0 处理。离线期间士兵训练会按实际离线秒数推进。

## 12. 军事模型

兵营训练表：

| 兵营等级 | 士兵上限 | 训练间隔 |
| ---: | ---: | ---: |
| 0 | 0 | 0 |
| 1 | 5 | 30 秒 / 士兵 |
| 2 | 10 | 20 秒 / 士兵 |
| >=3 | 10 | 20 秒 / 士兵 |

当前兵营开放等级后的成本会继续上涨，但军事上限和训练速度 2 级后不会继续成长。

训练公式：

```text
如果 士兵 < 士兵上限：
  训练进度 += 秒数
  每满训练间隔，士兵 +1
```

防御：

```text
士兵防御 = 士兵数量 * 1
```

## 13. 侦察、占领与分城获得

### 13.1 侦察

开启条件：古典时代及以后。

```text
侦察耗时 = 60 秒
最多同时侦察队 = 2
最大侦察距离 = 24 格
侦察不消耗资源，也不占用士兵
```

发现地点概率：

```text
地点概率 = min(1, 0.32 + 连续空地次数 * 0.14)
如果连续空地次数 >= 4，则必出地点
```

地点类型会受距离和连续发现中立地点次数影响。被发现地点的防御与推荐士兵会随距离增加。

### 13.2 地点模板

| 类型 | 所属 | 基础规模 | 基础防御 | 推荐士兵 | 占领后效果 |
| --- | --- | ---: | ---: | ---: | --- |
| 前哨 | 中立 | 1 | 3 | 3 | 威胁防御 +1 |
| 村镇 | 中立 | 2 | 4 | 4 | 粮食产出 +5% |
| 部落营地 | 部落 | 2 | 5 | 5 | 木材产出 +8% |
| 城邦 | 城邦 | 3 | 6 | 6 | 粮食产出 +6%，知识产出 +3% |
| 遗迹 | 遗迹守军 | 2 | 7 | 7 | 知识产出 +8% |

距离修正：

```text
地点防御 += max(0, 距离 - 1)
推荐士兵 = max(防御, 模板推荐士兵 + max(0, 距离 - 1))

粮食/木材/知识产出加成 += 0.01 * max(0, 距离 - 1)
威胁防御 += floor((距离 - 1) / 2)
```

### 13.3 占领

占领耗时：

```text
占领耗时 = 120 秒
```

中立地点：

```text
固定派出 1 士兵
成功占领
伤亡 0
```

非中立地点：

```text
成功条件 = 派出士兵 >= 地点防御

成功伤亡 = min(max(0, 派出士兵 - 1), floor(地点防御 / 3))
失败伤亡 = ceil(派出士兵 / 2)
```

占领不直接消耗资源，但会占用士兵，并可能在结算时损失士兵。

### 13.4 分城初始获得

占领地点后会为该地点创建城市：

```text
人口 = max(2, min(8, 地点规模 + 2))
粮食 = 80 + 地点规模 * 40
知识 = 地点规模 * 8
木材 = 如果地点类型为部落营地则 60，否则 25 + 地点规模 * 10
铁矿 = 0
石料 = 0
建筑 = 空
人口默认全部为农民
幸福度 = 100
```

领土效果只加到对应城市的 `buildingEffects` 上，不是所有城市共享同一个产出倍率。

## 15. 当前模型的明显待计算点

1. 兵营 2 级以后成本继续上涨，但士兵上限和训练速度不再成长。
2. 工坊、学院、神庙配置为 5 时代建筑，但当前实际解锁表没有放出。
3. 铁矿、石料已经进入资源栏，但没有产出、消耗和奖励闭环。
4. 维护成本已有配置但完全未生效；启用维护前需要先重算前期粮食、木材压力。
5. 领土威胁防御在城市派生属性里存在，但威胁事件需求检查没有吃到领土防御。
6. 任务奖励士兵会被兵营上限截断，可能出现“奖励写 +6，实际只补到上限”的情况。
7. 学院只提升学者知识，不提升人口基础知识；工坊目前不能建造，所以工匠倍率链路尚未实际进入游戏。

后续计算建议先按阶段拆：

- 阶段 A：原始 -> 聚落，校准粮食和知识节奏
- 阶段 B：聚落 -> 城邦，校准木材和工匠价值
- 阶段 C：城邦 -> 边境，校准兵营、士兵训练和任务奖励是否溢出
- 阶段 D：边境 -> 古典，校准瞭望台、兵营 2 级和古典侦察前资源压力
- 阶段 E：古典后，补铁矿/石料/维护/工坊学院神庙的长期循环

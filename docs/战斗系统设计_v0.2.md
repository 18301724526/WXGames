# 战斗系统设计 v0.2

日期：2026-05-29

## 文档定位

本文用于承接后续战斗系统重做，重点定义武将属性、成长、战斗公式、技能和战报结构。

当前代码中的 `attribute-auto-battle-v2` 是第一版正式战斗内核底座：速度排序、条件释放、冷却按自身行动推进和结构化战报已落地。战斗场景播放和前端消费战报的现状仍参考 `docs/回合制战斗场景与技能冷却设计_v0.2.md`。

本版按“一个问题一个问题确认”的方式推进。已经确认的内容写入“已确定”；未确认的内容只写入“待讨论”，不提前定案。

## 已确定：武将属性

武将采用六维属性：

| 属性 | 建议字段 | 职责 |
| --- | --- | --- |
| 统帅 | `command` | 影响部队防御、减伤和承伤稳定性。 |
| 武力 | `force` | 影响兵刃伤害。 |
| 智力 | `intelligence` | 影响谋略伤害。 |
| 魅力 | `charisma` | 后续影响寻访、探索、侦查事件等非直接战斗内容。 |
| 政治 | `politics` | 后续影响城建、治理、城市发展等内容。 |
| 速度 | `speed` | 影响出手先后顺序。 |

第一版战斗只使用：

- 统帅
- 武力
- 智力
- 速度

魅力和政治保留为完整武将属性，但暂不进入第一版战斗公式。

历史字段兼容：

- 旧 `strategy` 语义后续应迁移为 `intelligence`。
- 当前存档和代码仍可能存在 `strategy`，正式实装前需要补迁移和兼容读取。

## 已确定：属性值域

- 初始属性范围为 `1-100`。
- 当前面板属性不设置硬上限。
- 初始属性代表武将出生面板，不代表后续成长天花板。

属性超过 `100` 后的战斗收益曲线见“属性有效值”。第一版所有战斗属性先使用同一条递减收益曲线。

## 已确定：武将成长

武将属性由三部分组成：

```text
当前属性 = 初始属性 + 自动成长属性 + 玩家分配属性点
```

成长规则：

- 每升 `10` 级，固定获得 `10` 点自由属性点。
- 自由属性点由玩家分配。
- 自由属性点获取不受品质影响。
- 品质只影响自动成长数值。
- “自由属性点”和“技能点”需要区分；当前确认的是属性点，技能点暂不设计。

品质职责：

- 品质影响初始属性区间。
- 品质影响每个成长周期获得的自动成长总量。
- 品质不影响玩家每 `10` 级获得的自由属性点数量。

自动成长分配方向见“自动成长分配”。第一版按武将主定位决定成长倾向。

## 已确定：品质分层

武将品质先采用四档，不做复杂稀有度包装：

| 品质 | 字段建议 | 定位 | 每 10 级自动成长总点数 |
| --- | --- | --- | --- |
| 一般 | `common` | 基础可用，适合早期过渡。 | 4 |
| 良才 | `good` | 明显好用，有培养价值。 | 6 |
| 英杰 | `great` | 核心武将，主属性成长突出。 | 8 |
| 传奇 | `legendary` | 稀有强者，长期培养价值最高。 | 10 |

说明：

- 名人生成时直接生成品质，品质对玩家可见。
- 每 10 级的自由属性点仍固定为 `10`，不受品质影响。
- 品质只影响自动成长总点数。
- 自动成长点数后续会按武将定位分配到六维属性上。
- 品质影响初始属性区间、自动成长总点数和技能生成预算。
- 当前数值先保持温和，避免品质差距在早期直接压死培养空间。

寻访品质概率：

| 品质 | 概率 |
| --- | ---: |
| 一般 `common` | `32.4%` |
| 良才 `good` | `26%` |
| 英杰 `great` | `36%` |
| 传奇 `legendary` | `5.6%` |

寻访保底：

- 寻访暂时不设置冷却。
- 寻访暂时不消耗资源或抽卡道具。
- 寻访是当前阶段临时获得名人的入口，不接内购充值。
- 寻访保底只统计寻访来源，不跨事件投奔、战后归附或其他后续来源。
- 连续 `29` 次寻访未生成 `legendary` 时，第 `30` 次寻访强制生成 `legendary`。
- 生成 `legendary` 后，寻访保底计数清零。
- 第一版只开放寻访生成名人。
- 事件投奔、战后归附和世界探索获得名人暂时关闭；项目中保留历史数据兼容和展示文本，但不再生成新候选。
- 事件投奔、战后归附和世界探索获得名人的规则后续单独设计，不共享寻访保底。

## 已确定：等级与经验

武将经验主要来自战斗。

经验来源原则：

- 胜利和失败都会获得经验。
- 经验不按简单胜负二元发放，而按战斗实际参与和战果计算。
- 剿灭敌军越多，经验越多。
- 只剿灭极少量敌军时，经验应非常少。
- 战损也可以提供少量经验，用于表达苦战经历，但不能高于有效杀伤带来的经验。

第一版经验建议公式：

```text
战斗经验 = 敌军损失 × 1.0 + 己方损失 × 0.25 + 胜利奖励
```

其中：

- `敌军损失` 按实际消灭士兵数量计算。
- `己方损失` 按本方实际阵亡士兵数量计算。
- 胜利奖励用于补足成功作战的正反馈，但不能替代杀伤贡献。

胜利奖励建议：

```text
胜利奖励 = max(20, 敌军初始兵力 × 0.05)
```

例子：

- 只消灭 `1` 个敌军，失败撤退：约 `1` 点经验，几乎没有成长。
- 消灭 `300` 敌军，自损 `100`，失败撤退：约 `325` 点经验。
- 消灭 `500` 敌军并胜利，自损 `120`：约 `550` 点经验。

## 已确定：升级曲线

武将等级不设置硬上限。升级曲线采用平滑递增，避免长线游戏后期突然断崖式变慢。

每级所需经验：

```text
升到下一级所需经验 = roundTo10(100 + 35 × 当前等级 + 6 × 当前等级 ^ 1.5)
```

说明：

- `roundTo10` 表示四舍五入到最接近的 `10`。
- 当前等级从 `1` 开始。
- 经验允许累计，满足多级需求时可以连续升级。
- 曲线由线性项和缓慢幂次项组成，前期有成长反馈，后期仍能长期推进。

参考值：

| 当前等级 | 升到下一级所需经验 |
| --- | --- |
| 1 | 140 |
| 10 | 640 |
| 20 | 1340 |
| 50 | 3970 |
| 100 | 9600 |
| 200 | 24070 |
| 500 | 84680 |

每达到 `10` 的倍数等级时，触发一次成长节点：

- 获得 `10` 点自由属性点。
- 根据武将品质获得自动成长点数。
- 自动成长点数按武将定位权重分配。

## 已确定：速度

速度暂时只决定出手先后顺序。

第一版规则：

- 每轮开始按速度排序。
- 速度高者先行动。
- 速度低者后行动。
- 同速时默认己方先行动。
- 速度暂不影响行动频率。
- 速度暂不产生连动、追击、额外回合或行动条。

## 已确定：自动成长分配

自动成长点数按武将定位分配。第一版先用主定位，不做复杂多定位混合。

分配算法：

```text
本次自动成长点数 = 品质成长总点数
各属性获得点数 = 本次自动成长点数 × 定位权重
```

实现时先取整数部分，剩余点数按小数部分从高到低补足；若小数相同，优先补主属性。

定位权重：

| 定位 | 字段建议 | 统帅 | 武力 | 智力 | 魅力 | 政治 | 速度 |
| --- | --- | ---: | ---: | ---: | ---: | ---: | ---: |
| 统帅 | `commander` | 45% | 15% | 10% | 10% | 10% | 10% |
| 猛将 | `vanguard` | 15% | 45% | 5% | 10% | 5% | 20% |
| 谋略 | `strategist` | 10% | 5% | 50% | 10% | 15% | 10% |
| 内政 | `governor` | 5% | 5% | 15% | 20% | 50% | 5% |
| 魅力 | `charmer` | 10% | 5% | 15% | 45% | 20% | 5% |
| 斥候 | `scout` | 10% | 10% | 20% | 15% | 5% | 40% |

例子：

- `英杰` 谋略型每 `10` 级自动成长 `8` 点，主要落在智力，其次政治、统帅/魅力/速度。
- `传奇` 猛将型每 `10` 级自动成长 `10` 点，主要落在武力，其次速度和统帅。

## 已确定：伤害公式方向

第一版战斗只做确定性自动战斗，不做命中、闪避、暴击和行动条。

### 属性有效值

面板属性可以无限成长，但战斗收益使用有效属性，超过 `100` 后进入递减收益：

```text
有效属性 = 属性 <= 100
  ? 属性
  : 100 + 2 × (属性 - 100) ^ 0.75
```

说明：

- `100` 以内保持直观线性。
- `100` 以后继续成长，但收益逐步变缓。
- 该公式同时适用于统帅、武力、智力、速度等战斗属性。

### 通用伤害框架

士兵数量既是承伤池，也影响输出规模。兵力越少，后续伤害越低。

```text
兵力系数 = (当前兵力 / 100) ^ 0.65
攻击系数 = 0.75 + 有效攻击属性 / 120
防御减伤 = clamp(有效防御属性 / (有效防御属性 + 180), 0.05, 0.60)
最终伤害 = round(35 × 兵力系数 × 攻击系数 × (1 - 防御减伤) × 技能倍率)
```

伤害下限：

```text
最终伤害 = max(1, min(目标当前兵力, 最终伤害))
```

### 伤害类型

| 类型 | 使用属性 | 防御属性 | 用途 |
| --- | --- | --- | --- |
| 兵刃伤害 | 武力 | 目标统帅 | 普通攻击、突击、斩击等。 |
| 谋略伤害 | 智力 | 目标智力 | 火攻、计策、扰乱等。 |

第一版普通攻击默认是兵刃伤害，技能根据模板指定 `blade` 或 `strategy`。

说明：

- 统帅主要防御兵刃伤害。
- 智力既影响谋略伤害，也防御谋略伤害。
- 高智力武将使用谋略伤害攻击低智力目标时应有明显优势。

技能倍率建议：

| 行为 | 倍率范围 |
| --- | --- |
| 普通攻击 | `1.00` |
| 小技能 | `1.15-1.30` |
| 标准主动技能 | `1.35-1.60` |
| 高风险强技能 | `1.70-2.00` |

本阶段先不加入随机伤害浮动。后续如需增加战报变化，可以加入稳定的伪随机浮动，但必须由后端生成并写入战报。

## 已确定：士气字段

士气先保留为战斗字段，但第一版不实际影响伤害公式。

```js
{
  morale: 100,
  moraleEffectEnabled: false
}
```

说明：

- 战报可以展示士气。
- 后端战斗单位保留士气字段。
- 第一版伤害公式不乘士气系数。
- 暂不设计士气来源、恢复、崩溃、溃逃、补给和连续作战影响。
- 后续如果做真实大地图、补给线、远征疲劳、围城或地形，再统一接入士气系统。

## 已确定：技能系统 v1

武将技能按定位生成，不强制所有武将都拥有战斗技能。

第一版技能组分为三类：

| 定位 | 技能组 | 战斗行为 |
| --- | --- | --- |
| 统帅 `commander` | 战斗技能组 | `1` 个主动战法 + `1` 个战斗被动特质 |
| 猛将 `vanguard` | 战斗技能组 | `1` 个主动战法 + `1` 个战斗被动特质 |
| 谋略 `strategist` | 战斗技能组 | `1` 个主动战法 + `1` 个战斗被动特质 |
| 内政 `governor` | 内政技能组 | 不生成战斗技能，被派上战场时只普攻 |
| 魅力 `charmer` | 内政技能组 | 不生成战斗技能，被派上战场时只普攻 |
| 斥候 `scout` | 混合技能组 | `1` 个轻战斗主动战法 + `1` 个侦查/行军被动特质 |

战斗技能组第一版最多拥有：

- `1` 个主动战法。
- `1` 个被动特质。

主动战法：

- 有冷却。
- 冷却按自身行动次数推进。
- 开局不再默认必放技能，必须先检查技能释放条件。
- 当前行动满足释放条件且冷却结束时，释放主动战法。
- 当前行动不满足释放条件，或冷却未结束时，执行普通攻击。
- 第一版不做概率触发，避免战报不可解释。
- 只有拥有战斗主动战法的武将才可能释放技能；无战斗主动战法的武将始终普攻。
- 技能条件必须是确定性条件，不能使用战斗中随机触发。

主动战法释放结构：

```js
{
  cooldown: 3,
  castPolicy: 'conditional',
  castConditions: [
    { type: 'cooldownReady' },
    { type: 'targetAlive' }
  ]
}
```

第一版允许的释放条件：

| 条件 | 字段 | 含义 |
| --- | --- | --- |
| 冷却结束 | `cooldownReady` | 技能冷却为 `0`。 |
| 目标存活 | `targetAlive` | 目标当前兵力大于 `0`。 |
| 自身兵力低于比例 | `selfSoldierBelowPct` | 己方当前兵力比例低于阈值。 |
| 自身兵力高于比例 | `selfSoldierAbovePct` | 己方当前兵力比例高于阈值。 |
| 目标兵力低于比例 | `targetSoldierBelowPct` | 目标当前兵力比例低于阈值。 |
| 目标存在状态 | `targetHasStatus` | 目标身上存在指定状态。 |
| 自身存在状态 | `selfHasStatus` | 自身存在指定状态。 |
| 首次行动 | `firstOwnAction` | 该单位本场战斗第一次行动。 |

说明：

- `cooldownReady` 和 `targetAlive` 是所有主动战法的基础条件。
- 如果技能没有额外条件，只要冷却结束且目标存活即可释放。
- 如果技能有额外条件，必须全部满足才释放。
- 不满足条件不会保留“待释放队列”，本次行动直接普攻，下次自己行动前重新判断。
- `firstOwnAction` 可用于斥候 `firstStrike` 等首动技能，但也必须写入条件，不再作为全局默认规则。

被动特质：

- 只在准备阶段或明确条件下触发。
- 第一版优先用于属性修正、开战增益或简单状态。
- 不做复杂连锁、不打断行动顺序。

内政技能组第一版拥有：

- `2` 个内政技能。
- 不生成主动战法。
- 不生成战斗被动特质。
- 不参与战斗战报。
- 生成后挂在名人存档上，可在名人面板展示。
- 第一阶段不实现内政技能生效入口，不需要先设计岗位系统。
- 如果内政/魅力型武将被强行派去领兵，战斗中只执行普通攻击，不获得 fallback 战斗技能。

斥候混合技能组第一版拥有：

- `1` 个轻战斗主动战法。
- `1` 个侦查/行军被动特质。
- 战斗主动强度低于猛将和谋略定位。
- 战斗主动优先体现先手、扰乱、穿插和小规模爆发。
- 侦查/行军被动第一阶段可以先生成、存档和展示；真实侦查生效入口后续再接入。
- 斥候上战场时可以释放自己的轻战斗主动战法，不走 `basicAttackOnly`。

技能类型：

| 类型 | 字段 | 主要属性 | 作用 |
| --- | --- | --- | --- |
| 兵刃 | `blade` | 武力 | 造成兵刃伤害。 |
| 谋略 | `strategy` | 智力 | 造成谋略伤害。 |
| 守御 | `guard` | 统帅 | 提供减伤、护盾、防御类效果。 |
| 支援 | `support` | 统帅或智力 | 恢复、增益、削弱等简单效果。 |

内政技能类型：

| 类型 | 字段 | 主要属性 | 作用 |
| --- | --- | --- | --- |
| 生产 | `production` | 政治 | 提高资源产出、基础产能。 |
| 工程 | `construction` | 政治 | 提高建造速度、降低建造消耗。 |
| 治理 | `governance` | 政治或魅力 | 提高人口、幸福度、城市稳定。 |
| 人望 | `reputation` | 魅力 | 提高事件收益、安抚、说服、归附相关效果。 |
| 情报 | `intel` | 魅力或速度 | 提高侦查、报告质量、后续外交谈判表现。 |

第一版暂不做：

- 暴击。
- 闪避。
- 命中率。
- 连击。
- 控制跳过回合。
- 追击和额外行动。

## 已确定：技能生成 v1

技能系统采用“模块化肉鸽生成 + 确定性结算”。

核心原则：

- 普通武将不预先维护固定技能表。
- 普通技能由效果原子、组合规则、命名规则和强度预算生成。
- 技能一旦生成，就写入该武将存档。
- 战斗技能后续战斗按该技能确定结算。
- 内政技能第一阶段只生成、存档和展示，生效入口后续再设计。
- 战斗过程中不重新随机技能效果，不做概率触发。
- 后期如果出现剧情人物、特殊历史人物或活动人物，可以通过 `uniqueSkillId` 单独实现独特技能。
- 独特技能不进入普通随机池，避免破坏普通技能生成预算。

### 技能生成结构

普通技能生成分三层：

```text
效果原子
→ 组合规则
→ 命名规则
→ 写入武将存档
→ 战斗技能在战斗中确定性结算，内政技能先只展示
```

生成器输入字段：

```js
{
```

### 效果原子

效果原子按两批推进。`0.1.148` 已落地的第一批只开放能稳定解释、稳定结算的类型：

- `directDamage`
- `secondHit`
- `firstStrike`
- `lifesteal`
- `heal`
- `shield`
- `attributeBonus`

`armorBreak`、`burn`、`poison`、`lowSoldierGuard` 需要真实状态系统参与，暂列第二批，不进入 `0.1.148` 普通随机池。

| key | 用途 | 第一版结算方式 |
| --- | --- | --- |
| `directDamage` | 直接伤害 | 按技能倍率造成兵刃或谋略伤害。 |
| `secondHit` | 二段伤害 | 固定追加一次低倍率伤害，不做概率连击。 |
| `lifesteal` | 吸血 | 按实际造成伤害的一定比例恢复己方兵力。 |
| `heal` | 治疗 | 按最大兵力或属性计算恢复量。 |
| `shield` | 护盾 | 获得固定数值护盾或临时减伤。 |
| `armorBreak` | 破甲 | 在限定回合内降低目标有效统帅或防御减伤。 |
| `burn` | 灼烧 | 写入持续状态，每回合造成谋略持续伤害。 |
| `poison` | 中毒 | 写入持续状态，每回合造成较低伤害或降低恢复。 |
| `firstStrike` | 伏击 | 首次行动附加确定伤害，不做概率伏击。 |
| `attributeBonus` | 属性修正 | 战前或条件满足时修正指定属性。 |
| `lowSoldierGuard` | 低兵力守势 | 己方兵力低于阈值时获得减伤。 |

内政效果原子：

| key | 用途 | 第一版结算方式 |
| --- | --- | --- |
| `resourceOutputPct` | 指定资源产出提高 | 第一阶段只生成和展示，后续接城市生效入口。 |
| `allBasicOutputPct` | 基础资源小幅提高 | 第一阶段只生成和展示，后续接城市生效入口。 |
| `constructionSpeedPct` | 建造/升级速度提高 | 第一阶段只生成和展示，后续接城市生效入口。 |
| `constructionCostPct` | 建造/升级消耗降低 | 第一阶段只生成和展示，后续接城市生效入口。 |
| `knowledgeOutputPct` | 知识产出提高 | 第一阶段只生成和展示，后续接城市生效入口。 |
| `populationCapPct` | 人口上限提高 | 第一阶段只生成和展示，后续接城市生效入口。 |
| `happinessFlat` | 幸福度提高 | 第一阶段只生成和展示，后续接城市生效入口。 |
| `trainingSpeedPct` | 士兵训练速度提高 | 第一阶段只生成和展示，后续接军事或城市生效入口。 |
| `eventRewardPct` | 事件正面收益提高 | 第一阶段只生成和展示，后续接事件生效入口。 |
| `eventRiskReductionPct` | 事件负面风险降低 | 第一阶段只生成和展示，后续接事件生效入口。 |
| `settlementPacifyPct` | 新占领城市安抚效率提高 | 后续安抚系统启用后生效，第一版可保留字段。 |
| `famousRetentionPct` | 名人候选保留/说服效果提高 | 名人只开放寻访期间暂缓实现。 |
| `diplomacyBonusPct` | 外交、归附、谈判加成 | 后续外交或归附系统启用后生效。 |
| `scoutReportBonusPct` | 侦查情报质量提高 | 后续侦查辅助入口启用后生效。 |
| `cityStabilityPct` | 城市稳定提高 | 后续稳定/叛乱系统启用后生效。 |

旧效果口径迁移：

| 旧效果 | 新口径 |
| --- | --- |
| `combo` 概率连击 | 改为 `secondHit` 固定二段伤害。 |
| `counter` 概率反击 | 第一版规范化时移除，不进入战斗效果。后续可改成确定的“受击后反击标记”。 |
| `ambush` 概率伏击 | 改为 `firstStrike` 首次行动附加伤害。 |
| `morale` 士气增强 | 士气系统未启用前，改成 `attributeBonus(command)`。 |

规范化迁移规则：

- 新生成技能不再产出 `combo/counter/ambush/morale` 旧效果。
- 旧存档或旧候选进入 `normalizeState` 时自动迁移。
- 迁移后的效果保留 `migratedFrom` 字段，方便排查旧存档来源。
- `counter` 当前被移除，是因为第一版没有反击时机、受击触发和状态系统，保留它会造成“显示有反击但实际不生效”的误导。
- 旧主动技能迁移后仍按 `activeSkill` 参与战斗；内政/魅力型仍按 `basicAttackOnly` 不注入战斗技能。

### 组合规则

主动战法：

- 必须有 `1` 个主效果。
- 可以有 `0-1` 个副效果。
- 必须有冷却。
- 必须有确定性释放条件。
- 必须指定伤害类型或功能类型。
- 不能同时组合多个持续伤害。
- 不能同时组合多个恢复/防御副效果。
- 第一版不生成硬控制技能。
- 不能生成战斗中概率释放条件。

被动特质：

- 必须有明确触发时机。
- 第一版触发时机只允许：
  - `preBattle`：战前生效。
  - `firstRound`：第一回合生效。
  - `lowSoldier`：己方兵力低于阈值时生效。
- 被动特质不插入额外行动，不打断行动顺序。
- 被动特质必须写入准备阶段或状态变化战报。

内政技能：

- `governor` 和 `charmer` 必须生成 `2` 个内政技能。
- 第一技能使用 `civilPrimary` 槽位，第二技能使用 `civilSecondary` 槽位。
- 内政技能必须挂在名人存档上。
- 第一阶段触发条件统一写为 `passiveStored`，表示已生成、可展示、暂不参与结算。
- 内政技能的真实生效入口后续再设计，不阻塞当前战斗系统落地。
- 内政技能不进入战斗流程，不出现在战斗行动段。
- 内政技能不能生成直接战斗伤害、护盾、吸血、灼烧、中毒等战斗效果。
- 内政/魅力型武将被派上战场时，`battlePolicy` 固定为 `basicAttackOnly`。

斥候技能：

- 必须生成 `1` 个轻战斗主动战法。
- 必须生成 `1` 个侦查/行军被动特质。
- 主动战法可用效果优先为 `firstStrike`、`directDamage(strategy)`、低倍率 `secondHit`、速度修正。
- 主动战法不生成高倍率兵刃爆发，不生成强治疗，不生成强护盾。
- 侦查/行军被动第一阶段使用 `scoutTrait` 槽位和 `passiveStored` 触发条件，先挂在名人身上展示。
- 后续接入侦查和行军系统时，再让 `scoutTrait` 参与结算。

| 良才 | `good` | 120 | 50 |
| 英杰 | `great` | 145 | 65 |
| 传奇 | `legendary` | 170 | 80 |

斥候混合技能预算：

| 品质 | 字段 | 轻战斗主动预算 | 侦查/行军被动预算 |
| --- | --- | ---: | ---: |
| 一般 | `common` | 85 | 45 |
| 良才 | `good` | 100 | 55 |
| 英杰 | `great` | 120 | 70 |
| 传奇 | `legendary` | 140 | 90 |

内政技能预算：

| 品质 | 字段 | 主内政技能预算 | 副内政技能预算 |
| --- | --- | ---: | ---: |
| 一般 | `common` | 60 | 35 |
| 良才 | `good` | 80 | 45 |
| 英杰 | `great` | 105 | 60 |
| 传奇 | `legendary` | 135 | 80 |

内政效果强度建议：

| 品质 | 单项长期加成建议 |
| --- | --- |
| 一般 | `4%-6%` |
| 良才 | `6%-8%` |
| 英杰 | `8%-12%` |

```text

建造速度 +8% ≈ 建造消耗 -4%
幸福度 +5 ≈ 单资源 +6%
```

| 效果 | 预算消耗 |

### 定位权重

生成器按武将定位提高不同效果权重。

战斗定位：

| 定位 | 主效果倾向 | 副效果倾向 |

## 已确定：技能状态系统 v1

状态用于承载护盾、破甲、灼烧、中毒、易伤、低兵力守势、持续恢复等跨行动效果。状态必须真实参与结算，不能只作为战报文字。

### 状态结构

建议第一版状态结构：

```js
{
  key: 'burn',
  sourceSkillId: 'skill_generated_xxx',
  sourceSide: 'attacker',
  targetSide: 'defender',
  turnsRemaining: 2,
  stacks: 2,
  maxStacks: 3,
  value: 0.08,
  stackPolicy: 'stack',
  appliedAtRound: 1
}
```

字段说明：

- `key`：状态类型。
- `sourceSkillId`：来源技能。
- `sourceSide`：释放者。
- `targetSide`：承受者。
- `turnsRemaining`：剩余持续次数，按目标自己的行动前扣减。
- `stacks`：当前层数。
- `maxStacks`：最大层数。
- `value`：每层或当前状态的基础数值。
- `stackPolicy`：叠加机制。
- `appliedAtRound`：施加回合，用于战报和调试。

### 结算时机

状态在每个行动单位自己的行动前结算。

顺序：

```text
行动单位回合开始
→ 优先处理自身清状态、净化、免疫类状态
→ 结算自身剩余状态
→ 按状态规则扣减持续次数
→ 移除过期状态
→ 如果该单位兵力归零，则清空该单位身上所有状态
→ 该单位正式行动
```

说明：

- 这里的“回合开始”指每个单位行动前的小回合，不是双方完整大回合。
- 状态挂在谁身上，就在谁行动前结算和扣回合。
- 自身拥有清状态、净化、免疫类状态时，必须优先生效，再处理负面状态。

### 叠加规则

同类状态允许叠加，但必须有明确上限。

第一版建议分类型处理：

| 状态类型 | 叠加方式 | 上限建议 |
| --- | --- | --- |
| 持续伤害 | 增加层数，每层单独贡献伤害权重。 | `3` 层 |
| 易伤/破甲 | 增加层数或刷新强度，但总效果封顶。 | 总减防不超过 `30%` |
| 护盾 | 数值累加。 | 不超过最大兵力 `30%` |
| 持续恢复 | 增加层数或刷新持续时间。 | `3` 层 |
| 属性修正 | 同属性同方向可叠加。 | 单属性临时修正不超过有效属性 `30%` |

当新状态达到上限后：

- 如果是层数型状态，超过上限的新增层数无效，但可以刷新持续时间。
- 如果是数值型状态，超过上限的部分截断。
- 如果新状态强度更高，可以刷新 `value`，但总效果仍受上限约束。

### 持续时间

状态持续时间按目标自己的行动前扣减。

例子：

```text
陆骁给敌人挂灼烧 2 回合
敌人第 1 次行动前：灼烧生效，剩 1
敌人第 2 次行动前：灼烧生效，剩 0，移除
```

速度影响：

- 速度快的单位会更快消耗自己身上的状态。
- 速度慢的单位不会因为其他单位行动而额外扣状态。

### 持续伤害

灼烧、中毒等持续伤害每次触发时动态计算，不在施加时锁定伤害。

每次触发时读取：

- 释放者当前相关属性。
- 受害者当前相关防御属性。
- 双方当前状态修正，例如易伤、减伤、增智、降智。
- 当前层数和每层权重。

伤害属性：

- 灼烧、中毒默认视为谋略伤害。
- 谋略持续伤害使用释放者智力攻击目标智力。
- 兵刃持续伤害如后续存在，则使用释放者武力攻击目标统帅。

例子：

```text
第 1 次触发：释放者智力高、目标智力低，造成 28 伤害
第 2 次触发：目标获得智力提升，伤害降为 20
第 3 次触发：目标又被挂易伤，伤害提升
```

### 死亡与状态清空

任意单位兵力归零后：

- 该单位身上的所有状态立即清空。
- 该单位施加在别人身上的状态也清空。
- 后续不再由该单位的状态产生持续伤害、破甲、易伤或增益。

说明：

- 这样可以避免败退单位继续通过灼烧、中毒等状态影响战局。
- 战报需要记录关键状态清空，例如 `[陆骁] 败退，相关状态消散`。

## 已确定：战斗流程 v1

战斗是后端确定性自动战斗，玩家只做战前配置。

```text
准备阶段
→ 应用被动特质和准备效果
→ 按速度排序
→ 第 1 回合
  → 速度高者行动
  → 速度低者行动
→ 第 2 回合
...
→ 一方兵力为 0 或达到最大回合
→ 战后结算
```

最大回合暂定 `20`。

胜负规则：

- 敌方兵力为 `0`：胜利。
- 己方兵力为 `0`：失败。
- 达到最大回合仍未结束：己方剩余兵力大于等于敌方剩余兵力则胜利，否则失败。

同速规则：

- 同速时默认己方先行动。

## 已确定：战后结算

胜利：

- 地点进入占领流程。
- 敌方剩余兵力视为溃散。
- 第一版暂不触发战后归附候选。
- 发放胜利经验奖励。

失败：

- 地点保持未占领。
- 记录上次战斗结果。
- 仍按实际战斗表现获得经验。
- 暂不做武将受伤、俘虏、士气崩溃或长期负面状态。

伤亡：

- 己方伤亡按战斗中实际损失扣除。
- 敌方损失用于经验计算和战报展示。
- 出征士兵返回数量等于出征兵力减去战斗损失。

经验：

- 使用“等级与经验”章节中的公式。
- 胜负都会获得经验。
- 战后归附暂时关闭，后续如恢复，只允许在胜利时触发。

## 已确定：战报表现方向

战报参考“三国志战略版”这类结构化详细战报的信息密度，但不照搬视觉样式。

目标：

- 玩家能看懂每一回合谁先行动。
- 玩家能看懂普攻、技能、恢复、状态变化分别发生了什么。
- 玩家能看懂伤害类型、损失兵力和剩余兵力。
- 战报可被前端播放，也可被文本面板直接展示。

### 技能释放演出

主动技能释放时必须有明显的名人表现，不只是在小兵上播放特效。

第一版表现规则：

- 技能事件进入 `impact` 阶段时，释放者名人立绘从画面侧边或战场上方浮出。
- 立绘使用名人系统当前三层立绘资源。
- 立绘旁显示技能名。
- 技能名必须来自战报事件中的 `skillName`，前端不自行推断。
- 技能名和立绘只在 `actionType: 'skill'` 的事件中出现。
- 普攻不显示名人立绘，不显示技能名，只显示普通接战和伤害反馈。
- 若释放者没有立绘资源，前端使用圆形头像/姓名首字作为 fallback，但仍显示技能名。

建议的战报字段：

```js

  skillName: '血刃破阵',
  actorPortrait: {
    layers: {
      face: 'assets/art/famous-person/layers/fp-layer-v3-face-01.png',
      outfit: 'assets/art/famous-person/layers/fp-layer-v3-outfit-01.png',
      hair: 'assets/art/famous-person/layers/fp-layer-v3-hair-10.png'
    }
  },
  presentation: {
    cutIn: true,
    showSkillName: true,
    emphasis: 'skill'
  }
}
```

### 战报视图结构

第一版战报视图分为：

- 准备阶段：展示开战前被动效果、阵前效果、初始状态。
- 回合列表：按回合折叠或分页查看。
- 行动段：每个行动单位一段，展示行动者头像、名称、兵力、杀敌、关键行动。
- 简略/详细切换：简略只看结果和关键技能；详细展示每条事件。

颜色建议：

- 己方名称和关键数值用蓝色。
- 敌方名称和损失数值用红色。
- 恢复、增益用绿色。
- 减益和控制用橙色或紫色。

## 已确定：技能生成器落地接口

候选名人生成时，技能生成器输入必须随 `abilityKit` 一起写入存档，后续可用于追溯和复现。

当前落地输入：

```js
{
  quality: 'common' | 'good' | 'great' | 'legendary',
  archetype: 'commander' | 'vanguard' | 'strategist' | 'governor' | 'charmer' | 'scout',
  source: 'seek',
  seed: 'stable-seed',
  availableEffectPool: ['directDamage', 'secondHit'],
  generatorVersion: 'skill-gen-v1'
}
```

落地规则：

- `source` 第一版只允许真实生成来源为 `seek`。
- `seed` 来自名人候选来源，必须写入 `abilityKit.seed` 和 `abilityKit.generatorInput.seed`。
- 同一 `source + seed + quality + archetype` 在同一生成器版本下应可复现同一组技能模板。
- `availableEffectPool` 按 `domain` 过滤：战斗型使用第一批战斗效果，内政型使用内政效果，斥候型使用轻战斗和侦查/行军效果。
- 生成结果必须写入 `budgetChecks` 和 `budgetStatus`，第一版只接受 `withinLimit` 的普通生成结果。
- 主动战法、被动特质、内政技能和斥候特质都必须带 `description`，用于名人面板展示和后续战报/详情面板扩展。

当前生成输出补充字段：

```js
{
  source: 'seek',
  seed: 'player:timestamp:roll',
  generatorInput: {},
  availableEffectPool: [],
  budgetChecks: [
    { slot: 'activeSkill', id: 'skill_vanguard_double_cleave', cost: 112, limit: 145, withinLimit: true }
  ],
  budgetStatus: 'withinLimit'
}
```

## 已确定：技能面板展示 v2

名人面板第二版改为紧凑卡片。卡片只露出名人名称、称号、前两条技能名和六维蛛网图；技能冷却、触发条件、效果说明和未接入状态等详情不再常驻在卡片上，改为鼠标悬停或点击技能徽章时弹出。

展示规则：

- 卡片右侧展示六维蛛网图，固定为 `统帅 / 武力 / 智力 / 政治 / 魅力 / 速度`。
- `速度` 是武将六维之一，`工巧` 不属于武将/名人六维，不再生成、不再展示、不进入蛛网图。
- `activeSkill` 显示为 `主动战法：技能名`，冷却、释放条件和效果摘要放入悬停/点击详情。
- `passiveTrait` 显示为 `战斗被动：技能名`，触发时机和效果摘要放入悬停/点击详情。
- `civilPrimary` 显示为 `内政主技`，`civilSecondary` 显示为 `内政副技`。
- `scoutTrait` 显示为 `斥候特质`。
- `implementationStatus: storedOnly` 的能力必须在详情弹窗中展示 `暂未接入实际收益`，避免玩家误以为已经参与城市或侦查结算。
- 旧的 `skills` 文本摘要继续保留，用于兼容既有 UI 和测试。
- 卡片只展示前两条能力；后续如技能数量增加，再做技能详情弹窗或分页。
- 卡片不常驻展示来源、归附、招募等过程性文本，避免挤压六维和技能区域。
- 点击技能徽章会固定显示该技能详情；再次点击同一个技能徽章收起详情。
- 触屏设备固定详情后，点击名人面板内部空白区域只收起技能详情，不关闭名人面板。
- 鼠标悬停详情随 hover 展示，移出技能徽章后不应继续保持 hover 详情。

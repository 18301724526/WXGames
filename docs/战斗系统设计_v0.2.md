# 战斗系统设计 v0.2

日期：2026-05-29

## 文档定位

本文用于承接后续战斗系统重做，重点定义武将属性、成长、战斗公式、技能和战报结构。

当前代码中的 `speed-skill-cooldown-v1` 只视为兼容底座，不代表最终数值方案。战斗场景播放和前端消费战报的现状仍参考 `docs/回合制战斗场景与技能冷却设计_v0.2.md`。

本版按“一个问题一个问题确认”的方式推进。已经确认的内容写入“已确定”；未确认的内容只写入“待讨论”，不提前定案。

## 已确定：武将属性

武将采用六维属性：

| 属性 | 建议字段 | 职责 |
| --- | --- | --- |
| 统帅 | `command` | 影响部队防御、减伤和承伤稳定性。 |
| 武力 | `force` | 影响兵刃伤害。 |
| 智力 | `intelligence` | 影响谋略伤害。 |
| 魅力 | `charisma` | 后续影响寻访、探索、侦查事件等非直接战斗内容。 |
| 政治 | `politics` | 后续影响城建、治理、城市发展等内容。 |
| 速度 | `speed` | 影响出手先后顺序。 |

第一版战斗只使用：

- 统帅
- 武力
- 智力
- 速度

魅力和政治保留为完整武将属性，但暂不进入第一版战斗公式。

历史字段兼容：

- 旧 `strategy` 语义后续应迁移为 `intelligence`。
- 当前存档和代码仍可能存在 `strategy`，正式实装前需要补迁移和兼容读取。

## 已确定：属性值域

- 初始属性范围为 `1-100`。
- 当前面板属性不设置硬上限。
- 初始属性代表武将出生面板，不代表后续成长天花板。

属性超过 `100` 后的战斗收益曲线见“属性有效值”。第一版所有战斗属性先使用同一条递减收益曲线。

## 已确定：武将成长

武将属性由三部分组成：

```text
当前属性 = 初始属性 + 自动成长属性 + 玩家分配属性点
```

成长规则：

- 每升 `10` 级，固定获得 `10` 点自由属性点。
- 自由属性点由玩家分配。
- 自由属性点获取不受品质影响。
- 品质只影响自动成长数值。
- “自由属性点”和“技能点”需要区分；当前确认的是属性点，技能点暂不设计。

品质职责：

- 品质影响初始属性区间。
- 品质影响每个成长周期获得的自动成长总量。
- 品质不影响玩家每 `10` 级获得的自由属性点数量。

自动成长分配方向见“自动成长分配”。第一版按武将主定位决定成长倾向。

## 已确定：品质分层

武将品质先采用四档，不做复杂稀有度包装：

| 品质 | 字段建议 | 定位 | 每 10 级自动成长总点数 |
| --- | --- | --- | --- |
| 一般 | `common` | 基础可用，适合早期过渡。 | 4 |
| 良才 | `good` | 明显好用，有培养价值。 | 6 |
| 英杰 | `great` | 核心武将，主属性成长突出。 | 8 |
| 传奇 | `legendary` | 稀有强者，长期培养价值最高。 | 10 |

说明：

- 名人生成时直接生成品质，品质对玩家可见。
- 每 10 级的自由属性点仍固定为 `10`，不受品质影响。
- 品质只影响自动成长总点数。
- 自动成长点数后续会按武将定位分配到六维属性上。
- 品质影响初始属性区间、自动成长总点数和技能生成预算。
- 当前数值先保持温和，避免品质差距在早期直接压死培养空间。

寻访品质概率：

| 品质 | 概率 |
| --- | ---: |
| 一般 `common` | `32.4%` |
| 良才 `good` | `26%` |
| 英杰 `great` | `36%` |
| 传奇 `legendary` | `5.6%` |

寻访保底：

- 寻访暂时不设置冷却。
- 寻访暂时不消耗资源或抽卡道具。
- 寻访是当前阶段临时获得名人的入口，不接内购充值。
- 寻访保底只统计寻访来源，不跨事件投奔、战后归附或其他后续来源。
- 连续 `29` 次寻访未生成 `legendary` 时，第 `30` 次寻访强制生成 `legendary`。
- 生成 `legendary` 后，寻访保底计数清零。
- 第一版只开放寻访生成名人。
- 事件投奔、战后归附和世界探索获得名人暂时关闭；项目中保留历史数据兼容和展示文本，但不再生成新候选。
- 事件投奔、战后归附和世界探索获得名人的规则后续单独设计，不共享寻访保底。

## 已确定：等级与经验

武将经验主要来自战斗。

经验来源原则：

- 胜利和失败都会获得经验。
- 经验不按简单胜负二元发放，而按战斗实际参与和战果计算。
- 剿灭敌军越多，经验越多。
- 只剿灭极少量敌军时，经验应非常少。
- 战损也可以提供少量经验，用于表达苦战经历，但不能高于有效杀伤带来的经验。

第一版经验建议公式：

```text
战斗经验 = 敌军损失 × 1.0 + 己方损失 × 0.25 + 胜利奖励
```

其中：

- `敌军损失` 按实际消灭士兵数量计算。
- `己方损失` 按本方实际阵亡士兵数量计算。
- 胜利奖励用于补足成功作战的正反馈，但不能替代杀伤贡献。

胜利奖励建议：

```text
胜利奖励 = max(20, 敌军初始兵力 × 0.05)
```

例子：

- 只消灭 `1` 个敌军，失败撤退：约 `1` 点经验，几乎没有成长。
- 消灭 `300` 敌军，自损 `100`，失败撤退：约 `325` 点经验。
- 消灭 `500` 敌军并胜利，自损 `120`：约 `550` 点经验。

## 已确定：升级曲线

武将等级不设置硬上限。升级曲线采用平滑递增，避免长线游戏后期突然断崖式变慢。

每级所需经验：

```text
升到下一级所需经验 = roundTo10(100 + 35 × 当前等级 + 6 × 当前等级 ^ 1.5)
```

说明：

- `roundTo10` 表示四舍五入到最接近的 `10`。
- 当前等级从 `1` 开始。
- 经验允许累计，满足多级需求时可以连续升级。
- 曲线由线性项和缓慢幂次项组成，前期有成长反馈，后期仍能长期推进。

参考值：

| 当前等级 | 升到下一级所需经验 |
| --- | --- |
| 1 | 140 |
| 10 | 640 |
| 20 | 1340 |
| 50 | 3970 |
| 100 | 9600 |
| 200 | 24070 |
| 500 | 84680 |

每达到 `10` 的倍数等级时，触发一次成长节点：

- 获得 `10` 点自由属性点。
- 根据武将品质获得自动成长点数。
- 自动成长点数按武将定位权重分配。

## 已确定：速度

速度暂时只决定出手先后顺序。

第一版规则：

- 每轮开始按速度排序。
- 速度高者先行动。
- 速度低者后行动。
- 同速时默认己方先行动。
- 速度暂不影响行动频率。
- 速度暂不产生连动、追击、额外回合或行动条。

## 已确定：自动成长分配

自动成长点数按武将定位分配。第一版先用主定位，不做复杂多定位混合。

分配算法：

```text
本次自动成长点数 = 品质成长总点数
各属性获得点数 = 本次自动成长点数 × 定位权重
```

实现时先取整数部分，剩余点数按小数部分从高到低补足；若小数相同，优先补主属性。

定位权重：

| 定位 | 字段建议 | 统帅 | 武力 | 智力 | 魅力 | 政治 | 速度 |
| --- | --- | ---: | ---: | ---: | ---: | ---: | ---: |
| 统帅 | `commander` | 45% | 15% | 10% | 10% | 10% | 10% |
| 猛将 | `vanguard` | 15% | 45% | 5% | 10% | 5% | 20% |
| 谋略 | `strategist` | 10% | 5% | 50% | 10% | 15% | 10% |
| 内政 | `governor` | 5% | 5% | 15% | 20% | 50% | 5% |
| 魅力 | `charmer` | 10% | 5% | 15% | 45% | 20% | 5% |
| 斥候 | `scout` | 10% | 10% | 20% | 15% | 5% | 40% |

例子：

- `英杰` 谋略型每 `10` 级自动成长 `8` 点，主要落在智力，其次政治、统帅/魅力/速度。
- `传奇` 猛将型每 `10` 级自动成长 `10` 点，主要落在武力，其次速度和统帅。

## 已确定：伤害公式方向

第一版战斗只做确定性自动战斗，不做命中、闪避、暴击和行动条。

### 属性有效值

面板属性可以无限成长，但战斗收益使用有效属性，超过 `100` 后进入递减收益：

```text
有效属性 = 属性 <= 100
  ? 属性
  : 100 + 2 × (属性 - 100) ^ 0.75
```

说明：

- `100` 以内保持直观线性。
- `100` 以后继续成长，但收益逐步变缓。
- 该公式同时适用于统帅、武力、智力、速度等战斗属性。

### 通用伤害框架

士兵数量既是承伤池，也影响输出规模。兵力越少，后续伤害越低。

```text
兵力系数 = (当前兵力 / 100) ^ 0.65
攻击系数 = 0.75 + 有效攻击属性 / 120
防御减伤 = clamp(有效防御属性 / (有效防御属性 + 180), 0.05, 0.60)
最终伤害 = round(35 × 兵力系数 × 攻击系数 × (1 - 防御减伤) × 技能倍率)
```

伤害下限：

```text
最终伤害 = max(1, min(目标当前兵力, 最终伤害))
```

### 伤害类型

| 类型 | 使用属性 | 防御属性 | 用途 |
| --- | --- | --- | --- |
| 兵刃伤害 | 武力 | 目标统帅 | 普通攻击、突击、斩击等。 |
| 谋略伤害 | 智力 | 目标智力 | 火攻、计策、扰乱等。 |

第一版普通攻击默认是兵刃伤害，技能根据模板指定 `blade` 或 `strategy`。

说明：

- 统帅主要防御兵刃伤害。
- 智力既影响谋略伤害，也防御谋略伤害。
- 高智力武将使用谋略伤害攻击低智力目标时应有明显优势。

技能倍率建议：

| 行为 | 倍率范围 |
| --- | --- |
| 普通攻击 | `1.00` |
| 小技能 | `1.15-1.30` |
| 标准主动技能 | `1.35-1.60` |
| 高风险强技能 | `1.70-2.00` |

本阶段先不加入随机伤害浮动。后续如需增加战报变化，可以加入稳定的伪随机浮动，但必须由后端生成并写入战报。

## 已确定：士气字段

士气先保留为战斗字段，但第一版不实际影响伤害公式。

```js
{
  morale: 100,
  moraleEffectEnabled: false
}
```

说明：

- 战报可以展示士气。
- 后端战斗单位保留士气字段。
- 第一版伤害公式不乘士气系数。
- 暂不设计士气来源、恢复、崩溃、溃逃、补给和连续作战影响。
- 后续如果做真实大地图、补给线、远征疲劳、围城或地形，再统一接入士气系统。

## 已确定：技能系统 v1

每名武将第一版最多拥有：

- `1` 个主动战法。
- `1` 个被动特质。

主动战法：

- 有冷却。
- 冷却按自身行动次数推进。
- 开局第一次行动可以释放。
- 冷却结束后优先释放主动战法，否则执行普通攻击。
- 第一版不做概率触发，避免战报不可解释。

被动特质：

- 只在准备阶段或明确条件下触发。
- 第一版优先用于属性修正、开战增益或简单状态。
- 不做复杂连锁、不打断行动顺序。

技能类型：

| 类型 | 字段 | 主要属性 | 作用 |
| --- | --- | --- | --- |
| 兵刃 | `blade` | 武力 | 造成兵刃伤害。 |
| 谋略 | `strategy` | 智力 | 造成谋略伤害。 |
| 守御 | `guard` | 统帅 | 提供减伤、护盾、防御类效果。 |
| 支援 | `support` | 统帅或智力 | 恢复、增益、削弱等简单效果。 |

第一版暂不做：

- 暴击。
- 闪避。
- 命中率。
- 连击。
- 控制跳过回合。
- 追击和额外行动。

## 已确定：技能生成 v1

技能系统采用“模块化肉鸽生成 + 确定性战斗结算”。

核心原则：

- 普通武将不预先维护固定技能表。
- 普通技能由效果原子、组合规则、命名规则和强度预算生成。
- 技能一旦生成，就写入该武将存档，后续战斗按该技能确定结算。
- 战斗过程中不重新随机技能效果，不做概率触发。
- 后期如果出现剧情人物、特殊历史人物或活动人物，可以通过 `uniqueSkillId` 单独实现独特技能。
- 独特技能不进入普通随机池，避免破坏普通技能生成预算。

### 技能生成结构

普通技能生成分三层：

```text
效果原子
→ 组合规则
→ 命名规则
→ 写入武将存档
→ 战斗中确定性结算
```

主动战法结构：

```js
{
  id: 'skill_generated_xxx',
  name: '血刃破阵',
  kind: 'active',
  category: 'blade',
  damageType: 'blade',
  multiplier: 1.45,
  cooldown: 3,
  effects: [
    { key: 'directDamage', value: 1.45 },
    { key: 'lifesteal', value: 0.12 }
  ],
  budget: {
    tier: 'great',
    cost: 140,
    limit: 145
  },
  generatorVersion: 'skill-gen-v1'
}
```

被动特质结构：

```js
{
  id: 'trait_generated_xxx',
  name: '临阵坚守',
  kind: 'passive',
  trigger: 'preBattle',
  effects: [
    { key: 'attributeBonus', attribute: 'command', value: 8 }
  ],
  generatorVersion: 'skill-gen-v1'
}
```

### 效果原子

第一批效果原子只保留能稳定解释、稳定结算的类型：

| key | 用途 | 第一版结算方式 |
| --- | --- | --- |
| `directDamage` | 直接伤害 | 按技能倍率造成兵刃或谋略伤害。 |
| `secondHit` | 二段伤害 | 固定追加一次低倍率伤害，不做概率连击。 |
| `lifesteal` | 吸血 | 按实际造成伤害的一定比例恢复己方兵力。 |
| `heal` | 治疗 | 按最大兵力或属性计算恢复量。 |
| `shield` | 护盾 | 获得固定数值护盾或临时减伤。 |
| `armorBreak` | 破甲 | 在限定回合内降低目标有效统帅或防御减伤。 |
| `burn` | 灼烧 | 写入持续状态，每回合造成谋略持续伤害。 |
| `poison` | 中毒 | 写入持续状态，每回合造成较低伤害或降低恢复。 |
| `firstStrike` | 伏击 | 首次行动附加确定伤害，不做概率伏击。 |
| `attributeBonus` | 属性修正 | 战前或条件满足时修正指定属性。 |
| `lowSoldierGuard` | 低兵力守势 | 己方兵力低于阈值时获得减伤。 |

旧效果口径迁移：

| 旧效果 | 新口径 |
| --- | --- |
| `combo` 概率连击 | 改为 `secondHit` 固定二段伤害。 |
| `counter` 概率反击 | 第一版暂不进入普通池；后续可改成确定的“受击后反击标记”。 |
| `ambush` 概率伏击 | 改为 `firstStrike` 首次行动附加伤害。 |
| `morale` 士气增强 | 士气系统未启用前，先改成 `attributeBonus` 或不进入池。 |

### 组合规则

主动战法：

- 必须有 `1` 个主效果。
- 可以有 `0-1` 个副效果。
- 必须有冷却。
- 必须指定伤害类型或功能类型。
- 不能同时组合多个持续伤害。
- 不能同时组合多个恢复/防御副效果。
- 第一版不生成硬控制技能。

被动特质：

- 必须有明确触发时机。
- 第一版触发时机只允许：
  - `preBattle`：战前生效。
  - `firstRound`：第一回合生效。
  - `lowSoldier`：己方兵力低于阈值时生效。
- 被动特质不插入额外行动，不打断行动顺序。
- 被动特质必须写入准备阶段或状态变化战报。

### 强度预算

技能生成必须受强度预算约束。品质只决定预算上限，不直接决定固定技能。

建议预算：

| 品质 | 字段 | 主动战法预算 | 被动特质预算 |
| --- | --- | ---: | ---: |
| 一般 | `common` | 100 | 40 |
| 良才 | `good` | 120 | 50 |
| 英杰 | `great` | 145 | 65 |
| 传奇 | `legendary` | 170 | 80 |

效果预算建议：

| 效果 | 预算消耗 |
| --- | ---: |
| `1.20` 倍直接伤害 | 55 |
| `1.35` 倍直接伤害 | 70 |
| `1.50` 倍直接伤害 | 90 |
| `1.75` 倍直接伤害 | 125 |
| `secondHit` 追加 `0.30` 倍伤害 | 30 |
| `lifesteal` 恢复伤害 `10%-12%` | 25-35 |
| `shield` 最大兵力 `8%-12%` | 25-40 |
| `heal` 最大兵力 `8%-12%` | 30-45 |
| `armorBreak` 降低防御 `1-2` 回合 | 30-45 |
| `burn/poison` 持续 `2` 回合 | 35-50 |
| `firstStrike` 首次行动附加伤害 | 25-40 |
| `attributeBonus` 战前属性 `+5` 到 `+10` | 25-50 |

限制项可以返还预算：

| 限制 | 预算返还 |
| --- | ---: |
| 冷却从 `3` 增加到 `4` | +15 |
| 只在首回合生效 | +20 |
| 只在低兵力时生效 | +15 |
| 只对兵刃或谋略一种伤害生效 | +10 |

预算规则只负责生成时校验，不在战斗中展示。

### 定位权重

生成器按武将定位提高不同效果权重：

| 定位 | 主效果倾向 | 副效果倾向 |
| --- | --- | --- |
| 统帅 | `shield`、`lowSoldierGuard`、`attributeBonus(command)` | `armorBreak`、`heal` |
| 猛将 | `directDamage(blade)`、`secondHit`、`firstStrike` | `lifesteal`、`armorBreak` |
| 谋略 | `directDamage(strategy)`、`burn`、`poison` | `armorBreak`、`attributeBonus(intelligence)` |
| 内政 | 弱战斗主动或无主动 | `attributeBonus(politics)`，后续接城市效果 |
| 魅力 | `heal`、`attributeBonus(charisma)` | 士气系统启用后接士气效果 |
| 斥候 | `firstStrike`、`directDamage(strategy)` | `secondHit`、速度修正 |

第一版如果某个定位主要服务非战斗内容，也允许生成较弱战斗技能，但不要生成完全无效技能。

### 命名规则

技能名必须来自实际效果，不允许出现与效果脱节的名字。

命名输入：

- 主效果。
- 副效果。
- 伤害类型。
- 武将定位。
- 来源类型。

示例：

| 效果组合 | 合理名称 |
| --- | --- |
| 兵刃直接伤害 + 吸血 | `血刃破阵`、`饮血突击` |
| 兵刃直接伤害 + 二段伤害 | `破锋连击`、`裂阵双斩` |
| 谋略伤害 + 灼烧 | `火计焚阵`、`烈策攻心` |
| 谋略伤害 + 破甲 | `摧阵奇谋`、`裂甲毒计` |
| 护盾 + 低兵力减伤 | `临阵坚守`、`铁壁护军` |
| 治疗 + 统帅属性 | `振军复阵`、`整军回春` |

### 独特技能扩展

后期如需制作特殊人物，可以单独实现独特技能：

```js
{
  uniqueSkillId: 'unique_lu_xiao_blood_spear',
  name: '血枪先登',
  kind: 'active',
  implementation: 'unique',
  generatorVersion: null
}
```

独特技能规则：

- 不进入普通技能随机池。
- 不受普通命名器约束。
- 可以有特殊战报、特殊视觉表现和专属效果。
- 必须单独写测试，避免影响普通生成器。

## 已确定：技能状态系统 v1

状态用于承载护盾、破甲、灼烧、中毒、易伤、低兵力守势、持续恢复等跨行动效果。状态必须真实参与结算，不能只作为战报文字。

### 状态结构

建议第一版状态结构：

```js
{
  key: 'burn',
  sourceSkillId: 'skill_generated_xxx',
  sourceSide: 'attacker',
  targetSide: 'defender',
  turnsRemaining: 2,
  stacks: 2,
  maxStacks: 3,
  value: 0.08,
  stackPolicy: 'stack',
  appliedAtRound: 1
}
```

字段说明：

- `key`：状态类型。
- `sourceSkillId`：来源技能。
- `sourceSide`：释放者。
- `targetSide`：承受者。
- `turnsRemaining`：剩余持续次数，按目标自己的行动前扣减。
- `stacks`：当前层数。
- `maxStacks`：最大层数。
- `value`：每层或当前状态的基础数值。
- `stackPolicy`：叠加机制。
- `appliedAtRound`：施加回合，用于战报和调试。

### 结算时机

状态在每个行动单位自己的行动前结算。

顺序：

```text
行动单位回合开始
→ 优先处理自身清状态、净化、免疫类状态
→ 结算自身剩余状态
→ 按状态规则扣减持续次数
→ 移除过期状态
→ 如果该单位兵力归零，则清空该单位身上所有状态
→ 该单位正式行动
```

说明：

- 这里的“回合开始”指每个单位行动前的小回合，不是双方完整大回合。
- 状态挂在谁身上，就在谁行动前结算和扣回合。
- 自身拥有清状态、净化、免疫类状态时，必须优先生效，再处理负面状态。

### 叠加规则

同类状态允许叠加，但必须有明确上限。

第一版建议分类型处理：

| 状态类型 | 叠加方式 | 上限建议 |
| --- | --- | --- |
| 持续伤害 | 增加层数，每层单独贡献伤害权重。 | `3` 层 |
| 易伤/破甲 | 增加层数或刷新强度，但总效果封顶。 | 总减防不超过 `30%` |
| 护盾 | 数值累加。 | 不超过最大兵力 `30%` |
| 持续恢复 | 增加层数或刷新持续时间。 | `3` 层 |
| 属性修正 | 同属性同方向可叠加。 | 单属性临时修正不超过有效属性 `30%` |

当新状态达到上限后：

- 如果是层数型状态，超过上限的新增层数无效，但可以刷新持续时间。
- 如果是数值型状态，超过上限的部分截断。
- 如果新状态强度更高，可以刷新 `value`，但总效果仍受上限约束。

### 持续时间

状态持续时间按目标自己的行动前扣减。

例子：

```text
陆骁给敌人挂灼烧 2 回合
敌人第 1 次行动前：灼烧生效，剩 1
敌人第 2 次行动前：灼烧生效，剩 0，移除
```

速度影响：

- 速度快的单位会更快消耗自己身上的状态。
- 速度慢的单位不会因为其他单位行动而额外扣状态。

### 持续伤害

灼烧、中毒等持续伤害每次触发时动态计算，不在施加时锁定伤害。

每次触发时读取：

- 释放者当前相关属性。
- 受害者当前相关防御属性。
- 双方当前状态修正，例如易伤、减伤、增智、降智。
- 当前层数和每层权重。

伤害属性：

- 灼烧、中毒默认视为谋略伤害。
- 谋略持续伤害使用释放者智力攻击目标智力。
- 兵刃持续伤害如后续存在，则使用释放者武力攻击目标统帅。

例子：

```text
第 1 次触发：释放者智力高、目标智力低，造成 28 伤害
第 2 次触发：目标获得智力提升，伤害降为 20
第 3 次触发：目标又被挂易伤，伤害提升
```

### 死亡与状态清空

任意单位兵力归零后：

- 该单位身上的所有状态立即清空。
- 该单位施加在别人身上的状态也清空。
- 后续不再由该单位的状态产生持续伤害、破甲、易伤或增益。

说明：

- 这样可以避免败退单位继续通过灼烧、中毒等状态影响战局。
- 战报需要记录关键状态清空，例如 `[陆骁] 败退，相关状态消散`。

## 已确定：战斗流程 v1

战斗是后端确定性自动战斗，玩家只做战前配置。

```text
准备阶段
→ 应用被动特质和准备效果
→ 按速度排序
→ 第 1 回合
  → 速度高者行动
  → 速度低者行动
→ 第 2 回合
...
→ 一方兵力为 0 或达到最大回合
→ 战后结算
```

最大回合暂定 `20`。

胜负规则：

- 敌方兵力为 `0`：胜利。
- 己方兵力为 `0`：失败。
- 达到最大回合仍未结束：己方剩余兵力大于等于敌方剩余兵力则胜利，否则失败。

同速规则：

- 同速时默认己方先行动。

## 已确定：战后结算

胜利：

- 地点进入占领流程。
- 敌方剩余兵力视为溃散。
- 第一版暂不触发战后归附候选。
- 发放胜利经验奖励。

失败：

- 地点保持未占领。
- 记录上次战斗结果。
- 仍按实际战斗表现获得经验。
- 暂不做武将受伤、俘虏、士气崩溃或长期负面状态。

伤亡：

- 己方伤亡按战斗中实际损失扣除。
- 敌方损失用于经验计算和战报展示。
- 出征士兵返回数量等于出征兵力减去战斗损失。

经验：

- 使用“等级与经验”章节中的公式。
- 胜负都会获得经验。
- 战后归附暂时关闭，后续如恢复，只允许在胜利时触发。

## 已确定：战报表现方向

战报参考“三国志战略版”这类结构化详细战报的信息密度，但不照搬视觉样式。

目标：

- 玩家能看懂每一回合谁先行动。
- 玩家能看懂普攻、技能、恢复、状态变化分别发生了什么。
- 玩家能看懂伤害类型、损失兵力和剩余兵力。
- 战报可被前端播放，也可被文本面板直接展示。

### 战报视图结构

第一版战报视图分为：

- 准备阶段：展示开战前被动效果、阵前效果、初始状态。
- 回合列表：按回合折叠或分页查看。
- 行动段：每个行动单位一段，展示行动者头像、名称、兵力、杀敌、关键行动。
- 简略/详细切换：简略只看结果和关键技能；详细展示每条事件。

颜色建议：

- 己方名称和关键数值用蓝色。
- 敌方名称和损失数值用红色。
- 恢复、增益用绿色。
- 减益和控制用橙色或紫色。

### 战报事件结构

后端战报需要输出结构化事件，前端只负责渲染：

```js
{
  round: 1,
  actor: 'attacker',
  actorName: '陆骁',
  target: 'defender',
  targetName: '林地部落',
  actionType: 'skill',
  skillName: '火攻',
  damageType: 'strategy',
  damage: 132,
  soldiersBefore: { attacker: 500, defender: 394 },
  soldiersAfter: { attacker: 500, defender: 262 },
  lines: [
    '[陆骁] 开始行动',
    '[陆骁] 发动战法 [火攻]',
    '[林地部落] 受到谋略伤害 132（262）'
  ]
}
```

文本行示例：

```text
【第一回合】
[陆骁] 开始行动
[陆骁] 对 [林地部落] 发动普通攻击
[林地部落] 损失士兵 106（394）

[林地部落] 开始行动
[林地部落] 发动战法 [伏击]
[陆骁] 受到兵刃伤害 88（412）
```

括号内数字表示该次事件后的剩余兵力。

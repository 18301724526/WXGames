# 武将经验等级第一批落地任务 v0.1

日期：2026-05-30

目标版本：`0.1.164`

## 目标

把战斗结算中已经生成的经验真正写回出征主将，让自动战斗不只是一次性战报，而是能推动名人长期成长。

本批只做成长闭环第一步：

- 已加入名人拥有 `level / experience / totalExperience / freeAttributePoints / nextLevelExperience`。
- 征服战结算后，领队名人获得本场战报 `experience.total`。
- 经验可连续升级。
- 达到 `10` 的倍数等级时，获得 `10` 点自由属性点。
- 名人面板展示等级、当前经验、下级所需经验和待分配属性点。

本批不做：

- 手动分配属性点 UI。
- 品质自动成长属性写入。
- 技能点、技能升级或技能替换。
- 候选名人成长显示。

## 落地方案

### 后端

`FamousPersonService` 新增成长规则：

- `getLevelUpExperience(level)`：使用设计文档公式 `roundTo10(100 + 35 × 当前等级 + 6 × 当前等级 ^ 1.5)`。
- `normalizeProgression(raw)`：旧存档已加入名人自动补齐等级、经验和可分配属性点字段。
- `grantBattleExperience(gameState, leaderId, experienceSummary, now)`：按战报经验写回指定名人，并返回成长摘要。

`TerritoryService.resolveMission()` 在征服战写入 `territory.lastBattle` 后调用成长接口：

- `territory.lastBattle.leaderGrowth` 保存成长摘要。
- `territory.lastBattle.report.leaderGrowth` 同步保存成长摘要，方便前端或后续战报面板展示。
- `leader = unavailable` 或找不到名人时返回 `null`，不影响普通战斗结算。

### 前端

`UIStatePresenter.buildFamousPersonCard()` 对已加入名人输出：

- `growthText`：例如 `等级 2 · 经验 35/190`。
- `pointText`：例如 `可分配属性点 4`。
- `level / experience / nextLevelExperience / freeAttributePoints`。

候选名人仍不输出成长信息。

`CanvasGameRenderer.renderFamousPersonItem()` 在名人称号下方增加成长行；若有待分配属性点，成长行使用更醒目的颜色。

## 验证点

- 旧已加入名人进入 `normalizeState` 后自动变为 `等级 1 / 经验 0 / 下级 140`。
- 候选名人仍没有 `level` 字段。
- 领队征服战后，战报经验写回名人。
- 经验超过当前等级需求时会升级并结转剩余经验。
- 从 `9` 级升到 `10` 级时获得 `10` 点自由属性点。
- 名人面板显示成长信息，候选卡片不显示成长信息。

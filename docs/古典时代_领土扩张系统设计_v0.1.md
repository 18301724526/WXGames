# 古典时代领土扩张系统设计 v0.1

> 状态：本版实施范围文档
> 日期：2026-05-17

## 目标

古典时代不再只是数值进阶，而是让玩家从单一城邦扩张为一方势力。第一版用节点式疆域承载战争结果：玩家侦察土地、派士兵征服、占领后命名城市，并在拥有第二块领土时为势力命名。

```text
边境时代完成防御准备
→ 进入古典时代
→ 打开疆域页
→ 侦察周边土地
→ 派士兵占领
→ 为新城市命名
→ 拥有第二块领土后为势力命名
```

## 设计原则

- 后端驱动：领土状态、侦察、征服、命名、奖励与加成都由后端计算和持久化。
- 士兵统一：不拆守卫/战士，派出的士兵只是暂时进入任务。
- 节点疆域：第一版不做自由地图移动，只做有限节点与状态变化。
- 轻经营：占领城市只提供全局加成，不做每城独立人口、建筑、仓库。
- 命名强反馈：每块已占领领土都可以有城市名，第二块领土触发势力命名。

## 时代定义

| 字段 | 设计 |
|---|---|
| 时代名 | 古典时代 |
| 进入条件 | 食物 1400、木材 900、知识 520、士兵 6、已占领至少 1 块领土 |
| 解锁界面 | 疆域 |
| 核心玩法 | 侦察、征服、占领、命名 |
| 第一版新增建筑 | 无 |

> 古典时代第一版先把战争结果落到“领土”上，不引入金属、装备、多兵种或每城独立经营。

## 领土节点

| id | 自然名 | 类型 | 初始状态 | 防御 | 占领效果 |
|---|---|---|---|---:|---|
| capital | 起源之地 | capital | occupied | 0 | 无 |
| river_plain | 河湾平原 | plains | scoutable | 4 | 食物产出 +5% |
| north_forest | 北部林地 | forest | locked | 5 | 木材产出 +8% |
| hill_outpost | 丘陵哨站 | hills | locked | 6 | 边境防御 +2 |
| old_ruins | 旧日遗迹 | ruins | locked | 7 | 知识产出 +6% |

解锁规则：

- 首都默认已占领。
- 古典时代开放后，河湾平原可侦察。
- 占领河湾平原后，北部林地和丘陵哨站可侦察。
- 占领北部林地或丘陵哨站后，旧日遗迹可侦察。

## 行动规则

### 侦察

- 仅对 `scoutable` 领土开放。
- 不消耗资源。
- 成功后状态变为 `scouted`，展示防御、派兵建议和占领效果。

### 征服

- 仅对 `scouted` 领土开放。
- 玩家选择派兵数量，第一版前端提供 `推荐兵力` 一键派出。
- 派出的士兵进入 `warMissions`，暂时不计入可用士兵。
- 任务完成后需要玩家点击“完成占领”领取结果。

第一版结算保持确定性：

```text
出征战力 = 派出士兵数 × 1
如果 出征战力 >= 领土防御：
  占领成功，领土状态变 occupied
  伤亡 = floor(领土防御 / 3)，至少 0，最多 派出士兵 - 1
否则：
  占领失败，伤亡 = ceil(派出士兵数 / 2)
```

成功后，未伤亡士兵返回；失败后，未伤亡士兵也返回。第一版不做士兵长期驻扎。

## 命名规则

- 首都可改名。
- 每块 `occupied` 领土都可设置 `cityName`。
- 领土刚占领且未命名时，前端弹出城市命名弹窗。
- 当已占领领土数达到 2，且 `polity.name` 为空时，前端弹出势力命名弹窗。
- 后端校验命名长度：1-12 个字符，去除首尾空白。

推荐提示文案：

```text
你的旗帜已经越过最初的边界。
从一座城邦到多座城市，你已经成为一方势力。
为你的势力取一个名字吧。
```

## 前端展示

- 新增“疆域”标签页，古典时代前可显示未开放提示。
- 顶部展示势力名、已占领领土数量、可用士兵。
- 领土以卡片或节点形式展示，卡片使用领土美术 cutout。
- 卡片操作：
  - `scoutable`：侦察
  - `scouted`：派兵占领
  - `contested`：显示倒计时
  - `missionReady`：完成占领
  - `occupied`：显示城市名和改名入口

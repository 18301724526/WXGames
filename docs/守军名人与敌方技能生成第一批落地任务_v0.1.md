# 守军名人与敌方技能生成第一批落地任务 v0.1

日期：2026-05-30

目标版本：`0.1.168`

## 目标

本批把敌方技能挂到“守军名人/敌将”身上，而不是直接挂在守军模板上。战斗从本批开始更接近“我方主将 vs 敌方主将”：

- 敌对据点拥有稳定守军名人。
- 守军名人拥有名字、称号、品质、等级、六维、立绘和 `abilityKit`。
- 战斗结算读取守军名人的速度、属性和主动战法。
- 敌方释放技能时同样触发立绘 cut-in、技能名展示、冷却和状态结算。

## 范围

### 后端

- 新增 `DefenderLeaderService`，负责守军名人生成、归一化和旧据点补齐。
- 敌对据点发现或归一化时生成 `defenderLeader`。
- 中立无主地点不生成守军名人，仍走建立据点流程。
- `BattleService` 读取 `territory.defenderLeader` 作为敌方战斗载体。
- 敌方战报输出 `leaderId / leaderName / leaderTitle / appearance / abilityKit / skill`。
- 技能生成器普通战斗池补入已真实结算的 `armorBreak / burn / poison`。

### 前端

- 世界地点详情显示守将和敌方战法。
- 战斗场景标题、敌方称号和敌方技能 cut-in 使用守军名人信息。
- 敌方立绘复用名人系统 v3 三层立绘。

## 不做

- 不做守军名人俘虏、招降、归附或掉落。
- 不把守军名人加入玩家名人列表。
- 不新增独立敌方立绘素材池，第一批复用现有名人立绘层。
- 不改变玩家侧技能生成、等级成长和属性点规则。

## 数据规则

守军名人挂在据点字段：

```js
territory.defenderLeader = {
  id,
  name,
  title,
  source: { type: 'defender', territoryId, territoryName, owner, seed },
  archetype,
  abilityArchetype,
  quality,
  qualityLabel,
  level,
  attributes,
  abilityKit,
  skills,
  appearance,
  status: { assigned: 'defender' }
}
```

生成强度：

- `threat <= 2`：一般。
- `threat <= 4`：良才。
- `threat <= 6`：英杰。
- `threat > 6`：传奇。
- 等级按威胁、规模和防御推算。
- 属性按据点 owner 使用不同基础模板，并随威胁略微提高。
- seed 使用据点 id、owner、threat 和 defense，保证同一据点稳定。

## 验证点

- 敌对据点生成后拥有 `defenderLeader`。
- 中立据点没有 `defenderLeader`。
- 旧存档敌对据点归一化后补齐守军名人。
- 敌方战报中出现敌将名字、称号、技能和立绘。
- 敌方释放技能时 `presentation.cutIn === true`。
- 敌方技能能真实施加破甲、灼烧、中毒等状态。
- 世界地点详情能看到“守将”和“敌方战法”。

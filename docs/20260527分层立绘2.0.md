## 分层定义

当前层顺序：

```text
backHair -> body -> innerwear -> sideHair -> frontHair -> bangs -> outfit
```

层含义：

- `backHair`：后发、头后方发量和发髻。
- `body`：脸、耳朵、脖子、肩膀、裸身底。
- `innerwear`：内衬，负责遮住盔甲大领口露出的胸口。
- `sideHair`：鬓角、脸侧前景头发。
- `frontHair`：前发主体，不是刘海。
- `bangs`：刘海，只负责额前碎发。
- `outfit`：外层衣服/盔甲。

## 代码边界

允许代码做的事：

- 从 PNG sheet 拆分单层。
- 去掉绿色/白色背景。
- 按第一个/最后一个不透明像素裁掉上下左右多余空白。
- 写出透明 PNG。
- 写出 `fp-layer-v2-manifest.json`。
- 在实验台和游戏里按 manifest + transform 拼层。

不允许代码做的事：

- 用 JS/Canvas/矢量方式画正式立绘资源。
- 恢复 `scripts/generate-famous-portrait-v2-assets.js`。
- 恢复旧的 `frontCutY`、`backCutY`、split/crop 逻辑。

当前资源提取脚本：

- `scripts/extract-famous-portrait-v2-sheet.js`

当前本地预览脚本：

- `scripts/preview-famous-portrait-v2-composite.js`

## 实验台和游戏同步点

实验台：

- `frontend/tools/famous-portrait-lab.html`
- `frontend/tools/famous-portrait-lab.js`

游戏共用布局：

- `frontend/js/config/FamousPortraitLayout.js`

后端名人生成人物外观：

- `backend/services/FamousPersonService.js`

关键点：实验台和游戏必须使用同一套坐标模型。实验台从 `fp-layer-v2-manifest.json` 读 base 坐标，游戏从 `FamousPortraitLayout.js` 读 base 坐标，所以更新资源后必须把 manifest 里的 base 同步到 `FamousPortraitLayout.js`。

当前 `APPEARANCE_VERSION` 是：

```text
famous-portrait-v1.1
```

这会让旧存档里的 v1.0/v0.9/v0.2/unversioned 名人在 normalize 时刷新到 `art01` 新资源。

1. 回家后打开公网实验台。
2. 在实验台里手动微调 `global`、层级顺序、每层 `scale/x/y/visible`。

4. 把导出 JSON 的 `global`、`order`、`layers` transform 同步到：
   - `frontend/tools/famous-portrait-lab.js`
   - `frontend/js/config/FamousPortraitLayout.js`
5. 如果要让已有人物强制刷新，继续提升 `backend/services/FamousPersonService.js` 里的 `APPEARANCE_VERSION`。
6. 跑测试，提交并推双远端。

## 当前默认参数说明

现在默认参数不是最终美术定稿，只是为了让实验台打开后不再出现“衣服盖脸、4x2 sheet 坐标错位、游戏和实验台各算各的”。

当前默认拼接已经满足：

- 层级使用新五段头发结构。
- 内衬层存在。
- 游戏和实验台使用同一套分层文件。
- 游戏和实验台使用同一套绘制顺序。
- 后端生成和旧人物刷新都会指向 `art01` 新资源。

但美术上仍需要人工继续调：

- 刘海位置和大小。
- 鬓角与脸侧贴合。
- 身体肩线与盔甲肩线。
- 盔甲和内衬的上下比例。
- 后续其他衣服、发型必须以这套规格为基准重新产出。

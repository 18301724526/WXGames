# 08 已确认设计决策（权威 · 取代 99-open-questions）

> 2026-07-06 与用户逐系统 Q&A 敲定。**这是各系统的最终决策，codex 按本文 + `00`~`07` 建造；本文与设计文档冲突处以本文为准。** 每条含单一事实源/scope 提示。

## 系统 01 · 势力实体
1. **玩家和 AI 用同一个势力模型，零特例。** 玩家 = `factions` 里 `factionId = player_<playerId>` 的那一个；玩家身份从 `game_states.polity` 搬进 factions（polity 身份职责退役）。
2. **AI 势力 = 纯动态群雄割据。** 无固定开局势力；空城被 AI 认领即诞生新势力，从一城弱小起步随进程成长/扩张/被吞并；数量随进程增长。→ 需要「势力诞生/成长/吞并/灭亡」全生命周期（`05` 承接）。
3. **中立守军 = 轻量中立势力实体。** 部落/城邦/遗迹守卫也进 FactionRegistry（轻量），从而可被招降、有外交立场。不再是 `ownerFactionId=null`——它们是有 id 的轻量中立 faction。
4. **玩家也走势力级国库。** 资源从「每城各自」重构为「势力级国库」。⚠️ **大改（单列切片）**：现有 per-city 经济（CityService/城市资源产出）要收敛到势力级 treasury；需按 `refactor-no-debt` 纪律做读证等价迁移。

## 系统 02 · 人物性格 + 相性
1. **性格只驱动世界行为/事件**（相遇率、结仇、招降、背叛倾向等），**不影响战斗数值**（战斗留六维+技能，避免双源）。
2. **性格类型全进配置表、可自由增删**（结构固定、内容可扩）。
3. **相性 = 多维相性向量**（不是一维环）。⚠️ 存储/计算比一维重——向量维度与 compat 函数要在 `02` 钉死，配置表 `compat_matrix` 承载。
4. **好友来投 = 在野武将 + 挖墙角。** 既从在野武将来投，也能因关系网从**友好/敌对 AI 势力的低忠诚武将**倒戈来投（挖墙角）。→ AI 势力武将需带 loyalty（`03`/`05` 消费）。

## 系统 03 · 人物关系网 + 迭代
1. **现在就引入性别 + 取向。** person 新增 `gender` + 取向字段；`romance`（夫婦/恋慕）作为独立关系态。（字段归属 `02` person schema。）
2. **背叛带兵 = 只带个人亲兵小额固定值**（不按比例抽势力/编队兵力，避免与 `05` 势力兵力事实源双扣）。
3. **招降硬上限 + 宁死不降。** 招降成功率封顶（如 0.9）保留失败戏剧性；宿敌在军中时某些将「宁死不降」（成功率封顶 0）。
4. **特殊态不自动解除，但有和解事件。** 义兄弟/宿敌不因好感抖动自动解除；提供「和解」正向事件让宿敌可化敌为友。

## 系统 04 · 外交
1. **同盟可选参战。** 盟友被攻请援，是否出兵玩家/AI 自定；拒援扣好感、可能毁盟（RTK 式）。
2. **玩家↔玩家外交异步：对方离线时提案挂起进收件箱**，等其上线接受/拒绝（AI 同 tick 应答）。→ 需要一个「外交提案收件箱」持久结构（共享世界态）。
3. **朝贡首版不做**（先做好感/关系态/同盟/停战/宣战；朝贡+贸易协议推到经济系统/`05` 之后）。
4. **名声惩罚只波及认识/接壤的势力**（局部，非全世界广播）。
5. （默认）**好感度对称聚合取 `min(favAtoB, favBtoA)`**——一方冷淡则关系上不去。

## 系统 05 · AI 势力 + AI 城市
1. **有出生保护环 / 新手期**：保护环内（或教程期）AI 永不主动攻玩家。
2. **首都失守 = 败者复活流程（新玩法）：** 首都被攻下后，玩家在世界上**远离战场的随机地点刷新一个临时营地**，需完成**一串「势力重建」任务**才能重建势力。⚠️ 新增流程：临时营地实体 + 重建任务链 + 势力 lifecycle 从 collapsed→rebuilding→alive。**不淘汰玩家。**
3. **AI 之间会互相开战/吞并**（真正的群雄割据，世界自演化，弱势力被强势力吞并）。
4. **AI 行军进入某玩家视野 / 以其为目标时，才对该玩家投影**（性能友好、合战争迷雾）。

## 系统 06 · 科技树 Civ 式重设计
1. **保留「每时代选 N 选 1」的抉择/天赋感**（不改成 Civ 全解锁网）。Civ 深度体现在：**扩充节点 + 多种效果类型 + 每势力独立研究**，而非放开选择限制。`eraChoices` 保留。
2. **科技点由 knowledge 持续产出**（学院/知识产出→科技点，受科技率加成；AI 研究节奏自然）。不再按时代整包发放。
3. **科技的战斗加成战前一次性折进队伍战力**（不进实时 bitecs 战斗 tick 热循环，安全、不 NaN）。
4. **敌对/中立 AI 势力科技只经外交/侦察看粗粒度情报**（「已入古典时代」「偏军事路线」），不透明完整清单。

## 被放大 / 新增的设计（scope 提示，路线图据此更新）
1. **势力级国库**（系统01-4）：玩家经济从 per-city 重构到势力级——**较大改**，需读证等价迁移，单列一刀。
2. **纯动态群雄割据 + AI vs AI 互战**（05-1/3、01-2）：AI 需要完整生命周期 + 相互外交/战争的模拟，是 `05` 里最重的部分。
3. **首都失守→临时营地+势力重建任务链**（05-2）：全新玩法流程 + 势力 lifecycle 扩展。
4. **性别 + 取向**（03-1）：person schema 早期就加字段。
5. **多维相性向量**（02-3）：比一维环重，compat 函数/维度要先钉死。
6. **中立守军升级为轻量势力实体**（01-3）：GarrisonPolicy 的守军要能映射到轻量 faction（供招降/外交）。
7. **外交提案收件箱**（04-2）：异步真人外交需要持久提案结构。

## 建造顺序（沿用 `07` 阶段划分，据本文更新）
- **阶段 0（先，最安全）**：纯核 `shared/personalityCore`（多维相性向量 + 性格→行为权重，全配置表）、`shared/relationshipCore`（稀疏有向边迭代 + 特殊态 + 和解）、`shared/diplomacyCore`（好感度 min 聚合 + 5 态机 + 可选参战）+ 对应配置表。**直接给 ②b 捕获招降供相性/关系加成。**
- **阶段 1**：势力实体（含玩家=势力、中立轻量势力、势力级国库迁移）。
- **阶段 2**：世界人物注册表 + 性格/相性/性别落到 person。
- **阶段 3**：关系网（人）+ 外交（势力，含收件箱）。
- **阶段 4**：每势力科技 + 科技树重设计（保留选1、knowledge 产点、战前折算）。
- **阶段 5**：AI 城 + AI 决策循环（动态群雄、AI 互战、进视野投影、出生保护、首都失守重建）。

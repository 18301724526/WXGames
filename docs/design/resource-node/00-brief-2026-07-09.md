# 资源地系统 — 设计任务书（owner 需求，2026-07-09）

## 一、Owner 需求（逐条冻结，设计不得违背）

1. 世界地图上存在**资源地**，等级 1~10 级。
2. 资源地四种类型：**森林（木材）、石矿（石头）、农田（粮食）、铁矿（铁）**。
3. 占领资源地后，按资源地等级给占领者增加**每秒资源收益**（等级越高收益越高）。
4. 资源地有**守军**；玩家战胜守军后进入 **1 分钟部署期**，部署期结束资源地归玩家。
5. **部署期内队伍不可移动、不可撤退**；玩家尝试移动/撤退时必须提示：**"移动/撤退将会中止占领进程"**（确认则中止占领）。
6. 资源地可以**互相攻占**（玩家之间、以及与 PVPVE 世界观一致的势力之间可争夺）。
7. 每种资源地需要**独立美术素材**（后续由 Codex 用 sprite-forge/generate2dsprite 技能生成 — 设计文档需给出素材清单与规格）。

## 二、必须先读的项目现状（设计必须嫁接现有系统，禁止平行造轮子）

读代码，不要凭想象。以下是锚点（文件以仓库实际为准，引用时给真实路径）：

- **行军系统**：`shared/worldMarchCore.js`（可达性单源 MARCH_BLOCKED_TERRAINS）、后端 march 服务与"三条腿结算"（worker/心跳/动作写）；行军 UI/HUD。
- **战斗系统**：`backend` BattleSimService 权威引擎；已有"打野怪营地""占空城"两条完整战斗闭环 — 资源地战斗必须复用同一引擎与链路。
- **守军先例**：GarrisonPolicy（占城守军按距首城设防、打赢才占）— 资源地守军设计应参照其模式与配置方式。
- **野怪营地先例**：多营地确定性铺设 + 冷却重生 + 战利品（activity regions）— 资源地的地图铺设应参照。
- **配置表管线**：`config/tables/*.xlsx` + table-schemas + build-config-tables + `backend/config/generated` + ConfigTables 加载器 — **所有数值必须进配置表**（等级/收益/守军/部署时长等），文档要给出表结构。
- **资源经济**：找到现有资源产出/结算代码（每秒收益如何入账：服务端 tick？行军式结算三条腿？），设计必须与现有结算模型一致并说明选型理由。
- **战争迷雾/世界发现**：野怪=当前视野内才显示（可见性谓词 SSOT 在 WorldMapService）— 资源地的可见性规则要明确并复用该谓词。
- **PVPVE 设计蓝图**：`docs/design/` 五大系统（势力/外交/AI 势力占地）— 资源地的"互相攻占"必须与共享世界 PVPVE 模型兼容（AI 势力是否也占资源地？要给答案）。
- **i18n**：前端文案全走 t() 双语成对；后端直发中文；显示文本禁烘焙进存档。
- **教程/引导**：是否需要教程步（可标记为后续项，但要提及）。

## 三、待决设计问题（设计文档必须逐一给出决定+理由）

1. 守军规模/构成随等级如何缩放；守军是否随时间恢复/重生。
2. 占领后的防御方是什么：占领方军队驻守？快照守军？占领方可否增/换驻军。
3. 部署期被第三方攻击怎么办（部署被打断？战斗插队？）。
4. 每秒收益数值表（1~10 级 × 4 类型），与现有经济数值的量级关系（给出对照依据）。
5. 单玩家占领上限？收益是否随占领数衰减？
6. 资源地在地图上的铺设密度/分布/等级地理梯度（离出生环越远越高级？与 activity regions 关系）。
7. 离线收益如何结算（与"三条腿"模型的关系）。
8. UI：资源地信息面板（等级/类型/守军情报/收益/归属）、部署进度显示、移动/撤退中止确认框 — 注意本项目 UI 公理：按钮只发意图→Manager 管生命周期→面板独立文件（正在进行 button-scheduler 重构，新面板必须按新契约声明，不得走旧路径）。
9. 美术素材清单：4 类型 ×（等级视觉分档，建议 1-3/4-6/7-9/10 四档或给出你的分档）× 状态（无主/占领中/己方/敌方？），每张素材的尺寸/风格锚点（与现有地图元素一致，品红底抠图管线）。
10. 服务端权威与防作弊边界（守军情报"打了才知道"先例是否沿用）。

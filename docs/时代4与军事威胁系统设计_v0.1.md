# 时代4与军事威胁系统设计 v0.2

> 日期：2026-05-29
> 本文只约束边境时代威胁事件。古典时代占领与战斗场景分别见 `docs/古典时代_领土扩张系统设计_v0.2.md` 和 `docs/回合制战斗场景与技能冷却设计_v0.2.md`。

## 目标

时代4用于承接城邦时代后的外部压力：城邦不再只是内部建设，而是开始面对周边部落、商路、边界与秩序问题。边境时代本身只做“士兵产生实际用途”的军事威胁闭环；古典时代（时代5）已经在军事页内接入侦察、世界雷达和基础占领系统，见 `docs/古典时代_领土扩张系统设计_v0.2.md`。

```text
城邦时代建造兵营
→ 兵营自动训练士兵
→ 进入边境时代
→ 建造瞭望台
→ 威胁事件进入事件队列
→ 玩家选择应对方式
→ 后端按士兵/防御结算奖励、损失或 buff
```

## 设计原则

- 后端驱动：时代条件、威胁生成、选项、检定、士兵伤亡和资源变化全部由后端决定。
- 单位简化：第一版仍只使用“士兵”，不区分守卫、战士等兵种。古典时代出征会占用 `soldiersOnMission`，但边境威胁检定仍按当前士兵防御与瞭望台边境防御结算。
- 轻惩罚：威胁失败不会导致文明倒退，只造成少量资源或士兵损失。
- 边境时代不做地图：时代4通过事件队列表达外部冲突；地图侦察与占领属于时代5古典时代系统。
- 复用事件页：威胁事件使用事件红点和事件详情弹窗，不新增复杂战斗 UI。

## 时代4定义

| 字段 | 设计 |
|---|---|
| 时代名 | 边境时代 |
| 进入条件 | 食物 900、木材 500、知识 260、士兵 300 |
| 核心变化 | 外部威胁开始进入常规循环，边界建设成为明确目标 |
| 解锁建筑 | 瞭望台 |
| 军事定位 | 士兵与瞭望台共同组成威胁事件的防御能力 |

士兵条件不是资源消耗，不会在进阶时扣除。它只要求玩家已经建造兵营并完成最小规模训练。当前士兵已经迁移为百人制，但威胁事件里的“防御”仍是抽象评分：兵营士兵按 `defensePerSoldier = 0.01` 折算，300 士兵约等于 3 点士兵防御。

边境时代不是完整古典战争时代。当前版本不硬解锁工坊、学院、神庙等与边境威胁闭环无关的建筑；这些建筑留给后续真正的古典时代或更完整系统。瞭望台的作用是提供 `threatDefense`，让边境时代新增建筑直接服务威胁事件检定。

## 状态结构

```js
threatEventState: {
  nextAt: "2026-05-17T08:30:00.000Z",
  lastGeneratedAt: "2026-05-17T08:25:00.000Z",
  generatedCount: 0,
  recentTemplateIds: []
}

military: {
  soldiers: 300,
  soldierCap: 300,
  defense: 3,
  trainingProgress: 18,
  trainingIntervalSeconds: 30,
  trainingBatchSize: 10,
  defensePerSoldier: 0.01
},

buildingEffects: {
  threatDefense: 2
}
```

## 威胁生成规则

- 只在 `currentEra >= 4` 后生成。
- 事件类型为 `threat`。
- 威胁事件队列上限为 2 个，避免玩家压力过大。
- 第一版固定每 360 秒检查生成一个威胁事件。
- 最近 2 个威胁模板不重复；模板不足时允许回退。
- 威胁事件和常规事件共享事件页红点，但有独立生成节奏。
- 威胁事件生成后 5 分钟过期。
- 过期处理由后端执行：选择 `timeoutOptionId` 指定选项；如果模板未指定，则按效果评分选择惩罚最重的选项，并应用对应 `timeoutEffects`、`failureEffects` 或直接 `effects`。

## 威胁选项规则

威胁事件选项可以包含：

- `requirements`：显示/校验所需士兵或防御。
- `successEffects`：满足条件时应用。
- `failureEffects`：不满足条件时应用。
- `effects`：不做检定，直接应用。

第一版检定规则保持确定性：

```text
如果当前 (military.defense + buildingEffects.threatDefense) >= requiredDefense 且 soldiers >= requiredSoldiers
  应用 successEffects
否则
  应用 failureEffects
```

## 第一批威胁模板

| 模板 | 条件 | 成功 | 失败 |
|---|---:|---|---|
| 边境试探 | 防御 2 | 获得 30 食物、8 知识 | 损失 30 食物 |
| 夜间火光 | 防御 4 | 获得 25 木材、10 知识 | 损失 100 士兵、20 木材 |
| 盗匪勒索 | 士兵 300 | 获得 60 食物 | 损失 45 食物 |

当前代码里的威胁模板只保留一个主动应对选项：

| 模板 | 选项 |
|---|---|
| 边境试探 | 派士兵巡边 |
| 夜间火光 | 保护道路 |
| 盗匪勒索 | 驱赶盗匪 |

早期文档中的“交付食物直接化解”等第二选项尚未在当前代码中开放。

## 前端展示

- 事件列表卡片通过 `event.type === "threat"` 增加威胁样式。
- 事件弹窗继续展示后端下发的 `label` 和 `preview`。
- 军事页展示后端下发的士兵、总防御、训练进度。总防御 = 士兵防御 + 瞭望台边境防御。
- 建造页展示瞭望台卡片和“边境防御”效果，效果文本来自后端共享配置。
- 时代进阶条件中如果包含 `soldiers`，文明页显示“士兵 当前/要求”。

## 延后内容

- 边境时代内主动出征与地图占领（已放到时代5古典时代）
- 多兵种
- 装备和金属资源
- 敌方单位与战斗动画
- 士兵驻防/出征拆分
- 战争科技树

边境时代只把“士兵有用”这件事打通。古典时代已经通过 `TerritoryService` 开放侦察、世界点占领、城市命名和势力命名；威胁事件后续可以继续与这些远征、据点和边境系统产生联动。

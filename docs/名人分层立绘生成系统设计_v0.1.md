# 名人分层立绘生成系统设计 v0.1

更新时间：2026-05-26 00:47:08 +08:00

## 1. 目标

名人立绘不采用“每个名人单独画一整张图”的方式，而采用分层纸娃娃系统。每个名人在生成时得到一份稳定的外观配置，前端 Canvas 按层叠加透明 PNG，组合出独立头像或半身立绘。

这样可以让名人系统继续保持肉鸽生成感，同时避免资源量失控。后续增加新服装、新发型、新时代风格时，只需要追加部件，不需要重画所有名人。

## 2. 第一版范围

第一版只做可测试雏形：

- 统一 `512x512` 透明 PNG 画布。
- 部件以正面胸像为基准，角色朝向、头部位置、肩部宽度保持一致。
- 先覆盖 4 个层级：肤色/脸部、发型、服装、配饰。
- 每类先准备少量素材，验证叠图逻辑和风格统一。
- 暂不做全身像、动态表情、换装动画、复杂武器遮挡。

## 3. 分层顺序

Canvas 绘制顺序固定如下：

```text
backHair      后发、披肩长发、帽冠背面
body          脖颈、脸、耳朵、基础肤色
face          五官、眉眼、表情、胡须
outfit        衣服、盔甲、袍服、披肩
frontHair     刘海、前发、发冠前层
accessory     头饰、伤疤、耳饰、小武器、身份饰件
frameEffect   阵营光效、来源光效，第一版可选
```

如果某个部件不需要该层，配置里可以为空。

## 4. 文件命名

素材放在：

```text
frontend/assets/art/famous-person/layers/
frontend/assets/art/famous-person/preview/
```

命名规则：

```text
fp-layer-<layer>-<theme>-<index>.png
fp-preview-<archetype>-<index>.png
```

示例：

```text
fp-layer-body-skin-01.png
fp-layer-frontHair-short-01.png
fp-layer-outfit-vanguard-01.png
fp-layer-accessory-scar-01.png
```

## 5. 外观配置

名人存档建议增加 `appearance` 字段。该字段由后端生成并保存，前端只负责渲染。

```js
appearance: {
  version: "famous-portrait-v0.2",
  seed: "player:time:roll",
  palette: "military_red",
  layers: {
    backHair: "fp-layer-backHair-long-01.png",
    body: "fp-layer-body-skin-02.png",
    face: "fp-layer-face-calm-01.png",
    outfit: "fp-layer-outfit-vanguard-01.png",
    frontHair: "fp-layer-frontHair-short-01.png",
    accessory: "fp-layer-accessory-scar-01.png"
  }
}
```

为了保证同一名人每次打开都长一样，生成算法必须只使用服务端已保存的种子，不依赖前端随机数。

## 6. 生成规则

第一版生成时按名人定位限制素材池：

| 定位 | 服装倾向 | 发型/配饰倾向 |
| --- | --- | --- |
| `vanguard` 突击领队 | 皮甲、轻甲、红铜色边境战装 | 短发、伤疤、护额、短兵器 |
| `guardian` 守备领队 | 厚甲、披风、盾纹 | 束发、沉稳表情、护肩 |
| `tactician` 谋略领队 | 深色袍服、腰封、羽饰 | 长发、发冠、卷轴 |
| `warden` 城市治理 | 礼服、织纹、玉佩 | 温和表情、冠带 |
| `artisan` 工巧人才 | 工匠围裙、铜铁护具 | 工具、护目镜、扎发 |
| `scholar` 知识人才 | 学者长袍、书卷纹样 | 发冠、书卷、冷静表情 |

肤色、脸型、基础表情可以跨定位复用；服装和配饰按定位加权。

## 7. 前端渲染方案

Canvas 渲染名人卡片时：

1. 读取 `person.appearance.layers`。
2. 按固定层级绘制图片。
3. 使用同一个头像区域裁切，例如圆形或方形圆角遮罩。
4. 如果缺少部件，回退到现在的文字头像。

第一版建议先实现一个独立方法：

```js
drawFamousPortrait(person, x, y, size, options)
```

以后名人列表、出征领队选择、战报弹窗都调用同一个方法，避免再出现多套 UI 逻辑。

## 8. 后端生成方案

名人生成器新增外观池：

```js
const APPEARANCE_POOLS = {
  body: [...],
  face: [...],
  frontHair: [...],
  backHair: [...],
  outfit: {
    vanguard: [...],
    guardian: [...],
    tactician: [...],
    warden: [...],
    artisan: [...],
    scholar: [...]
  },
  accessory: {
    military: [...],
    governance: [...],
    craft: [...],
    knowledge: [...]
  }
};
```

`createFamousPersonCandidate` 生成候选时同步生成 `appearance`。接纳名人时保留该字段。旧存档名人如果没有 `appearance`，可以在 normalize 时补一个稳定配置。

## 9. 风格约束

- 风格和现有建筑/科技图标保持同一游戏感：厚涂、清晰轮廓、偏写实但不过分照片化。
- 不做现代服饰，不做抽卡金边卡牌感。
- 立绘主体需要有较强轮廓，缩小到 `64x64` 仍能分辨。
- 每个部件必须透明背景，不带底色、不带投影底板。
- 所有部件中心点一致：头部中心约在画布 `x=256, y=190`，肩部约在 `y=330`。

## 10. 测试方法

第一步只测试素材：

1. 在 `frontend/assets/art/famous-person/layers/` 查看每张 PNG 是否透明。
2. 用预览合成图检查脸、头发、服装、配饰是否对齐。
3. 缩小到 `64x64` 检查是否还能辨认角色气质。

第二步测试代码：

1. 打开名人面板。
2. 寻访或接纳一位名人。
3. 名人卡片显示组合立绘，而不是单字头像。
4. 刷新页面后，同一名人立绘不变化。

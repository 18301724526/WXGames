# 武将属性点分配第一批落地任务 v0.1

日期：2026-05-30

目标版本：`0.1.165`

## 目标

在 `0.1.164` 已经让战斗经验写回领队名人，并产生自由属性点后，本批把自由属性点接到名人面板，让玩家可以直接把点数分配到六维属性。

本批只做最小闭环：

- 已加入名人若有 `freeAttributePoints`，名人卡片显示六维加点入口。
- 每点击一次对应属性的 `+`，消耗 `1` 点自由属性点。
- 后端校验名人存在、属性属于六维、剩余自由属性点大于 `0`。
- 加点直接写回 `person.attributes`，后续战斗快照自然读取更新后的六维。
- 记录 `assignedAttributePoints` 和 `lastAttributeAssignedAt`，方便后续做重置、日志或展示。

本批不做：

- 撤销、重置或二次确认。
- 单属性硬上限提示。
- 品质自动成长属性写回。
- 技能点、技能升级或技能替换。

## 规则

六维固定为：

- 统帅：`command`
- 武力：`force`
- 智力：`intelligence`
- 政治：`politics`
- 魅力：`charisma`
- 速度：`speed`

每次加点：

1. 前端发送 `assignFamousAttributePoint`。
2. 请求携带 `personId` 和 `attribute`。
3. 后端读取对应名人。
4. 若 `freeAttributePoints <= 0`，返回 `NO_FREE_ATTRIBUTE_POINTS`。
5. 若属性不在六维中，返回 `INVALID_ATTRIBUTE`。
6. 成功时该属性 `+1`，`freeAttributePoints -1`，`assignedAttributePoints[attribute] +1`。

为了避免已经加过点的旧名人在归一化时被重新压回 `99`，本批区分初始生成上限和已加入名人归一化上限：

- 新生成初始属性仍限制在 `1..99`。
- 已加入名人归一化允许超过 `99`，第一批使用宽上限 `9999`。

## 落地范围

### 后端

- `FamousPersonService.assignAttributePoint()`：处理加点校验和写回。
- `FamousPersonService.normalizeProgression()`：归一化 `assignedAttributePoints`。
- `FamousPersonService.normalizeAttributes()`：初始生成与已加入名人使用不同上限。
- `/game/action`：接入 `assignFamousAttributePoint`。
- 新手引导和任务校验白名单允许该 action，不阻断普通操作。

### 前端

- `GameAPI.assignFamousAttributePoint(personId, attribute)` 调用共享 action endpoint。
- `UIStatePresenter.buildFamousPersonCard()` 为有自由属性点的已加入名人输出 `attributeActions`。
- `CanvasGameRenderer.renderFamousPersonItem()` 在名人卡片中绘制六维 `+` 控件。
- `CanvasActionController` 和 `CanvasGameApp` 转发并应用加点后的最新状态。
- 加点后名人面板保持打开，玩家可以连续分配。

## 验证点

- 有自由属性点的已加入名人显示六维加点入口。
- 候选名人不显示等级、经验和加点入口。
- 点击 `统帅 +` 后，统帅数值增加 `1`，自由属性点减少 `1`。
- `assignedAttributePoints.command` 增加 `1`。
- 当属性从 `99` 加到 `100` 后，再归一化仍保持 `100`。
- 没有自由属性点时返回失败，不修改属性。
- 加点后名人面板不关闭。
- 后续战斗读取的名人六维应使用加点后的属性。

## 实现记录

- 2026-05-30：`0.1.165` 落地武将属性点分配第一批。自由属性点可以在名人面板按六维一键 `+1`，后端写回名人属性并记录分配明细；本批暂不做撤销、重置和品质自动成长。

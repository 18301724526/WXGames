# 20260528 名人立绘切图交接

## 当前结论

名人立绘 v3 现在只保留三层：`face`、`outfit`、`hair`。实验台和游戏内渲染已经统一为同一套布局逻辑，并且缩放锚点已经改为 512 画布中心，避免缩放时按左上角漂移。

本地实验台：

```text
http://127.0.0.1:8080/tools/famous-portrait-lab.html
```

如果本地服务没开，运行：

```powershell
node scripts/local-preview-server.js
```

## 正式资源

当前正式游戏资源只以这个目录为准：

```text
frontend/assets/art/famous-person/layers/
```

当前目标文件固定为 30 张透明 PNG：

```text
fp-layer-v3-face-01.png   ... fp-layer-v3-face-10.png
fp-layer-v3-outfit-01.png ... fp-layer-v3-outfit-10.png
fp-layer-v3-hair-01.png   ... fp-layer-v3-hair-10.png
```

每张必须保持：

```text
透明背景
同一内部坐标规格
```

不要提交额外生成过程图、整图 sheet 或临时素材。重新切图时，直接替换 `layers` 目录里的同名透明 PNG。

## 当前默认参数

当前校准后的默认参数：

| 层级 | 文件 | scale | x | y |
| --- | --- | ---: | ---: | ---: |
| face | fp-layer-v3-face-01.png | 0.88 | 0 | 3 |
| outfit | fp-layer-v3-outfit-01.png | 2 | 0 | 83 |
| hair | fp-layer-v3-hair-10.png | 0.88 | 8 | -90 |

## 实验台操作

实验台左侧是参数区，右侧固定预览区。右侧一屏内同时显示：

```text
完整预览
游戏头像区域
卡片实显
10 套规格检查
```

`套装编号` 会快速切换同编号的衣服、脸型、发型；每层文件下拉可以单独切换，用来检查混搭或定位某张问题图。

本地替换 PNG 后，不需要重启服务，点实验台里的 `刷新图片` 按钮即可重新加载。

## 注意事项

当前工作中已经确认：正式提交只保留 `layers` 里的成品透明图和布局参数。回家后如果继续改切片，先确认 `git status`，避免覆盖自己刚切好的文件。

# 战斗状态可视化第二批落地任务 v0.1

日期：2026-05-30
目标版本：`0.1.163`

## 目标

第一批状态已经能真实结算，但只靠飘字容易错过。本批把状态和技能冷却做成战斗画面里的常驻可读信息，让自动战斗过程更容易验收。

## 本批范围

- 战斗回放视图输出双方当前状态徽标。
- 战斗回放视图输出双方主动战法的当前冷却状态。
- Canvas 战斗场景在双方小兵阵列上方显示状态和冷却。
- 不改变后端伤害公式、状态持续规则、技能释放条件。

## 玩家可见表现

| 信息 | 展示方式 | 说明 |
| --- | --- | --- |
| 主动战法 | 队伍状态栏第一行展示战法名。 | 没有战斗技能时显示 `无战法 / 只普攻`。 |
| 冷却 | `可释放`、`正在释放`、`冷却 N 回合`。 | `N` 取战报中的剩余冷却，不重新推算结算。 |
| 守御 | `守御 N`。 | `N` 是当前剩余护盾值。 |
| 破甲 | `破甲 N回合`。 | `N` 是剩余持续回合。 |
| 灼烧/中毒 | `灼烧 N回合`、`中毒 N回合`。 | 持续伤害仍在目标自己行动前结算。 |

## 实现步骤

| 步骤 | 状态 | 内容 | 验收 |
| --- | --- | --- | --- |

## 不做项

- 不做状态图标资源，只用文字徽标先保证可读性。
- 不做状态 tooltip。
- 不做玩家手动暂停、快进、逐手回放。
- 不做敌方技能生成扩展。

## 人工测试用例

1. 进入一场带守御技能的战斗。
   - 预期己方释放后，己方小兵阵列上方出现 `守御 N`。
   - 后续受击后，`守御 N` 的数值会随战报阶段变化。

2. 进入一场带破甲、灼烧或中毒状态的战斗。
   - 预期目标小兵阵列上方出现对应状态和剩余回合。
   - 飘字仍在状态触发或施加时出现。

3. 观察主动战法冷却。
   - 释放前显示 `正在释放` 或 `可释放`。
   - 释放结算后显示 `冷却 N 回合`。
   - 纯内政/魅力型无战斗技能时显示 `无战法 / 只普攻`。

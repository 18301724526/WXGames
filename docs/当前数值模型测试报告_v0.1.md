# 当前数值模型测试报告 v0.1

更新时间：2026-05-23 23:38:37 +08:00  
代码基线：`0.1.93`，提交 `6ff903e Smooth open-ended building effect curve`  
测试脚本：`scripts/economy-balance-model.js`

## 1. 测试目的

这份报告回答三个问题：

1. 当前收支是否平衡？
2. 哪些系统会导致无脑选择？
3. 后续调数值应该用什么模型重复验证？

结论先行：当前不是“所有都坏了”，但核心经济闭环没有真正成立。新手主线靠任务奖励报销成本；脱离任务奖励后，知识是唯一明显瓶颈；粮食和木材建筑回本过快，长期缺少维护消耗。

## 2. 可执行测试模型

脚本路径：

```bash
node scripts/economy-balance-model.js
```

模型直接读取当前代码配置：

- 建筑配置：`backend/config/BuildingConfig.js`
- 建筑效果：`backend/calculators/BuildingEffectCalculator.js`
- 资源公式：`backend/calculators/ResourceTickCalculator.js`
- 时代成本：`backend/config/EraConfig.js`

模型当前跑三类测试：

- 主线闭环仿真：按当前新手主线奖励走到古典时代
- 去掉主线报销后的底层经济压力：看自然产出多久能满足时代进阶
- 建筑边际回本测试：看升级成本和新增产出多久回本

## 3. 主线闭环测试结论

按当前主线任务奖励和必做步骤走到古典时代：

| 项目 | 数值 |
| --- | ---: |
| 总等待时间 | 2.56 秒 |
| 初始资源 | 粮食 100 |
| 在线自然产出 | 粮食 10，知识 0.38 |
| 主线任务奖励 | 粮食 3830，木材 1735，知识 1185 |
| 事件奖励 | 木材 20 |
| 时代额外奖励 | 知识 5 |
| 建筑总消耗 | 粮食 940，木材 135，知识 300 |
| 时代总消耗 | 粮食 3000，木材 1600，知识 885 |
| 总消耗 | 粮食 3940，木材 1735，知识 1185 |

占比：

| 来源 | 粮食 | 木材 | 知识 |
| --- | ---: | ---: | ---: |
| 自然产出 / 总消耗 | 0.25% | 0% | 0.03% |
| 主线任务奖励 / 总消耗 | 97.2% | 100% | 100% |

这说明：当前新手到古典的主要推进力不是经济系统，而是任务奖励直接发下一步所需资源。玩家只要按任务点，几乎不需要理解产出、人口、建筑搭配。

## 4. 奖励覆盖测试

当前所有主线奖励都精确覆盖下一步成本：

| 任务 | 覆盖情况 |
| --- | --- |
| 备齐聚落物资 | 粮食 120/120，知识 5/5 |
| 备齐伐木物资 | 粮食 50/50，木材 15/15 |
| 聚落分工 | 粮食 500/500，木材 200/200，知识 100/100 |
| 城邦守备 | 粮食 260/260，知识 80/80 |
| 守备成军 | 粮食 900/900，木材 500/500，知识 260/260 |
| 边境瞭望 | 粮食 180/180，木材 120/120，知识 60/60 |
| 扩编兵营 | 粮食 420/420，知识 160/160 |
| 古典远行 | 粮食 1400/1400，木材 900/900，知识 520/520 |

问题：

- 奖励设计太“报销制”，任务变成传送带。
- 经济产出不再是主要推进条件，只剩 UI 数字在动。
- 玩家不会因为更好的人才方针、更好的建筑搭配而明显更快进入下一阶段。

建议方向：

- 主线奖励不要 100% 覆盖时代/建筑成本。
- 奖励应该补关键缺口，而不是完整替代生产。
- 第一轮教学可以报销 60%-80%，后续逐步降到 20%-40%。
- 任务目标可以从“领奖后直接够”改成“领奖后进入 1-3 分钟可完成窗口”。

## 5. 去掉任务报销后的底层经济压力

如果不靠主线完整报销，只靠当前自然产出：

| 阶段 | 目标成本 | 当前产速 | 预计等待 | 瓶颈 |
| --- | --- | --- | ---: | --- |
| 聚落 -> 城邦 | 粮食 500，木材 200，知识 100 | 粮食 3.7/s，木材 2/s，知识 0.2/s | 7.92 分钟 | 知识 |
| 城邦 -> 边境 | 粮食 900，木材 500，知识 260 | 粮食 3.3/s，木材 4/s，知识 0.45/s | 9.63 分钟 | 知识 |
| 边境 -> 古典 | 粮食 1400，木材 900，知识 520 | 粮食 3.3/s，木材 4/s，知识 0.45/s | 19.3 分钟 | 知识 |

判断：

- 如果完全取消任务报销，前期会变成知识卡点。
- 粮食不是长期瓶颈，木材压力比粮食大，但仍明显小于知识。
- 知识基础产出慢，而学院又未实际解锁，所以知识经济在古典前缺少可控提升手段。

建议方向：

- 聚落后需要更早给玩家可控的知识增长方式。
- 任务奖励可以保留知识补给，但不应直接填满全部时代成本。
- 如果要让知识作为文明推进瓶颈，需要给它配套选择：学者价值、建筑、事件、方针都要能影响节奏。

## 6. 建筑回本测试

### 6.1 农田

按 3 农民计算：

| 升级 | 成本 | 新增粮食/s | 直接回本 |
| --- | --- | ---: | ---: |
| 0->1 | 免费 | +1.5 | 0 秒 |
| 1->2 | 粮食 50 | +1.5 | 33.3 秒 |
| 2->3 | 粮食 120 | +1.5 | 1.33 分钟 |
| 3->4 | 粮食 300 | +1.5 | 3.33 分钟 |
| 4->5 | 粮食 350 | +1.08 | 5.39 分钟 |
| 5->6 | 粮食 410 | +0.9 | 7.61 分钟 |

判断：

- 农田早期升级非常强，基本无脑。
- 只消耗粮食、又增加粮食，没有维护或其他资源约束，属于自我复制型投资。
- 开放等级后的曲线已经变平滑，但前 1-4 级回本依然过快。

### 6.2 伐木场

按 1 工匠计算：

| 升级 | 成本 | 新增木材/s | 直接回本 |
| --- | --- | ---: | ---: |
| 0->1 | 粮食 50，木材 15 | +2 | 7.5 秒 |
| 1->2 | 粮食 80，木材 30 | +2 | 15 秒 |
| 2->3 | 粮食 150，木材 60 | +2 | 30 秒 |
| 3->4 | 粮食 180，木材 75 | +1.44 | 52 秒 |
| 4->5 | 粮食 220，木材 90 | +1.2 | 1.25 分钟 |
| 5->6 | 粮食 260，木材 110 | +1.05 | 1.75 分钟 |

判断：

- 伐木场回本极快，比农田更无脑。
- 只要有工匠，伐木场升级几乎没有决策成本。
- 如果未来木材承担建造压力，伐木场会成为必升项，不是策略项。

### 6.3 学院

按 2 学者计算：

| 升级 | 成本 | 新增知识/s | 直接回本 |
| --- | --- | ---: | ---: |
| 0->1 | 粮食 180，木材 60，知识 30 | +0.06 | 8.33 分钟 |
| 1->2 | 粮食 260，知识 180 | +0.06 | 50 分钟 |
| 2->3 | 粮食 360，知识 260 | +0.06 | 72.2 分钟 |
| 3->4 | 粮食 440，知识 320 | +0.04 | 123 分钟 |

判断：

- 学院当前未实际解锁，但按配置看，知识回本远慢于粮食、木材。
- 如果将来解锁，学院升级不太像无脑项，反而可能太慢。
- 它只提升学者知识，不提升人口基础知识，所以少学者时收益很低。

## 7. 是否“全都有问题”

不是全都有问题，当前失衡集中在以下几类。

### A. 任务奖励结构有问题

严重程度：高。

表现：

- 新手到古典几乎 100% 靠主线奖励付费。
- 自然产出在主线闭环中占比接近 0。
- 玩家不需要经营，只需要领取。

这是当前最大的经济问题。

### B. 粮食和木材生产建筑太无脑

严重程度：高。

表现：

- 农田早期 33 秒到 3 分钟回本。
- 伐木场早期 7.5 秒到 30 秒回本。
- 没有维护消耗，没有空间/宜居惩罚实际扣资源，没有资源替代成本。

结果：

- 只要有资源就升，几乎不需要判断。
- 建筑等级越高，长期资源越容易滚起来。

### C. 知识经济被任务奖励遮住

严重程度：中高。

表现：

- 去掉任务奖励后，知识是所有阶段的最大瓶颈。
- 但主线奖励又把知识 100% 送满。
- 学者方针能影响知识，但玩家在主线里没有明显体感。

结果：

- 真实瓶颈存在，但被奖励系统隐藏。

### D. 铁矿和石料不是经济资源

严重程度：中。

表现：

- 无持续产出。
- 无主线奖励。
- 无建筑成本。
- 无时代成本。

结果：

- 现在只是 UI 上存在，不参与策略。

### E. 兵营 2 级后断层

严重程度：中。

表现：

- 兵营开放等级成本继续涨。
- 但士兵上限和训练速度 2 级后不再成长。

结果：

- 继续升级兵营是坏投资。

## 8. 下一步调参建议

优先级 1：改任务奖励结构。

- 不再 100% 报销下一步成本。
- 第一轮教学可以高覆盖，后续逐步降低。
- 让奖励把玩家送进“短等待可完成”的区间，而不是立即完成。

建议目标：

| 阶段 | 建议任务覆盖率 | 目标等待 |
| --- | ---: | ---: |
| 农耕 -> 聚落 | 80%-100% | 0-30 秒 |
| 聚落 -> 城邦 | 50%-70% | 1-3 分钟 |
| 城邦 -> 边境 | 40%-60% | 2-4 分钟 |
| 边境 -> 古典 | 30%-50% | 3-6 分钟 |

优先级 2：让生产建筑有选择成本。

- 维护成本可以先轻量启用，不要一步到位重压。
- 生产建筑升级应消耗跨资源，比如农田升级也吃木材/石料，伐木场升级也吃粮食/知识/石料。
- 宜居度可以继续影响人口成长，但不要替代资源成本。

优先级 3：补知识的可控增长链路。

- 聚落或城邦阶段需要一个早期知识建筑/机制。
- 或者让民居/攻略/事件提供小额稳定知识选择。
- 学院解锁前，学者应有更强存在感。

优先级 4：补铁矿、石料闭环。

- 石料适合作为民居、瞭望台、学院、神庙升级材料。
- 铁矿适合作为兵营、军事、工坊升级材料。
- 默认没有产出可以，但要尽快设计采石/采矿/贸易/事件来源。

优先级 5：修兵营开放等级曲线。

- 兵营 3 级后应继续增加士兵上限或训练速度。
- 否则应限制兵营继续升级，避免坏投资。

## 9. 后续每次调数值的验收标准

每次改完数值，跑：

```bash
node scripts/economy-balance-model.js
```

建议看四个指标：

1. 主线任务奖励占总消耗比例：不应长期接近 100%。
2. 每阶段自然等待时间：早期应在 30 秒到 6 分钟之间逐步拉开。
3. 建筑回本时间：核心生产建筑早期可以短，但不应普遍低于 1 分钟。
4. 瓶颈资源：不能永远只有知识，也不能完全没有瓶颈。

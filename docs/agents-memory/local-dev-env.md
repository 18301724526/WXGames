**Git push→deploy pipeline (identical mechanism to the server):**
- Bare repo `/home/git/wxgame.git`; git-http on **:3001** via nginx + `git-http-backend` (fcgiwrap, run as **root** via a systemd override so it can write the root-owned repo and drive root's pm2) + basic auth `/home/git/.htpasswd` (`wxgame:wxgame123`).
- Branch-aware `post-receive`: pushing **`codex-battle`** (or `codex/battle-core-test-server`) checks it out to a worktree, sources `/home/git/wxgame-local.env` (local path/port overrides), and runs the **real `scripts/deploy-test-server.sh` → `deploy.sh`** — the same deploy scripts as prod, just env-overridden. `SKIP_DEPLOY_GATE=1` (the prod CI gate `test-server-ci-gate.sh` runs full lint+prettier+tests — too heavy per local push; run manually when wanted). `ALLOW_WXGAME_DEPLOY_PATH_OVERRIDE=1` bypasses the `/www/wwwroot/h5` path guard.
- Local layout under `/root/wxgame-test/`: `worktree/` (checkout), `public/` (nginx root, frontend published with `API_BASE` rewritten to `/wxgame-test-api` + LOCAL-TEST badge), `backend/` (runtime, `civilization.db` = the real trimmed save), `.wxgame/` (deploy state + config release). Backend on **PORT 3002**, pm2 `wxgame-test-server` + `wxgame-test-world-worker` (dual process). `NODE_ENV=production` → JWT/CORS come from `/root/wxgame-test/main.env` (copied to `backend/.env`; `JWT_SECRET` also exported in `wxgame-local.env` so pm2 `--update-env` passes it); the config-release gate is satisfied because `deploy.sh` publishes a release. `MAIN_DB_PATH` points at a nonexistent file so `prepare-test-server-runtime.sh` skips overwriting the DB → local save persists across deploys (deploy's backend rsync excludes `*.db`).

**Real save:** online DB trimmed (`DELETE api_logs/client_operation_logs; VACUUM`, 417MB→2.4MB, kept 37 players + world map) lives at `tmp/wxlocal.db` (Windows) and seeded into `backend/civilization.db`.

**Use it:**
- Browser: **http://localhost** (works now). For **http://wxgame.local** the user must add a hosts line as Administrator (the tool can't elevate): `Add-Content $env:SystemRoot\System32\drivers\etc\hosts "`n127.0.0.1 wxgame.local"`. nginx `server_name` already includes both.
- **Push to deploy:** from the Windows repo, remote `local` = `http://wxgame:wxgame123@localhost:3001/wxgame.git` is configured → `git push local codex-battle` triggers the full deploy. (Pushing the same commit is a no-op — the hook only fires on actual ref updates.)
- **`wx {deploy [branch]|up|restart|logs|status}`** (`/usr/local/bin/wx`): `wx deploy` runs a deploy without pushing; `wx up` after a reboot; `wx logs` tails the pm2 apps. pm2 auto-starts on WSL boot (`pm2-root.service` enabled); nginx + fcgiwrap are systemd too.

**DEPLOY RUNBOOK (rewired 2026-07-03 — wsl.exe is no longer needed for deploys):**
- The post-receive hook now deploys **`codex-battle` and ANY `codex/*` branch** (I widened the case via `\\wsl.localhost\Ubuntu-24.04\home\git\wxgame.git\hooks\post-receive`, + a zero-hash guard so branch DELETIONS don't run deploy_test under `set -euo`). **Deploy = `git push local <branch>`** — runs fully over HTTP inside the VM, immune to the wsl.exe session wedge. Hook is synchronous: the push blocks ~30-60s and streams `[Deploy] 部署完成` + health JSON back.
- To redeploy the SAME commit (hook fires only on ref change): `git push --force local '<sha>^:refs/heads/<branch>'` then `git push local <branch>` (double deploy, ends correct).
- Verify from Windows over HTTP (mirrored networking): `curl -s --noproxy '*' http://localhost:3002/api/health` → `appVersion.deployedCommit`.
- **⚠️ config-release 版本漂移坑 (2026-07-06)**: 本地 `npm run test:architecture` 的 config-pipeline 门比对**git baseline manifest**，但 WSL 部署的 config-release **publish 门比对服务器当前 active release 版本**——两者会分叉。症状：改了某 registry(如 `backend/config/defaultTaskDefinitions.json`/`TechTreeConfig.js`)内容、bump 了版本、**过了本地门禁**，但 `git push local` 部署时 hook 报 `X: minor version bump required; expected >= A.B.C, got <你的版本>` 且 `config-release publish success:false` → **git ref 推上去了但部署卡在 config-release、没生效**(health 仍显旧 deployedCommit)。原因=服务器 active release 的该 registry 版本比本地 baseline **更高**(早先某次部署留下的)。修法：把该 registry 版本 bump 到 **服务器要求的 `expected >=` 值**(或更高)，新 commit 触发重新部署(hook 只在 ref 变化时跑，重推同 commit 无效)。部署成功标志：`config-release publish success:true` + `drift matchesCurrent:true` + health `deployStatus:succeeded/complete exitCode 0`。
- **wsl.exe session-creation wedges REPEATEDLY on this machine** (twice on 2026-07-03: once from leaked client pileup with distro Stopped, once with distro Running — new sessions hang on `echo` while `wsl -l -v`, the user's existing session, HTTP into the VM, and `\\wsl.localhost` 9P file access ALL still work). So: never build automation on new wsl sessions; use push→hook (deploy), HTTP (verify), `\\wsl.localhost` (file edits). Killing wsl.exe clients kills the user's open terminal sessions — ASK FIRST; and after clients die the VM idle-stops (60s default) taking localhost down.
- `.wslconfig` now has `vmIdleTimeout=86400000` under `[wsl2]` (24h; takes effect at next `wsl --shutdown`/reboot) so the VM no longer needs a babysitter terminal.

**Setup gotchas (cost real time):**
- `/root` is mode 700 → nginx (www-data) 403/500 on static. Fixed `chmod 711 /root` (+ `a+rX` on served dirs).
- **PowerShell→wsl→bash mangles space-paths and truncates CJK/box-drawing output.** Use `wsl -d Ubuntu-24.04 -u root --cd "F:\AI Project\WXGamesLocal\..."` (space-path becomes one PowerShell arg) and run **LF-normalized script files** (see [[line-endings-lf]]), not long inline `bash -lc`. For output you must read, `tee` it to a file under `tmp/` and Read that (note: PowerShell `>` redirection writes UTF-16; prefer in-script `tee`).

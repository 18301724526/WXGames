The frontend render layer (NOT the de-shell scope) is the next architecture target — it's where "single-source ≠ one-step localization" bites. Found while live-verifying the de-shell.

**Bug fixed (641ee6e3, deployed private + local WSL):** tapping a world site threw `Failed to execute 'createLinearGradient' … non-finite` at `WorldMapSiteOverlayRenderer.js:732`. Root: `WorldMapCanvasRenderer.get width/height` returned `undefined` and there was no `bottomSafeArea` getter, on the composition path where the host exposes ctx/canvas but not width. The city-command overlay reads `this.width/height/bottomSafeArea` DIRECTLY (whereas `render()` takes them as params — that's why the map drew fine but the overlay math NaN'd: `this.width*0.2`→NaN→`primaryX`→createGradient throw). Fix = those getters fall back to live `canvas.clientWidth/clientHeight` + 12px default; plus `createGradient`/`createRadialGradient` (CanvasSurfaceRenderer + H5UpdateRuntimeAdapter) now return the solid fallback on non-finite coords. 2 lock tests added.

**The durable lesson (answers user's "你真的能一步定位吗 / 检查你漏了多少"):** NO, not for render bugs. Localizing this took ~25 file reads because:
1. **Host-delegation getters silently yield `undefined`** — `get width() { return this.host?.width; }` with no `Number()||fallback` (unlike the canonical `CanvasSurfaceRenderer` which guards `Number(this.host?.width)||0`). The chain is 4+ levels deep (SiteOverlay → WorldMapCanvasRenderer → host → … → H5CanvasGameRenderer), every link `this.host?.x` no fallback. One missing link → `undefined` → NaN-cascades ~8 functions deep before throwing somewhere unrelated-looking.
2. **No per-stage trace in the render pipeline** (the presenter has `WorldMarchTrace.logDedup`; renderer/bake/overlay stages have none). So a NaN coord can't be traced to its origin — you crawl.
3. This is the [[refactor-server-push-deploy-502-fix]]-era **office host-bridge retirement (65821c0c)** fallout: deleting the auto-forwarding Proxy exposed every child renderer that relied on implicit host forwarding. Earlier same-class fixes: blank terrain `651fde11` (ctx-threading), modal/water-mask `b9b07412`.

**Sweep + lock DONE (6a57f310, deployed private + WSL):** all 56 numeric-geometry host-delegator getters (width/height/viewport*/bottomSafeArea/pixelRatio) across 19 renderers converted to canonical `Number(this.host?.x) || fallback` — the undefined→NaN crash class is eliminated layer-wide, not just the one spot. Locked by extending `scripts/check-renderer-host-bridge-retired.js` (already in architecture-smoke) with an `unguarded-numeric-host-getter` rule (+ mutation test): a bare `return this.host?.width;` now FAILS the gate, so it can't regress. Also removed the redundant direct `worldActorLayerRenderer` write in CanvasGameShellWorldMapLayerRuntime (single write point via `syncWorldActorOverlayRendererLinks`).

**Still open (lower priority):** per-stage render trace (bake/overlay stages still lack the presenter's `WorldMarchTrace`-style logging) — for one-step localization *when* a render bug occurs. The NaN-crash class itself is now structurally prevented, so this is observability-nice-to-have, not crash-prevention. Relates to [[architecture-refactor]] north star and [[refactor-no-debt-for-safety]].

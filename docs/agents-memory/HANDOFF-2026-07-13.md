# 交接包（2026-07-13，监督者到执行席）

本目录是当时协作记忆的仓库镜像。本文件只保留仍可复用的执行纪律、环境边界和交接格式；已经退役的产品域排查队列不再记录。

## 当前可复用事实

- 每个修复单应绑定一个明确基线、一个独立提交和可机械复核的验证结果。
- 定位结论必须区分产品缺陷、工具缺陷和环境偏差；没有运行时证据时应降低置信度。
- worktree 任务使用 detached 基线，并同时连接根目录与 backend 的依赖目录；任务结束后清理联结和后台进程。
- 本地验证禁止指向远程测试流量，只允许对已明确授权的健康端点做只读检查。

## 验证分工

定位席负责给出完整调用链、矛盾状态和最小修复建议；执行席负责实现；验证席依据固定基线复跑命令并保留原始汇总。评审者必须亲核 diff 和测试输出，不把执行席自报当成证据。

## 纪律速查

- 一任务一提交，完成后立即停下等待审查。
- 代码定位优先使用 CodeGraph，避免无边界全文搜索。
- 每个测试命令记录退出码、通过数、失败数和关键原文。
- 行为变更、范围裁剪或新增外部成本需要 owner 明确裁决。
- 报告文件默认不提交，除非任务明确要求纳入版本控制。

## 日终回执

日终记录完成项（提交号与验证数字）、失败或搁置原因、剩余队列和新发现的架构风险，供下一位监督者对账。

Splitting the frontend god-files by responsibility (user: "巨石文件至少先按职责切分一下"). Proven procedure (each cut is one commit, pushed to private + local):
1. Pick a self-contained method cluster in the god-file.
2. Extract into a new `install(TargetClass)` mixin module (codebase idiom: `Object.assign(TargetClass.prototype, {...})`) — public API unchanged (methods still land on the same prototype via both installs).
3. Wire 4 places: require+install in the host class file; `<script>` in `frontend/index.html` **before** the host's script tag; `require()` in `frontend/minigame/game.js`; co-install in the host's `.test.js` harness.
4. Maintain guard baselines: if the moved methods have mode/panel state refs, re-point their rows in `docs/development_logs/2026-06-25-frontend-ecs-0a-mode-ownership-baseline.md` (count-based per file+symbol — relocation, not growth; then `prettier --write` that .md to realign the table).
5. Gate green: npm test, architecture-smoke, lint, prettier, git diff --check, script-manifest, suppressions budget. Then commit + push both remotes.

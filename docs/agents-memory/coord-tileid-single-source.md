**How to apply (reusable lessons for the next cluster):**
- **Load-order trap:** a module that captures a global at LOAD time (`const X = global.X || require(...)`) but loads BEFORE that global is defined (check `index.html` script order) gets `null` in the browser, so its "dead fallback" is actually the LIVE path ŌĆö and Node tests stay green via the require shim (false confidence). `ClientOperationLog` (loads :17, before TileCoord:24) was this. Fix: convert to a call-time getter, THEN collapse. Always check the `<script>` order before assuming a fallback is dead.
- **Variant vs duplicate:** several "collapse onto X" targets were march-shape RESHAPE wrappers (`{q,r,tileId}`, no x/y, test-pinned). Don't bare-swap to the canonical (changes shape, breaks tests) ŌĆö keep the wrapper, delete only the dead inner fallback, route the tileId format to the canonical.
- **Guard the LOGIC signature, not the name:** the strong guard bans the duplicated *format* (`tile_${...}`) with a tiny path-allowlist, which beats banning helper names (false-positives on legit delegators, misses `normalizeTileCoord`/`coordKey`).
- After deleting a fallback, `npm run lint` flags orphaned `toInteger` (frontend keeps `toNumber`/`toInteger` copies per the shared-helpers guard's frontend exclusion ŌĆö leave those that stay used). Removing a dead require-shim shifts `catch (error)` suppression counts ŌåÆ re-run `npx eslint . --prune-suppressions`.

# 名人头像册与详情页落地任务 v0.1

日期：2026-05-30

目标版本：`0.1.166`

## 背景

`0.1.165` 已经实现自由属性点分配，但入口放在名人列表卡片内。实际体验上玩家不容易理解哪里能加点，且列表卡片同时承载立绘、技能、六维和按钮，后续继续塞内容会变成难维护的大卡片。

本批把名人面板拆成两层：

- 头像册：用于快速浏览所有已加入名人。
- 详情页：用于查看立绘、等级经验、技能、六维和属性点分配。

## 目标

- 名人主面板展示所有已加入名人的头像框、名字、等级和可分配点提示。
- 已加入名人按品质排序：传奇、英杰、良才、一般；同品质再按等级和名称排序。
- 不同品质使用明显外框：金、紫、蓝、白。
- 点击头像进入名人详情页。
- 详情页展示立绘、名称、称号、品质、等级经验、可分配属性点、下次属性点等级提示、六维蛛网图、两条技能。
- 只有详情页在 `freeAttributePoints > 0` 时显示六维 `+` 加点入口。

## 不做范围

- 不改变经验、升级、属性点发放规则。
- 不做撤销、重置、批量加点或二次确认。
- 不做品质自动成长属性写回。
- 不改名人生成、立绘素材、技能生成和战斗结算。

## 结构分工

- `UIStatePresenter`：负责品质映射、排序、详情选择、属性点提示和 renderer 所需 view state。
- `CanvasGameRenderer`：只按 view state 绘制头像册、详情页、品质边框、蛛网图、技能徽章和命中区。
- `CanvasActionController / CanvasActionDispatcher`：只处理本地 UI action：打开详情、关闭详情、翻页清空详情。
- `CanvasGameApp / CanvasGameShell`：保存 `selectedFamousPersonId`，保证 H5 shell 和 app render 参数一致。

这样后续继续扩展名人详情时，优先加 presenter 字段和小 renderer helper，避免把业务排序、属性点提示和 Canvas 绘制混成一坨。

## 落地范围

### Presenter

- 新增品质信息：`legendary/great/good/common` 映射到 `金/紫/蓝/白` 框和排序 rank。
- 新增 `sortFamousPeopleForRoster()`。
- `buildFamousPersonCard()` 输出：
  - `qualityLabel / qualityRank / qualityFrame`
  - `pointText`
  - `attributePointHint`
  - `nextAttributePointLevel`
  - `openDetailAction`
- `buildFamousPersonViewState()` 支持 `selectedPersonId`，输出 `selectedPerson`。

### Renderer

- 新增头像框绘制 helper：`drawFamousAvatarCard()`。
- 新增头像册分页网格：`renderFamousRosterGrid()`。
- 新增详情页：`renderFamousPersonDetail()`。
- 列表页不再绘制六维 `+`；详情页继续复用 `drawFamousAttributePointControls()`。

### 交互状态

- `openFamousPersonDetail`：点击头像进入详情。
- `closeFamousPersonDetail`：详情页返回头像册。
- 翻页、关闭面板、切换页签、寻访/接纳/放弃候选时清空选中名人。
- 加点成功后保持当前名人详情页打开。

## 验证点

- 名人列表按品质和等级排序。
- 品质框颜色明显，且 `legendary/great/good/common` 分别映射到金/紫/蓝/白。
- 头像册只显示头像、名称、等级和可分配点数字，不显示六维 `+`。
- 点击头像进入详情页。
- 详情页显示 `可分配属性点 0`，即使当前没有点也不会让玩家误以为入口丢失。
- 无点时显示 `下次属性点：Lv.X`。
- 有点时详情页六维显示 `+`，点击后属性点写回并保持详情页。
- 技能详情悬停/点击仍可用。
- 翻页和关闭面板会清理详情页与技能 tooltip 状态。

# 名人立绘三层方案 v3.0

## 当前结论

名人立绘停止使用旧的拆分头发方案，不再维护 `hairBase`、`bangs`、`sideHair`、`outfitBack`、`outfitFront` 等分层。

当前方案只保留三层：

```text
outfit -> face -> hair
```

每层 10 张 PNG：

- `fp-layer-v3-outfit-01.png` 到 `fp-layer-v3-outfit-10.png`
- `fp-layer-v3-face-01.png` 到 `fp-layer-v3-face-10.png`
- `fp-layer-v3-hair-01.png` 到 `fp-layer-v3-hair-10.png`

组合数量是 `10 * 10 * 10 = 1000`。

## 运行时约定

- 后端 `FamousPersonService` 只生成 `outfit`、`face`、`hair` 三个字段。
- 外观版本为 `famous-portrait-v3.0`，旧版本名人会在规范化时重新生成三层外观。
- 前端游戏渲染和实验台都读取 `FamousPortraitLayout.js`，避免再出现实验台和游戏两套坐标逻辑。
- 当前素材是 v3 占位 PNG，重点是先清空旧逻辑并打通新链路，后续美术只需要替换同名 PNG。

## 清理范围

已清理旧 v2 素材目录、候选素材、备份素材和旧实验台逻辑。后续不要恢复旧的 v2 manifest 或拆发型脚本。

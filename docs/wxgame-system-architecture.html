wxgame 真实系统架构

Canvas/H5 客户端 + Node/Express 权威服务端 + 独立世界周期任务 + 双 SQLite 写锁域

权威写入路径

  • Canvas 操作 → 游戏接口命令信封
  • JWT 路由 → 所有者解析 → 幂等存储
  • 命令定义执行后通过数据仓储层持久化

状态同步与渲染

  • 游戏状态同步服务轮询心跳，并按需刷新权威状态
  • 游戏操作投影生成客户端读取模型
  • H5 游戏主机应用状态后驱动画面、世界地图和界面渲染器

后台世界演进

  • 独立世界后台任务仅处理最近活跃玩家
  • 城市、探索、事件、人物社交与阵营外交统一走命令管线
  • 业务状态与高频观测日志拆分为独立 SQLite 写锁域

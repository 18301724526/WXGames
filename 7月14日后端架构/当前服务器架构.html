wxgame 当前服务器架构

事实快照：2026-07-14 · 生产提交 29e01cc4 · 单台阿里云 ECS · Nginx + PM2 + Node/Express + SQLite WAL。图中只陈述已从运行进程、端口、文件哈希、部署输出和当前源码验证的关系，不包含优化结论。

已验证拓扑

  • 单台 ECS 同时承载 Nginx、Git HTTP、生产、测试和重构进程
  • 生产 API 与 world worker 是两个独立 PM2 进程
  • 生产静态目录与部署 work tree 使用同一路径
  • 线上 backend 与本地提交逐文件一致

并发事实

  • API 与 worker 共享一个 civilization.db
  • WAL 支持并发读取，但所有写入仍属于单 writer domain
  • owner_locks 是逻辑资源锁，也保存在同一 SQLite 主库
  • observability 已拆到独立数据库

访问与安全事实

  • Nginx 暴露 80/443，Git HTTP 使用 3001
  • 生产 Node 进程绑定 *:3000；测试/重构绑定 *:3002/*:3003
  • 玩家接口使用 JWT，Git 推送使用 HTTP Basic Auth
  • SSH 与同机多环境隔离策略需单独评审

待架构评审

  • 逻辑 owner 并发模型是否匹配 SQLite 物理写锁模型
  • 后台周期任务与交互请求是否需要统一写入调度
  • 部署是否具备失败不影响线上静态目录的原子性
  • 容量、可观测性、备份恢复与安全边界是否足够
